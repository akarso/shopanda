# PR-103 — Permission middleware + UI

## Summary

Adds `RequirePermission()` HTTP middleware, permission-gated admin schemas,
and plugin permission registration. Admin routes now enforce fine-grained
permissions instead of a blanket `RequireRole(admin)` check.

## Changes

### `internal/interfaces/http/auth_middleware.go`
- Added `RequirePermission(perm rbac.Permission) Middleware` — rejects guests
  with 401 and authenticated users lacking the permission with 403.

### `internal/domain/admin/registry.go`
- Added `formPermissions` and `gridPermissions` maps to `Registry`.
- Added `SetFormPermission`, `FormPermission`, `SetGridPermission`,
  `GridPermission` methods (thread-safe via existing RWMutex).

### `internal/interfaces/http/schema_handler.go`
- `GetForm()` and `GetGrid()` now check the registry for a required
  permission after the schema lookup. Returns 403 if the caller's role
  does not grant the permission.

### `internal/domain/rbac/plugin_permissions.go` (new)
- `RegisterPluginPermission(perm, roles...)` — lets plugins define new
  permissions at startup. Rejects empty permissions and core-permission
  overrides.
- `ResetPluginPermissions()` — test helper to clear plugin state.

### `internal/domain/rbac/role_permissions.go`
- `HasPermission` now checks both core (static) and plugin-registered
  permissions.

### `internal/platform/plugin/plugin.go`
- Added `RegisterPermission(perm, roles...)` to `App`, delegating to
  `rbac.RegisterPluginPermission`.

### `cmd/api/main.go`
- Replaced `requireAdmin` (single role check) with per-permission
  middleware: `requireProductsRead`, `requireProductsWrite`,
  `requireOrdersRead`, `requireMediaWrite`, `requireSettingsRead`,
  `requireSettingsWrite`.
- Schema endpoints (`/admin/forms/{name}`, `/admin/grids/{name}`) now use
  `requireAuth` + internal permission check.
- Permissions assigned to schemas: `product.form` → `products.write`,
  `product.grid` → `products.read`, `page.form` → `settings.write`,
  `page.grid` → `settings.read`.

## Route → Permission mapping

| Route pattern | Permission |
|---|---|
| `GET /admin/products` | `products.read` |
| `POST /admin/products` | `products.write` |
| `PUT /admin/products/{id}` | `products.write` |
| `POST /admin/products/{id}/variants` | `products.write` |
| `PUT /admin/products/{id}/variants/{variantId}` | `products.write` |
| `GET /admin/orders` | `orders.read` |
| `GET /admin/orders/{orderId}` | `orders.read` |
| `POST /admin/media/upload` | `media.write` |
| `GET /admin/forms/{name}` | per-schema (auth required) |
| `GET /admin/grids/{name}` | per-schema (auth required) |
| `GET /admin/pages` | `settings.read` |
| `POST /admin/pages` | `settings.write` |
| `PUT /admin/pages/{id}` | `settings.write` |
| `DELETE /admin/pages/{id}` | `settings.write` |
| `GET /admin/stores` | `settings.read` |
| `POST /admin/stores` | `settings.write` |
| `PUT /admin/stores/{id}` | `settings.write` |
