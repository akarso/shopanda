# PR-056 — Jobs Queue System

## Summary

Introduces a durable job queue with a domain-level `Queue` interface, PostgreSQL implementation using `FOR UPDATE SKIP LOCKED`, and a polling `Worker` that dispatches jobs to registered handlers.

## Changes

### Domain (`internal/domain/jobs/`)

- **`job.go`** — `Job` struct (ID, Type, Payload, Status, Attempts, MaxRetries, RunAt, timestamps), `Status` type with four constants (`pending`, `processing`, `done`, `failed`), `Queue` interface (Enqueue, Dequeue, Complete, Fail), `Handler` interface (Type, Handle), `NewJob` constructor with validation, `DefaultMaxRetries = 3`.
- **`worker.go`** — `Worker` with handler registry, configurable poll interval, context-based cancellation, and `Stop()` signal. `processNext` dequeues one job, looks up handler by type, executes, then calls Complete or Fail.
- **`job_test.go`** — 5 unit tests for domain constructor and status constants.
- **`worker_test.go`** — 6 unit tests covering happy path, unknown type, handler error, stop signal, duplicate handler panic, and empty queue.

### Migration

- **`migrations/017_create_jobs.sql`** — `jobs` table with UUID primary key, `type`, `payload` (JSONB), `status`, `attempts`, `max_retries`, `run_at`, timestamps. Partial index `idx_jobs_dequeue` on `(created_at) WHERE status = 'pending'` for efficient dequeue.

### Infrastructure (`internal/infrastructure/postgres/`)

- **`job_queue.go`** — `JobQueue` implementing `jobs.Queue`. Enqueue inserts with JSON-marshaled payload. Dequeue uses `BEGIN` → `SELECT ... FOR UPDATE SKIP LOCKED LIMIT 1` → `UPDATE status='processing'` → `COMMIT` for concurrent-safe claiming. Fail implements retry with 10-second delay or permanent failure when attempts exhausted.
- **`job_queue_test.go`** — 7 integration tests (Enqueue, Dequeue, Dequeue_Empty, Complete, Fail_Retry, Fail_Permanent, Complete_NotFound).

### Wiring (`cmd/api/main.go`)

- Creates `JobQueue` and `Worker` instances.
- Starts worker goroutine with cancellable context before `ListenAndServe`.
- Placeholder comment for future handler registration.

### Diagrams

- **`c4-code.md`** — Added `Job`, `Status`, `Queue` interface, `Handler` interface, `Worker`, `JobQueue` classes with relationships.
- **`c4-component.md`** — Added `PostgresJobQueue` (infrastructure) and `JobWorker` (platform) components with relationships.

## Architecture Notes

- `Queue` is a domain port; `JobQueue` is the Postgres adapter — follows hexagonal pattern.
- `FOR UPDATE SKIP LOCKED` enables multiple workers to dequeue concurrently without blocking.
- Retry uses a fixed 10-second delay (v0); exponential backoff can be added later.
- Worker lives in domain layer alongside the `Queue` interface it depends on.
