# PR-212: Free Shipping Threshold

## Summary

Per-zone configurable free shipping threshold. When a cart subtotal meets or exceeds the zone's threshold, a free shipping rate option is returned alongside the paid rate.

## Changes

| File | Change |
|------|--------|
| `internal/domain/shipping/shipment.go` | Added `MethodFreeShipping = "free_shipping"` constant, updated `IsValid` |
| `internal/domain/shipping/zone.go` | Added `FreeShippingThreshold shared.Money` field to `Zone` struct (zero value = disabled) |
| `internal/domain/shipping/zone_rate.go` | Added `Subtotal shared.Money` to `RateRequest`; changed `CalculateRate` return to `[]ShippingRate`; added `qualifiesForFreeShipping` helper |

| `internal/interfaces/http/shipping_zone_admin_test.go` | Added 5 threshold admin CRUD tests |

## Design Decisions

- **`[]ShippingRate` return**: `CalculateRate` now returns a slice so both paid and free rates coexist. Existing callers get rates[0] for the paid rate.
- **Zero value = disabled**: `FreeShippingThreshold` with amount 0 means no free shipping. No pointer needed.
- **Currency matching**: Free shipping only applies when threshold currency matches request currency (case-insensitive).
- **DB storage**: Two columns (`free_shipping_threshold BIGINT`, `free_shipping_currency TEXT`) with CHECK >= 0. Empty currency when threshold is 0.
