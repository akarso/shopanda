# PR-029 -- Event Bus (In-Process)

## Summary

Adds an in-process event bus with synchronous and asynchronous dispatch,
following the envelope structure defined in docs/EVENT_SYSTEM.md.

## Files Changed

| File | Change |
|------|--------|
| internal/platform/event/event.go | New -- Event envelope, New(), WithMeta(), WithVersion() |
| internal/platform/event/bus.go | New -- Bus with On(), OnAsync(), Publish(), Handlers() |
| internal/platform/event/bus_test.go | New -- 13 tests |

## Design Decisions

- **Platform package** -- The event bus is cross-cutting infrastructure, placed
  alongside logger, config, and auth in internal/platform/event/.
- **Event envelope** -- Event struct with ID, Name, Version, Timestamp,
  Source, Data (interface{}), Meta (map[string]string). Matches the
  JSON envelope spec in EVENT_SYSTEM.md.
- **Two-phase dispatch** -- Publish() runs sync handlers first (in registration
  order). If any sync handler errors, it returns immediately and async handlers
  are skipped. Async handlers run in separate goroutines; errors are logged but
  not propagated.
- **Handler type** -- func(ctx context.Context, evt Event) error. Simple
  function type, no interface wrapping needed.
- **Thread safety** -- Handler maps protected by sync.RWMutex; registration
  locks for write, publish locks for read.
- **Immutable helpers** -- WithMeta() and WithVersion() return copies,
  leaving the original event unchanged.

## Tests

13 tests covering:

- Event envelope: creation, WithMeta, WithVersion
- Sync handlers: invocation, registration order, error aborts remaining
- Async handlers: invocation, error does not propagate
- Mixed: sync error skips async, sync+async combined
- Edge cases: no handlers, handler count, data type assertion
