# PR-076 — Category Import/Export

## Summary

Adds CSV-based category import and export commands (`import:categories`, `export:categories`). Categories are hierarchical (adjacency list via `parent_id`), so both importer and exporter handle tree ordering.

## Changes

| `internal/application/exporter/categories.go` | `CategoryExporter` — loads all categories, writes CSV in tree order (parents before children) |
| `internal/application/importer/categories_test.go` | 18 tests for the category importer |

## CSV Format

```csv
name,slug,parent_slug,position
Electronics,electronics,,0
Phones,phones,electronics,1
Smartphones,smartphones,phones,0
```

- **name** — required, category display name
- **slug** — required, unique identifier
- **parent_slug** — optional, references another category's slug (from same file or existing DB)
- **position** — optional, sort order (default 0)

## Design Decisions

1. **Parent reference by slug** — more user-friendly than opaque IDs; resolved via `FindBySlug`
2. **Topological sort** — Kahn's algorithm ensures parents are inserted before children, with cycle detection
3. **Upsert semantics** — existing slugs are updated; new slugs create categories with generated IDs
4. **No explicit transaction** — follows the customer importer pattern; CategoryRepo uses `*sql.DB` directly
5. **Tree-order export** — DFS traversal ensures parents always appear before children in the CSV, making the output directly re-importable
6. **Formula injection protection** — all string fields sanitized via `sanitizeCSVCell`
