# PR-069 — Product Page Template

## Summary

Adds server-side rendered (SSR) product page support via a default theme template
that consumes the PDP composition pipeline.

## Architecture

```
GET /products/{slug}
  → StorefrontHandler.Product()
    → repo.FindBySlug(slug)
    → composition.NewProductContext(product)
    → pdp.Execute(ctx)
    → engine.Render(w, "product", ctx)
  ← text/html response
```

The SSR route is **gated by `frontend.enabled`** (default: `false`). When disabled,
the server runs in headless/API-only mode with no theme overhead.

## Configuration

```yaml
frontend:
  enabled: true
  mode: ssr           # ssr | headless | hybrid
  theme_path: themes/default
```

Environment variables: `SHOPANDA_FRONTEND_ENABLED`, `SHOPANDA_FRONTEND_MODE`, `SHOPANDA_FRONTEND_THEME_PATH`.

## Template Data

The product template receives a `*composition.ProductContext`:

- `Product.Name`, `Product.Description`, `Product.Status`
- `Currency`, `Country`
- `Blocks` (populated by PDP pipeline steps / plugins)
- `Meta` (arbitrary metadata map)

## Tests

5 tests covering:

1. **Product_OK** — renders HTML with product name and description
2. **Product_NotFound** — returns 404 when repo returns nil
3. **Product_NotFoundError** — returns 404 when repo returns apperror.NotFound
4. **Product_RepoError** — returns 500 on internal error
5. **Product_WithPipeline** — verifies PDP pipeline executes before rendering
