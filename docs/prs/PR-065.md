# PR-065 — Cache Cleanup Scheduled Job

## Summary

Registers a `cache.cleanup` cron job that runs every 5 minutes, enqueuing a job into the existing job queue. A new `CleanupHandler` processes these jobs by calling `DeleteExpired()` on the cache store to remove stale entries.

## Changes

### Application Layer
- `internal/application/cache/handler.go` — `CleanupHandler` implementing `jobs.Handler`. Accepts an `ExpiredDeleter` interface (satisfied by `*postgres.CacheStore`). Calls `DeleteExpired()` and logs the count of removed entries. Exports `JobType = "cache.cleanup"` constant.

### CLI Wiring (`cmd/api/main.go`)
- **`runServe`**: Keeps `*postgres.CacheStore` reference alongside `cache.Cache` interface so the handler can access `DeleteExpired()`. Registers `cacheApp.NewCleanupHandler(cacheStore, log)` with the job worker.
- **`runScheduler`**: Activates the previously commented-out cron registration — `sched.Register("cache.cleanup", "*/5 * * * *", ...)` — which creates and enqueues a `cache.cleanup` job. Errors from `NewJob` and `Enqueue` are now logged instead of discarded.

### Tests
- `internal/application/cache/handler_test.go` — 5 unit tests: Type, Handle_Success, Handle_Error, PanicsOnNilDeleter, PanicsOnNilLogger. Uses stub implementations of `ExpiredDeleter` and `Logger`.

### Diagrams
- `docs/diagrams/c4-component.md` — Added `CacheCleanupHandler` component in application layer, relationships from `JobWorker` and to `pgCacheStore`
- `docs/diagrams/c4-code.md` — Added `CacheCleanupHandler` class, `Handler` implementation relationship, dependency on `PostgresCacheStore`

## Architecture

```text
CronScheduler --("*/5 * * * *")--> JobQueue.Enqueue(cache.cleanup)
JobWorker --poll--> JobQueue.Dequeue
JobWorker --dispatch--> CleanupHandler.Handle
CleanupHandler --> CacheStore.DeleteExpired
```

## Dependencies
- No new external dependencies
