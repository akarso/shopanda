# PR-005 — Platform: ID Generation

## Summary

Adds a UUID v4 helper package at `internal/platform/id/` using `crypto/rand`. No external dependencies.

## Files

| File | Change |
|------|--------|
| `internal/platform/id/id.go` | New — `New()` returns UUID v4 string, `IsValid()` validates format |
| `internal/platform/id/id_test.go` | New — 5 tests (valid output, uniqueness, format, accept valid, reject invalid) |

## Public API

```go
// New generates a new UUID v4 string.
func New() string

// IsValid reports whether s is a valid UUID v4 string.
func IsValid(s string) bool
```

## Design Decisions

- **No external dependency** — UUID v4 is trivial to implement with `crypto/rand` + RFC 4122 bit masking.
- **String return type** — keeps the API simple; a dedicated `ID` type can be introduced later if needed.
- **`IsValid()`** — included for API input validation at system boundaries (future handlers will accept UUID path params).
- **Panic on rand failure** — `crypto/rand.Read` only fails if the OS entropy source is broken, which is unrecoverable.

## Tests

5 tests:

| Test | Validates |
|------|-----------|
| `TestNew_ReturnsValidUUID` | Output passes `IsValid` check |
| `TestNew_ReturnsUniqueValues` | 1000 consecutive calls produce no duplicates |
| `TestNew_HasCorrectFormat` | Length 36, dashes at positions 8/13/18/23, version nibble `4`, variant nibble `[89ab]` |
| `TestIsValid_AcceptsValidUUID` | Known-good UUID passes |
| `TestIsValid_RejectsInvalidStrings` | Empty, truncated, wrong version, wrong variant, no-dash formats all rejected |
