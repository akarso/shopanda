# PR-096 — Translation System

## Summary

Introduces the system translation infrastructure: a `translations` table, a `t()` function backed by a repository, language resolution middleware, and a `Language` field on the Store domain entity.

## Changes

### Domain — `internal/domain/translation/`

| File              | Purpose                                        |
|-------------------|-------------------------------------------------|
| `translation.go`  | `Translation` entity (`Key`, `Language`, `Value`) with `NewTranslation` constructor + validation |
| `repository.go`   | `TranslationRepository` interface (`FindByKeyAndLanguage`, `ListByLanguage`, `Upsert`, `Delete`) |
| `context.go`      | `WithLanguage` / `LanguageFromContext` context helpers (defaults to `"en"`) |
| `translator.go`   | `Translator.T(ctx, key)` — resolves language from context, returns translated value or key as fallback |

### Domain — `internal/domain/store/`

| File       | Change |
|------------|--------|
| `store.go` | Added `Language` field to `Store` struct; added `NormalizeLanguage()` (lowercases, validates 2- or 5-char BCP 47 tags); updated `NewStore` and `NewStoreFromDB` signatures |

### Infrastructure — `internal/infrastructure/postgres/`

| File                    | Purpose |
|-------------------------|---------|
| `translation_repo.go`  | PostgreSQL implementation of `TranslationRepository` (find, list, upsert with `ON CONFLICT`, delete) |
| `store_repo.go`        | Updated `storeColumns`, `hydrateStore`, `Create`, `Update` to include `language` column |

### Interfaces — `internal/interfaces/http/`

| File                       | Purpose |
|----------------------------|---------|
| `language_middleware.go`   | Language resolution: query param `?lang=` → `Accept-Language` header → `store.Language` → `"en"` default |
| `store_admin.go`           | Added `Language` to `createStoreRequest` / `updateStoreRequest`; wired into `NewStore` call and Update handler |

### Migration

| File                                | Purpose |
|-------------------------------------|---------|
| `031_create_translations.sql`       | `CREATE TABLE translations (key, language, value)` with composite PK; `ALTER TABLE stores ADD COLUMN language` |

### Wiring — `cmd/api/main.go`

- Added `translationRepo` creation (placeholder; wired in admin PR)
- Added `LanguageMiddleware()` after `StoreMiddleware`

## Language Resolution Priority

```
1. ?lang=de          (query parameter)
2. Accept-Language   (HTTP header, first 2/5-char tag)
3. store.Language    (from store context)
4. "en"              (hardcoded default)
```
