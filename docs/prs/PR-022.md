# PR-022 — Dev Auth Provider

## Summary

Implements a development-only `TokenParser` using simple `userID:role` tokens (no cryptography), plus test helpers for injecting authenticated identities into HTTP test requests. Wires the dev parser into the HTTP server middleware chain.

## Scope

Phase 4 — Minimal Auth (second PR).

## Changes

| File | Description |
|------|-------------|
| `doc.go` | Package documentation (dev-only warning) |
| `parser.go` | `Parser` — implements `auth.TokenParser`; parses `userID:role` tokens (last colon is delimiter); rejects guest role tokens; `Token()` helper generates dev token strings |
| `parser_test.go` | 9 tests: valid customer, valid admin, missing colon, empty string, invalid role, empty userID, guest role rejection, colon in userID, Token() generation |

| `testhelper_test.go` | 3 tests: full round-trip (header + context), admin shortcut, customer shortcut |

### Wiring: `cmd/api/main.go`

- Added `devauth` import
- Created `devauth.NewParser()` and registered `AuthMiddleware(tokenParser)` in the middleware chain (after logging, before route handlers)

## Token Format

```
userID:role
```

- Last colon is the delimiter (userIDs may contain colons)
- Role must be `customer` or `admin` (guest tokens rejected)
- No signing, no expiry — development only

### Examples

```
Authorization: Bearer user-1:customer
Authorization: Bearer admin-1:admin
Authorization: Bearer org:team:user-1:admin
```

## Test Helper Usage

```go
req := httptest.NewRequest("GET", "/orders/1", nil)
req = testhelper.AdminRequest(req, "admin-1")
// req now has Authorization header AND identity in context
```

## Architecture

```text
domain/identity  ←  platform/auth  ←  infrastructure/devauth
                     platform/auth/testhelper ──→ infrastructure/devauth
```

The dev parser is infrastructure (adapter); test helpers are platform utilities that compose devauth + auth context.
