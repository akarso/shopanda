# PR-071 — Import Pipeline Upgrade

## Summary

Upgrades the product CSV importer with attribute mapping, type-aware parsing,
group-based validation, and per-row error reporting. Any CSV column not in the
known set (`name`, `slug`, `sku`, `description`, `variant_name`) is treated as
an attribute column and stored in `variant.Attributes`.

## Design

### Attribute Column Detection

After parsing the CSV header, any column not in the `knownColumns` set is
identified as an attribute column. Empty attribute values in a row are omitted
from the attribute map.

### Type Parsing

When an `AttributeRegistry` is provided via `WithAttributeValidation`, values
are parsed according to the attribute's declared type:

```text
text    → string (as-is)
number  → float64 (via strconv.ParseFloat)
boolean → bool (accepts true/false, 1/0, yes/no, case-insensitive)
select  → string (validated against options by the registry)
```

Attributes not found in the registry fall back to raw strings.

### Group Validation

When both a registry and a group code are provided, the parsed attribute map is
validated against the group via `AttributeRegistry.ValidateAttributes`. This
enforces:

- Required attributes are present
- Undeclared keys are rejected
- Select values are within allowed options

Validation errors are reported per row with line numbers in `Result.Errors`.

### Backward Compatibility

The `NewProductImporter` constructor signature is unchanged. Attribute support
is opt-in via the `WithAttributeValidation` builder method. Without it, extra
columns are stored as raw strings with no validation — same effective behavior
as before for CSVs that only contain the known columns.

## Relationship to Existing Code

Builds on PR-070's `AttributeRegistry` for type information and group validation.
Parsed attributes are stored in `Variant.Attributes` (the existing
`map[string]interface{}` JSONB field). The CLI wiring in `main.go` does not yet
pass a registry — that can be added when attribute registration is configured at
startup.
