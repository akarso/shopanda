# PR-013 — Product HTTP Handlers (Read)

## Summary

Introduces read-only product HTTP endpoints (`List`, `Get`) served through the composition pipeline. Wires the product repository, PDP/PLP pipelines, and routes into `main.go`.

## Files

| File | Purpose |
|------|---------|
| `internal/interfaces/http/product.go` | `ProductHandler` — `List()` and `Get()` handlers, pagination parsing, response builders |
| `internal/interfaces/http/product_test.go` | 11 tests covering happy paths, errors, pagination, pipeline execution |
| `cmd/api/main.go` | Wires DB, `ProductRepo`, pipelines, handler, and routes in `runServe` |

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/products` | List products with `?offset=&limit=` pagination |
| `GET` | `/api/v1/products/{id}` | Get single product by ID |

## Design Decisions

- **Composition-first**: Both endpoints build a context (`ListingContext` / `ProductContext`), execute the pipeline, then serialize the enriched context. Plugins add steps to the pipeline later.
- **Pagination defaults**: `offset=0`, `limit=20`, max `100`. Invalid values return 422 before hitting the repo.
- **Pointer conversion for List**: `repo.List` returns `[]Product` by value; handler converts to `[]*Product` for `ListingContext` (shared pointer semantics per PR-012 fix).
- **Empty pipelines**: Pipelines are wired empty in `main.go`. No-op execution returns the raw data — plugins register steps to enrich responses.
- **Handler method pattern**: Each endpoint is a method returning `http.HandlerFunc`, matching the project convention from `HealthHandler()`.

## Response Shape

```json
// GET /api/v1/products
{ "data": { "products": [...], "blocks": [], "filters": [], "meta": {} }, "error": null }

// GET /api/v1/products/{id}
{ "data": { "product": {...}, "blocks": [], "meta": {} }, "error": null }
```

## Tests

11 tests:
- List: OK (2 products), default pagination (offset=0, limit=20), invalid offset (422), invalid limit (422), with pipeline step adding block, repo error (500)
- Get: OK (returns product), not found (404), repo error (500), with pipeline step adding block, pipeline error (500)
