# PR-048 — Plugin Interface + Loader

## Summary

Introduces the plugin system foundation: `Plugin` interface, `App` context struct, and `Registry` for loading and initializing plugins at startup.

## Design

### Plugin interface

```text
Plugin.Name() string         — unique identifier
Plugin.Init(app *App) error  — called once at startup; registers extensions
```

### App struct

Provides system facilities to plugins during initialization:

- `Logger` — structured logging
- `Bus` — event subscription (sync/async)
- `Config` — application configuration access

### Registry

- `Register(p Plugin)` — adds plugin in `loaded` state; panics on nil, empty name, or duplicate
- `InitAll(app *App) int` — calls `Init()` on each loaded plugin; returns count of successes
- Failed plugins are marked `failed` with error, logged, and skipped (system continues)
- `Entries()` returns a defensive copy of all entries with state
- Plugin states: `loaded` → `active` (success) or `failed` (error)

### Lifecycle

```text
load → init → active (passive thereafter)
         ↘ failed (logged, skipped)
```

### Wiring

Registry created after event bus, initialized before application services. No plugins registered yet — PR-049 adds the registration APIs (`RegisterPricingStep`, etc.).

## Test coverage
