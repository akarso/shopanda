# PR-003: Platform — structured logger

## Phase
Phase 0 — Project Scaffold

## Scope
JSON logger to stdout, `Logger` interface

## Files Created
- `internal/platform/logger/logger.go` — `Logger` interface (`Info`, `Warn`, `Error`), `JSONLogger` impl writing JSON lines to `io.Writer`, `Level` type with `ParseLevel`, `NewWithWriter` for testing, mutex-protected writes
- `internal/platform/logger/logger_test.go` — 10 tests: JSON output for all 3 levels, error message inclusion, nil error/context, level filtering (warn suppresses info, error suppresses info+warn), `ParseLevel` defaults, multiple writes, interface compliance

## Files Modified
- `cmd/api/main.go` — replaced `fmt.Println` with structured logger; `log.Info("app.config.loaded", ...)` and `log.Info("app.ready", nil)`

## Key Decisions
- No external dependency — uses `encoding/json` from stdlib
- Three levels: `info`, `warn`, `error` (per spec)
- Level filtering: logger at "warn" suppresses "info", at "error" suppresses both
- `NewWithWriter` constructor enables test injection without interfaces
- `entry` struct maps directly to spec's JSON format: `timestamp`, `level`, `event`, `message`, `context`
- Context propagation (request_id, user_id) deferred to PR-004

## Output Format
```json
{"timestamp":"2026-03-29T12:46:54Z","level":"info","event":"app.config.loaded","context":{"config":"server=0.0.0.0:8080 ..."}}
{"timestamp":"2026-03-29T12:46:54Z","level":"info","event":"app.ready"}
```
