# PR-079 — Core Seed Data

## Summary

Implements the three core seeders that populate a fresh database with usable
data: an admin user, store configuration, and a minimal product catalog.

This PR builds on the seed framework from PR-078 and registers the seeders
in the `runSeed` function in `main.go`.

## Changes

### New files

| File | Description |
|------|-------------|
| `internal/seed/admin.go` | `AdminSeeder` — creates `admin@example.com` with bcrypt-hashed password, `RoleAdmin` |
| `internal/seed/config.go` | `ConfigSeeder` — sets `default_currency` to `EUR` |
| `internal/seed/catalog.go` | `CatalogSeeder` — creates 2 categories, 3 products, 4 variants, prices, and stock |
| `internal/seed/seeders_test.go` | Interface checks, name tests, registration test, seed data validation |

### Modified files

| File | Change |
|------|--------|

- Products: `FindBySlug` — skip if exists, reuse ID for variants
- Variants: `FindBySKU` — skip if exists, reuse ID for prices/stock
- Prices: `Upsert` (SQL `ON CONFLICT`)
- Stock: `SetStock` (SQL `ON CONFLICT`)

## Execution Order

1. `store-config` — ensures currency is available
2. `admin-user` — creates admin account
3. `catalog` — creates categories → products → variants → prices → stock

## Architecture

Seeders create postgres repos from `Deps.DB` and use domain constructors
for entity creation. No new domain types or repository methods were added.

```
seed.Registry
  ├── ConfigSeeder  → config.Repository (Get/Set)
  ├── AdminSeeder   → customer.CustomerRepository (FindByEmail/Create)
  └── CatalogSeeder → catalog + pricing + inventory repos
```
