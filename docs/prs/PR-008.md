# PR-008 — Error Handling Foundation

## Scope

`apperror.Error` type, error-to-HTTP mapping, standard API response envelope.

## What Changed

### New: `internal/platform/apperror/`

- **`apperror.go`** — Structured `Error` type with `Code`, `Message`, and wrapped `Err`.
  - Six error codes: `not_found`, `conflict`, `validation`, `unauthorized`, `forbidden`, `internal`.
  - Constructor helpers: `New()`, `Wrap()`, `NotFound()`, `Conflict()`, `Validation()`, `Unauthorized()`, `Forbidden()`, `Internal()`.
  - `Is(err, code)` — checks whether any error in the chain matches a given code (uses `errors.As`).
  - `Error()` — formats as `"code: message"` (or `"code: message: wrapped"` when wrapping).
  - `Unwrap()` — supports standard `errors.Is` / `errors.As` chains.

- **`apperror_test.go`** — 7 tests covering `Error()`, `Unwrap`, `Is` (direct, wrapped, non-app-error), and all six constructors.

### New: `internal/interfaces/http/response.go`

- **`Response`** — Standard API envelope: `{"data": ..., "error": null}`.
- **`ErrorBody`** — Error portion: `{"code": "...", "message": "..."}`.
- **`JSON(w, status, data)`** — Writes a success response with the envelope.
- **`JSONError(w, err)`** — Maps `*apperror.Error` to the correct HTTP status + envelope. Falls back to 500 for unknown errors.
- **`StatusFromCode(code)`** — Maps `apperror.Code` → HTTP status:
  - `not_found` → 404
  - `conflict` → 409
  - `validation` → 422
  - `unauthorized` → 401
  - `forbidden` → 403
  - `internal` (default) → 500

- **`response_test.go`** — 4 tests: `JSON()` envelope, `JSONError()` with `apperror.Error`, `JSONError()` with generic error, `StatusFromCode` for all codes.

### Changed: `internal/interfaces/http/middleware.go`

- Recovery middleware now returns the standard envelope format:
  `{"data":null,"error":{"code":"internal","message":"internal server error"}}`
