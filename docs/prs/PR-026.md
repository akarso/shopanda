# PR-026 — Cart Postgres Repository

## Summary

Implements the PostgreSQL storage layer for the cart aggregate, including
migration 006 and the `CartRepo` type.

## Design Decisions

### Storage layout

- **`carts`** table: `id`, `customer_id` (nullable), `status` (CHECK constraint),
  `currency`, timestamps. Partial index on `(customer_id, status)` for the
  `FindActiveByCustomerID` query.
- **`cart_items`** table: composite PK `(cart_id, variant_id)` — domain Items
  have no separate ID. `unit_price` stored as `BIGINT` (smallest currency unit)
  matching `shared.Money` internals. CASCADE delete on `cart_id` FK.

### Status reconstruction

Cart's `status` field is unexported (PR-025 review). Added `SetStatusFromDB`
to allow the repository to reconstruct the aggregate from database rows without
bypassing domain invariants.

### Save semantics

`Save` is a transactional upsert: inserts or updates the cart header, then
deletes and re-inserts all items. This delete-and-reinsert approach is simple
and correct for the small item counts typical in carts.
