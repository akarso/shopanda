# PR-101 — Cache Invalidation

## Phase
28 — Performance & CDN

## Goal
Introduce event-driven cache invalidation so that product and price changes
automatically purge stale cache entries. Cache keys follow the convention
`product:<productID>:` allowing prefix-based deletion across all
store/language/currency variants.

## Design Decisions

- **Prefix-based deletion**: Cache keys are `product:<id>:<store>:<lang>:<currency>`. Invalidation deletes by `product:<id>:` prefix, clearing all locale variants in one operation.
- **Async event handlers**: Invalidation uses `bus.OnAsync` so cache misses don't block the write path.
- **ProductID in price event**: The importer already resolves the variant (which carries `ProductID`), so the event includes it directly — no extra lookup needed by the subscriber.
- **Nil-guarded bus**: The importer skips event publishing when bus is nil, keeping CLI imports simple.
