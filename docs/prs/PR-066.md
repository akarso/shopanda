# PR-066 — Admin Schema Registry

## Summary

Introduces the admin schema domain model and an in-memory registry for form and grid definitions. Plugins (and core) register schemas declaratively; the registry provides lookup for future API exposure and UI rendering.

## Changes

### Domain Layer (`internal/domain/admin/`)
- **`schema.go`** — Value types: `Field`, `Option`, `Form`, `Column`, `Action`, `Grid`. Fields carry type, label, required flag, default, select options, and metadata. Columns and actions include callback functions for value resolution and execution.
- **`registry.go`** — `Registry` struct with registration and lookup API:
  - `RegisterForm(name, Form)` / `RegisterFormField(formName, Field) error`
  - `RegisterGrid(name, Grid)` / `RegisterGridColumn(gridName, Column) error` / `RegisterAction(gridName, Action) error`
  - `Form(name) (Form, bool)` / `Grid(name) (Grid, bool)`
  - Appending to unknown forms/grids returns an error. Re-registering a form/grid replaces it.

### Tests
- **`registry_test.go`** — 11 unit tests covering: register + retrieve forms and grids, field/column/action appending, unknown-form/grid errors, not-found lookups, and form replacement.

### Diagrams
- `docs/diagrams/c4-component.md` — Added `AdminSchemaRegistry` component in domain layer with plugin relationship.
- `docs/diagrams/c4-code.md` — Added `AdminRegistry`, `Form`, `Field`, `Grid`, `Column`, `Action` classes with relationships.

## Architecture

```text
Plugin / Core  -->  Registry.RegisterForm("product.form", form)
Plugin / Core  -->  Registry.RegisterFormField("product.form", field)
Plugin / Core  -->  Registry.RegisterGrid("product.grid", grid)
Future handler -->  Registry.Form("product.form")  --> Form{Fields}
Future handler -->  Registry.Grid("product.grid")   --> Grid{Columns, Actions}
```

## Design Decisions

- **Struct-based registry** (not global functions) to avoid global state mutation per AGENTS.md.
- **Domain layer placement** — schemas are infrastructure-agnostic contracts, not HTTP or DB concerns.
- **Callback fields** on `Column.Value` and `Action.Execute` follow the spec; serialization for API exposure deferred to a future PR.
