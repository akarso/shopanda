# PR-043 — Manual Payment Provider

## Summary

Adds the built-in **manual payment provider** — the default, in-process
implementation of `payment.Provider` that immediately approves every
payment.

## Scope

| Area | Files |
|------|-------|
| Provider implementation | `internal/infrastructure/manualpay/provider.go` |
| Tests | `internal/infrastructure/manualpay/provider_test.go` |

## Design

The `manualpay.Provider` implements the `payment.Provider` interface
(defined in PR-042):

```go
type Provider interface {
    Method() PaymentMethod
    Initiate(ctx context.Context, p *Payment) (ProviderResult, error)
}
```

### Behavior

- `Method()` returns `payment.MethodManual`.
- `Initiate()` succeeds (`Success: true`) with a deterministic
  provider reference of the form `"manual:<paymentID>"` for any valid
  payment. Returns an error only if the payment pointer is nil.

### Placement

Lives under `internal/infrastructure/manualpay/` — a dedicated infrastructure
package, consistent with the hexagonal architecture (infrastructure implements
domain interfaces).

- `TestProvider_Method` — returns `MethodManual`
- `TestProvider_Initiate_OK` — success result with correct provider ref
