# PR-064 — Configuration: DB Layer

## Summary

Adds a database-backed configuration table with a domain repository interface, PostgreSQL adapter, and two CLI commands (`config:export`, `config:import`) for dumping/loading config entries.

## Changes

### Domain Layer
- `internal/domain/config/repository.go` — `Entry` struct (Key, Value) and `Repository` interface: `Get`, `Set`, `Delete`, `All`

### Infrastructure Layer
- `internal/infrastructure/postgres/config_repo.go` — `ConfigRepo` implementing `config.Repository` with PostgreSQL JSONB storage. Uses `INSERT ... ON CONFLICT` for upsert, returns `(nil, nil)` on cache miss, idempotent delete.

### Migration
- `migrations/020_create_config.sql` — Creates `config` table (`key TEXT PRIMARY KEY`, `value JSONB NOT NULL`)

### CLI Commands (wired in `cmd/api/main.go`)
- `config:export` — Reads all DB config entries via `ConfigRepo.All()`, expands dot-notation keys into nested YAML, writes to stdout
- `config:import <file.yaml>` — Reads a YAML file, flattens nested keys to dot-notation, upserts each entry into the DB via `ConfigRepo.Set()`
- Helper `flattenYAML()` — Recursively converts nested map to flat dot-notation key-value pairs

### Tests
- `internal/infrastructure/postgres/config_repo_test.go` — 8 integration tests (`postgres_test` package): SetAndGet, GetMiss, Upsert, Delete, DeleteMissing, All, AllEmpty, ComplexValue. Uses `setupConfigRepo(t)` helper with `testDB(t)` + `migrate.Run`.

### Diagrams
- `docs/diagrams/c4-component.md` — Added `pgConfigRepo` component and DB relationship
- `docs/diagrams/c4-code.md` — Added `ConfigRepository` interface, `PostgresConfigRepo` adapter, and implements relationship

## Architecture

```
domain/config/Repository  ←  infrastructure/postgres/ConfigRepo  ←  cmd/api (CLI commands)
                                        ↓
                                   PostgreSQL (config table)
```
