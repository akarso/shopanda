# PR-054 — Postgres Search Implementation

## Summary

Implements the `search.SearchEngine` interface (PR-053) using PostgreSQL full-text search with `tsvector`, filtering, sorting, and category facets.

## Changes

### Migration

- `migrations/016_add_search_vector.sql`
  - Adds `search_vector tsvector` column to `products` table
  - Backfills existing products with `to_tsvector('english', name || ' ' || description)`
  - Creates GIN index `idx_products_search_vector` for fast full-text queries
  - Creates trigger `products_search_vector_trigger` to auto-update the vector on name/description changes

### Infrastructure Adapter

- `internal/infrastructure/postgres/search_engine.go` — `SearchEngine`
  - `Name()` → returns `"postgres"`
  - `IndexProduct(ctx, product)` → explicitly updates `search_vector` (useful for reindexing)
  - `RemoveProduct(ctx, productID)` → clears `search_vector` to NULL
  - `Search(ctx, query)` → full-text search with:
    - Text matching via `plainto_tsquery` + `ts_rank` relevance ordering
    - Category filter via `product_categories` JOIN
    - Price range filter via `variants`/`prices` JOINs (supports `gt`, `gte`, `lt`, `lte`)
    - Sort options: `name`, `-name`, `created_at`, `-created_at`, relevance (default with text)
    - Pagination via `LIMIT`/`OFFSET`
    - Category facets computed from the filtered result set
    - Only returns `active` products with non-NULL search vectors

## Design Decisions

- **Trigger-based indexing**: The trigger auto-updates `search_vector` on any product name/description change, so normal CRUD operations keep the index current without explicit calls. `IndexProduct` exists for batch reindex scenarios.
- **Category facets only (v0)**: Attribute-based facets from JSONB would add significant query complexity. Category counts cover the most common use case.
- **search_vector NULL = removed**: `RemoveProduct` sets the column to NULL rather than deleting the row, keeping the search engine decoupled from product lifecycle.
- **DISTINCT on results**: Prevents duplicate rows when price/category JOINs match multiple variants or categories per product.
