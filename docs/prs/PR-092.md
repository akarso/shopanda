## Design

### Meta tag steps

- `ProductMetaStep`: title = product name, description = product description (truncated to 160 chars)
- `ListingMetaStep`: title = "Products", description = "Browse N products"
- Both are pure transformations — no I/O dependencies

### JSON-LD step

- Produces Schema.org `Product` with `Offer` containing price and availability
- Dependencies: `VariantRepository`, `PriceRepository`, `StockRepository`
- Fetches first variant for the product, then price (in context currency, defaulting to EUR) and stock
- Graceful degradation: missing data is simply omitted from the JSON-LD block
- Price formatted as decimal string (e.g. "29.99") per Schema.org spec

### Context enhancement

- Added `Ctx context.Context` to `ProductContext` and `ListingContext` to enable I/O-bound composition steps
- Backward-compatible: defaults to `context.Background()`, existing callers unchanged
