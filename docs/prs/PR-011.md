# PR-011 — Product Postgres Repository

## Summary

Implements `catalog.ProductRepository` against PostgreSQL, plus the initial `products` table migration.

## Files

| File | Purpose |
|------|---------|
| `migrations/001_create_products.sql` | Creates `products` table with UUID PK, UNIQUE slug, JSONB attributes, TIMESTAMPTZ timestamps |
| `internal/infrastructure/postgres/product_repo.go` | `ProductRepo` struct implementing all 5 repository methods |
| `internal/infrastructure/postgres/product_repo_test.go` | 10 integration tests (skipped without `SHOPANDA_TEST_DSN`) |

## Design Decisions

- **Compile-time interface check**: `var _ catalog.ProductRepository = (*ProductRepo)(nil)` ensures the struct satisfies the interface.
- **`(nil, nil)` for not found**: `FindByID` and `FindBySlug` return `(nil, nil)` when no row matches, per the repository contract.
- **Validation in `List`**: Rejects negative offset and non-positive limit with `apperror.Validation`. Caps limit at 100.
- **`Update` returns not_found**: Uses `RowsAffected()` to detect zero rows and returns `apperror.NotFound`.
- **JSONB round-trip**: Attributes are marshalled to JSON on write and unmarshalled on read; empty attributes default to `{}`.
- **Scanner abstraction**: Private `scanner` interface satisfied by both `*sql.Row` and `*sql.Rows` to share scan logic.
- **Integration tests skip cleanly**: All tests use `t.Skip` when `SHOPANDA_TEST_DSN` is unset.

## Migration Schema

```sql
CREATE TABLE products (
    id          UUID PRIMARY KEY,
    name        TEXT NOT NULL,
    slug        TEXT UNIQUE NOT NULL,
    description TEXT NOT NULL DEFAULT '',
    status      TEXT NOT NULL DEFAULT 'draft',
    attributes  JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## Tests

10 integration tests covering:
- Create + FindByID round-trip
- FindByID not found → (nil, nil)
- FindBySlug round-trip
- FindBySlug not found → (nil, nil)
- List with pagination (offset/limit)
- List validation errors (negative offset, zero/negative limit)
- Update + verify changes
- Update not found → apperror.NotFound
- Create duplicate slug → error
- Attributes JSONB round-trip
