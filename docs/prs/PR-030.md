## Design Decisions

- **Post-event semantics** â€” All events use the `-ed` suffix convention from
  EVENT_SYSTEM.md (e.g. `cart.item.added`). They are emitted *after* the
  write succeeds. Publish errors are intentionally ignored (`_ = bus.Publish`)
  because the core operation already committed.
- **Cart service (application layer)** â€” Events emitted from the service,
  which is the correct orchestration layer.
- **Product/Variant handlers (interface layer)** â€” No separate catalog
  application service exists yet. Events are emitted from handlers that
  currently act as the write path. When a catalog service is introduced,
  event emission can move there.
- **Inventory skipped** â€” No inventory write path (service or handler) exists
  yet. Events will be wired when the inventory service is created.
- **Importer skipped** â€” The CSV importer has complex multi-branch transaction
  logic. Wiring events there would significantly expand the PR scope.
