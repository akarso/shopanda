# PR-072 — Export Pipeline

## Summary

Adds a `ProductExporter` that writes all products and their variants to CSV,
plus the `export:products` CLI command to invoke it.

## Changes

### New: `internal/application/exporter/products.go`

- `ProductExporter` struct using `ProductRepository` and `VariantRepository`
- `Export(ctx, w io.Writer) (*Result, error)` — paginates through all products,
  fetches variants per product, discovers attribute keys dynamically, sorts
  attribute columns alphabetically, writes CSV with header + data rows
- `formatAttrValue` helper for type-aware value formatting (string, float64, int,
  int64, bool, nil)
- `Result` struct with `Products` and `Variants` counts

### New: `internal/application/exporter/products_test.go`

8 tests:
- `TestExport_BasicCSV` — 2 products, 3 variants, correct CSV structure
- `TestExport_WithAttributes` — dynamic attribute columns sorted alphabetically, type formatting
- `TestExport_EmptyDatabase` — header-only output
- `TestExport_ProductWithNoVariants` — product without variants produces no rows
- `TestExport_AttributeNilValuesAreEmpty` — missing attributes are empty strings
- `TestExport_ListProductsError` — propagates product listing error
- `TestExport_ListVariantsError` — propagates variant listing error
- `TestExport_RoundTrip` — full field-by-field verification of a single exported row

### Modified: `cmd/api/main.go`

- Added `export:products` CLI subcommand with `runExportProducts` function
- Usage: `app export:products <file.csv>`
- Opens DB, creates repos, writes CSV to file, logs summary

### Modified: `docs/diagrams/c4-component.md`

- Added `ProductExporter` component in the Application layer
- Added relationship: `ProductExporter → PostgreSQL Repositories`

## Architecture

```text
CLI (main.go)
  → ProductExporter (application layer)
    → ProductRepository.List (paginated)
    → VariantRepository.ListByProductID (per product)
    → CSV writer (io.Writer)
```

## CSV Format

Columns: `name, slug, sku, description, variant_name, [sorted attribute columns...]`

One row per variant. Attribute columns are discovered dynamically from all
variants and sorted alphabetically for deterministic output.
