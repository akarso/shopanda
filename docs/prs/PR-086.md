# PR-086 — Promotion HTTP Endpoints + Catalog Pricing

## Summary

Integrates promotions into the pricing pipeline and adds coupon endpoints to
the cart API. Catalog promotions apply per-item discounts automatically (or via
coupon). Coupon codes can be applied/removed on a cart via
`POST/DELETE /api/v1/carts/{cartId}/coupon`.

## Design

### CatalogPromotionStep

Inserted in the pricing pipeline after `BasePriceStep`, before `TaxStep`:

```text
base → catalog_promotions → tax → plugins → finalize
```

For each active catalog promotion:
1. Check `IsEligible(now)` (active + time window)
2. If `CouponBound`, verify `Meta["coupon_code"]` matches a valid coupon
3. Decode `Conditions` JSON — supported types: `always`, `min_quantity`
4. Decode `Actions` JSON — supported types: `percentage`, `fixed`
5. For each matching item, compute discount and add `AdjustmentDiscount`

Promotions are evaluated in priority order (from `ListActive`). Multiple
promotions stack — each one reduces the item total cumulatively.

### Condition/action JSON format

```json
// Conditions
{"type": "always"}
{"type": "min_quantity", "value": 3}

// Actions
{"type": "percentage", "percentage": 10}
{"type": "fixed", "amount": 500}
```

Null/empty conditions default to `always`. Fixed discounts are per-unit
(multiplied by quantity) and capped at item total.

### Coupon flow

```text
POST /api/v1/carts/{cartId}/coupon  { "code": "SAVE20" }
  → validate coupon exists + CanRedeem()
  → validate promotion exists + IsEligible()
  → cart.ApplyCoupon(code)
  → recalculate pricing (coupon_code forwarded to pipeline Meta)
  → save cart
  → return cart JSON

DELETE /api/v1/carts/{cartId}/coupon
  → cart.RemoveCoupon()
  → recalculate pricing
  → save cart
  → return cart JSON
```
