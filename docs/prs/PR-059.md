# PR-059 ‚Äî Email Notifications

## Summary

Adds a `Mailer` domain interface, SMTP infrastructure implementation, HTML template registry, and mail configuration ‚Äî zero external dependencies beyond Go stdlib.

## Changes

### Domain (`internal/domain/mail/`)

- **`mail.go`** ‚Äî `Message` struct (`To`, `Subject`, `Body`) and `Mailer` interface with `Send(ctx, msg) error`.
- **`template.go`** ‚Äî `Templates` registry: `Register(name, subject, body)` parses subject + body as `html/template`, `Render(name, to, data)` produces a `Message`. Thread-safe via `sync.RWMutex`.
- **`mail_test.go`** ‚Äî 5 tests: register panics (empty name, bad template), render happy path, unknown template error, message struct fields.

### Infrastructure (`internal/infrastructure/smtp/`)

- **`mailer.go`** ‚Äî `Mailer` struct implementing `mail.Mailer`. Builds RFC-compliant message with headers (From, To, Subject, MIME-Version, Content-Type), optional `PlainAuth` when credentials provided, sends via `net/smtp.SendMail`.
- **`mailer_test.go`** ‚Äî 1 integration test with in-process SMTP stub server: verifies headers (From, To, Subject) and HTML body delivery. Compile-time interface check `var _ mail.Mailer = (*Mailer)(nil)`.

### Config (`internal/platform/config/`)

- **`config.go`** ‚Äî Added `MailConfig` + `SMTPConfig` structs, `Mail` field on `Config`, defaults (`smtp` driver, `localhost:587`, `noreply@localhost`), env overlay (`SHOPANDA_MAIL_SMTP_*`), flatten entries.

### Diagrams

- **`c4-code.md`** ‚Äî Added `Mailer` interface, `Message` struct, `Templates` class, `SMTPMailer` adapter class with `implements` relationship.
- **`c4-component.md`** ‚Äî Added `SMTPMailer` component in Infrastructure boundary.

## Design

```go
// Template registration (at startup)
tmpls := mail.NewTemplates()
tmpls.Register("order.confirmed",
    "Order {{.OrderID}} confirmed",
    "<h1>Thank you, {{.Name}}!</h1><p>Order {{.OrderID}} is confirmed.</p>")

// Render + send (in event handler or job worker)
msg, _ := tmpls.Render("order.confirmed", customer.Email, data)
_ = mailer.Send(ctx, msg)
```

Templates are rendered before enqueue. The `Mailer` does not execute business logic ‚Äî it only delivers pre-built messages. PR-060 will wire `order.paid` events to actually send confirmation emails via the job queue.
