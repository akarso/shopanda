# PR-091 — CMS Pages

## Phase
Phase 23 — URL Routing & CMS

## Scope
Page domain, storage, GET /pages/{slug}, admin CRUD, URL rewrite integration.

## Files

### Domain
- `internal/domain/cms/page.go` — `Page` entity with private fields, `NewPage` constructor (validates id, slug, title), `NewPageFromDB` for reconstruction, getters, setters (`SetSlug`, `SetTitle`, `SetContent`, `SetActive`)
- `internal/domain/cms/repository.go` — `PageRepository` interface: `FindByID`, `FindBySlug`, `List`, `Create`, `Update`, `Delete`
- `internal/domain/cms/events.go` — `EventPageCreated`, `EventPageUpdated` constants and `PageCreatedData`, `PageUpdatedData` payload structs

### Migration
- `migrations/027_create_pages.sql` — `pages` table with UUID PK, unique slug, title, content (default ''), is_active (default true), timestamps; indexes on slug and is_active

### Infrastructure
- `internal/infrastructure/postgres/page_repo.go` — `PageRepo` implementing `cms.PageRepository`; hydration helper, `pq.Error` 23505 handling for slug/id conflicts, `FindBySlug` filters `is_active = true`, `List` orders by `created_at DESC`

### HTTP Handlers
- `internal/interfaces/http/page.go` — `PageHandler` with `Get()` for `GET /api/v1/pages/{slug}` (public, returns id/slug/title/content)
- `internal/interfaces/http/page_admin.go` — `PageAdminHandler` with `List`, `Create`, `Update`, `Delete`; publishes page events on create/update

### Admin
- `internal/application/admin/page_schema.go` — `RegisterPageSchemas` registers form (title, slug, content, is_active) and grid (id, title, slug, is_active, created_at, updated_at)

### Wiring
- `cmd/api/main.go` — `pageRepo`, `pageHandler`, `pageAdmin` instantiation; schema registration; public route `GET /api/v1/pages/{slug}`; admin routes (GET/POST/PUT/DELETE `/api/v1/admin/pages`)
- `internal/application/rewrite/subscriber.go` — Added `HandlePageCreated`, `HandlePageUpdated` and registered them in `Register()`

### Tests
- `internal/domain/cms/page_test.go` — 10 tests: constructor valid/empty-id/empty-slug/empty-title, SetSlug valid/empty, SetTitle valid/empty, SetContent, SetActive
- `internal/interfaces/http/page_test.go` — 12 tests: public Get OK/not-found/repo-error; admin List OK; Create OK/missing-title/missing-slug; Update OK/not-found; Delete OK/repo-error
- `internal/application/rewrite/subscriber_test.go` — Added 2 tests: HandlePageCreated, HandlePageUpdated; updated TestRegister to assert 6 handlers

## Design Decisions
- `FindBySlug` filters by `is_active = true` so inactive pages are hidden from the public endpoint
- Admin endpoints expose all fields including `is_active` and timestamps
- Page events (`cms.page.created`, `cms.page.updated`) are published on admin create/update, triggering URL rewrite registration via the existing subscriber
- Content is stored as plain text (HTML sanitization deferred to a future PR as an explicit concern)
- `Update` uses pointer fields (`*string`, `*bool`) for partial updates — only provided fields are applied
