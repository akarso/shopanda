# PR-099 — Price History (EU Omnibus Price Indication)

## Summary

Records price changes over time and displays the lowest price in the last 30
days on the PDP, satisfying EU Omnibus directive requirements for price
indication on discounted products.

## Scope

## Price indication block

When the current price is higher than the lowest recorded price in the last 30
days, the PDP response includes:

```json
{
  "type": "price_indication",
  "data": {
    "current_price": "39.99",
    "lowest_30d_price": "29.99",
    "currency": "EUR",
    "recorded_at": "2026-04-03T12:00:00Z"
  }
}
```

The block is omitted when no history exists, or when the lowest historical price
is equal to or higher than the current price.

## Design decisions

* **Optional history in importer** — `PriceHistoryRepository` may be nil, so
  existing non-history workflows (exports, tests) are unaffected.
* **Composition step for display** — follows the established PDP pipeline
  pattern; frontends read the `price_indication` block to render the Omnibus
  badge.
* **Snapshot per upsert** — each price change creates a new row. No
  deduplication — multiple snapshots at the same amount are valid since they
  establish the time window.
* **Store-scoped** — snapshots include `store_id` to match the store-scoped
  pricing model from PR-095.
* **Descending index on `recorded_at`** — optimises the "lowest in last 30
  days" query which scans a bounded time window.
