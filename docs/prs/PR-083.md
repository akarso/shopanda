# PR-083 — Clean Infrastructure Ports

## Summary

Removes `WithTx(*sql.Tx)` from domain repository interfaces, adds nil-guard
and `error` return to all `New*Repo` constructors, and updates every caller.
The domain layer no longer imports `database/sql`.

## Motivation

`WithTx(*sql.Tx)` on domain interfaces violates hexagonal architecture by
coupling the domain to a Postgres-specific concept. Moving it to the
infrastructure layer keeps domain ports clean while retaining full
transactional capability for callers that need it.

Adding nil-guard + `error` return to constructors prevents nil-pointer panics
at runtime and makes wiring failures explicit at startup.

## Design Decisions

### WithTx stays on infra, returns domain interface

`WithTx` methods remain on concrete infrastructure types (e.g.
`*postgres.ProductRepo`) and return the domain interface type. This allows the
application layer to define narrow local interfaces:

```go
type txProductRepo interface {
    catalog.ProductRepository
    WithTx(tx *sql.Tx) catalog.ProductRepository
}
```

Callers use type assertions with a graceful fallback to the base repo when
the concrete type doesn't support transactions.

### NewConfigRepo unchanged

`NewConfigRepo` accepts a `configDB` interface (not `*sql.DB`) because it's
intentionally used with both `*sql.DB` and `*sql.Tx`. Adding a nil-guard
here would require a different pattern.
