# PR-062 ‚Äî Media: Upload Endpoint

## Summary

Adds `POST /api/v1/admin/media/upload` endpoint for uploading media files. Introduces the `AssetRepository` domain port, PostgreSQL implementation, media application service with upload flow, and HTTP handler with multipart form parsing.

## Changes

### Domain (`internal/domain/media/`)

- **`repository.go`** ‚Äî `AssetRepository` interface: `Save(ctx, *Asset)`, `FindByID(ctx, id)`.

### Infrastructure (`internal/infrastructure/postgres/`)

- **`asset_repo.go`** ‚Äî PostgreSQL implementation of `AssetRepository`. Insert with JSON meta, scan with meta unmarshal.

### Migration (`migrations/`)

- **`018_create_assets.sql`** ‚Äî `assets` table (id, path, filename, mime_type, size, meta JSONB, created_at) with index on created_at.

### Application (`internal/application/media/`)

- **`service.go`** ‚Äî `Service` with `Upload` method. Flow: validate MIME type ‚Üí generate UUID path ‚Üí save to storage ‚Üí create asset ‚Üí persist ‚Üí publish `asset.uploaded` event ‚Üí return asset + URL. Cleans up stored file on persist failure. Allowed types: image/jpeg, image/png, image/gif, image/webp.
- **`service_test.go`** ‚Äî 4 tests: happy path, unsupported MIME type, storage failure, persist failure (verifies file cleanup).

### Interfaces (`internal/interfaces/http/`)

- **`media.go`** ‚Äî `MediaHandler` with `Upload()` handler. Parses multipart form file, enforces 10MB limit via `MaxBytesReader`, sanitizes filename via `filepath.Base`, delegates to service.
- **`media_test.go`** ‚Äî 2 tests: successful upload with explicit MIME header, missing file returns 422.

### Wiring (`cmd/api/main.go`)

- Creates `AssetRepo`, `MediaService`, `MediaHandler`.
- Registers `POST /api/v1/admin/media/upload` behind `RequireRole(admin)`.
- Removes `_ = mediaStorage` placeholder from PR-061.

### Diagrams

- **`c4-component.md`** ‚Äî Added `MediaHandler` (interfaces), `MediaService` (application), and relationships.
- **`c4-code.md`** ‚Äî Added `AssetRepository` interface and `PostgresAssetRepo` adapter.

## API

```text
POST /api/v1/admin/media/upload
Authorization: Bearer <admin-token>
Content-Type: multipart/form-data

Form field: file (required)

Response 201:
{
  "data": {
    "id": "uuid",
    "path": "uploads/<uuid>/<filename>",
    "filename": "photo.jpg",
    "mime_type": "image/jpeg",
    "size": 102400,
    "url": "/media/uploads/<uuid>/photo.jpg",
    "created_at": "2026-04-08T12:00:00Z"
  },
  "error": null
}
```

## Security

- Admin-only (RequireRole)
- 10MB upload limit (MaxBytesReader)
- Filename sanitized via filepath.Base
- MIME type validated against allow list
- Storage-level directory traversal prevention (from PR-061)
