# PR-018: Pricing Domain Models

## Summary

Introduces the core pricing domain types: `PricingContext`, `PricingItem`, `Adjustment`, and the `PricingStep` interface.

## Changes

### New Files

- `internal/domain/pricing/adjustment.go` — `AdjustmentType` enum, `Adjustment` struct, `NewAdjustment` constructor
- `internal/domain/pricing/context.go` — `PricingContext` and `PricingItem` structs with constructors
- `internal/domain/pricing/step.go` — `PricingStep` interface
- `internal/domain/pricing/adjustment_test.go` — 4 tests
- `internal/domain/pricing/context_test.go` — 6 tests

## Design

### PricingContext

Central object passed through the pricing pipeline. Holds currency, line items, totals (subtotal, discounts, tax, fees, grand total), top-level adjustments, and arbitrary metadata. All monetary fields use `shared.Money` and are initialised to zero for the given currency.

### PricingItem

A single line item with variant ID, quantity, unit price, and computed total. Supports item-level adjustments for per-line discounts/taxes.

### Adjustment

Represents any price modification (discount, tax, fee). Typed via `AdjustmentType` enum with validation. Carries an amount, a code (e.g. `VAT`, `PROMO10`), and an `Included` flag for tax-inclusive pricing.

### PricingStep

Interface with `Name() string` and `Apply(*PricingContext) error`. Steps execute sequentially; an error halts the pipeline. No pipeline executor yet — that is PR-020.

## Tests

10 unit tests covering:
- Adjustment creation (valid, invalid type, empty code)
- AdjustmentType validation
- PricingContext creation (valid currency, invalid currency)
- PricingItem creation (valid, empty variant ID, zero quantity, negative quantity)
- PricingItem total computation (unit price × quantity)

## Milestone

Phase 3 — Pricing. Domain models only; storage (PR-019) and pipeline executor (PR-020) follow.
