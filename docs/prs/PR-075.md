# PR-075 — Attribute & Group Import/Export

## Summary

Adds `AttributeImporter` and `AttributeExporter` for CSV-based attribute and
attribute-group definitions management, plus `import:attributes` and
`export:attributes` CLI commands. Persists definitions to the config store
(PostgreSQL `config` table, JSONB values) under keys `catalog.attributes` and
`catalog.attribute_groups`.

## Changes

### New: `internal/application/importer/attributes.go`

- `AttributeImporter` struct using `config.Repository`
- `Import(ctx, r io.Reader) (*AttributeResult, error)` — parses CSV with
  required columns `code`, `label`, `type`; optional columns `required`,
  `options`, `group`, `group_label`
- Validates attribute type via `AttributeType.IsValid()` (text/number/boolean/select)
- Parses `required` field (true/false/1/yes), `options` as comma-separated values
- Collects group memberships: `group` + `group_label` columns
- Validates groups reference only known attributes via temporary `AttributeRegistry`
- Persists sorted attribute and group slices to config store
- `AttributeResult` struct: `Attributes`, `Groups`, `Skipped`, `Errors []string`
- Validates record length before accessing required column indexes
- Uses `FieldsPerRecord = -1` to allow rows with fewer fields than header
- On duplicate code, removes the code from previous group memberships before
  re-assigning

### New: `internal/application/exporter/attributes.go`

- `AttributeExporter` struct using `config.Repository`
- `Export(ctx, w io.Writer) (*AttributeResult, error)` — reads attribute and
  group definitions from config store, writes CSV with columns `code`, `label`,
  `type`, `required`, `options`, `group`, `group_label`
- Builds reverse mapping: attribute code → group references
- Multi-group attributes produce one row per group membership
- Ungrouped attributes produce one row with empty group fields
- Output sorted by attribute code for deterministic results
- Applies `sanitizeCSVCell()` to code, label, options, group, group_label fields
- `decodeAttributes()` / `decodeGroups()` helpers: marshal→unmarshal round-trip
  to decode `interface{}` values from JSONB config store

## CSV Format

Single file with 7 columns:

```csv
code,label,type,required,options,group,group_label
color,Color,select,true,"red,blue,green",apparel,Apparel
weight,Weight,number,false,,physical,Physical
featured,Featured,boolean,false,,
```

## Design Decisions

- **Config store over new migration**: Attributes are already stored in-memory
  (`AttributeRegistry`). Using `config.Repository` (existing JSONB key-value
  store) avoids a new migration while providing persistence. Config keys:
  `catalog.attributes` and `catalog.attribute_groups`.
- **Single CSV file**: Both attributes and group membership in one file (via
  `group`/`group_label` columns) keeps the workflow simple for users.
- **No round-trip via AttributeRegistry for export**: Exporter reads directly
  from config store, not from the in-memory registry, to ensure exported data
  matches persisted state.
