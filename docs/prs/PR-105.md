# PR-105 — Rate limiting

## Summary

Adds per-IP token-bucket rate limiting as HTTP middleware.
Supports a configurable default limit for all routes and optional
per-route overrides for sensitive prefixes (e.g. `/api/v1/auth`).
Returns 429 Too Many Requests with a `Retry-After` header when the
limit is exceeded. Disabled by default; opt-in via config or env vars.

## Changes

### `internal/platform/ratelimit/limiter.go` (new)
- `Limiter` struct — per-key token-bucket with `sync.Mutex`.
- `NewLimiter(rate, burst)` — constructor, starts background eviction goroutine.
- `Allow(key) bool` — refills tokens based on elapsed time, returns false when exhausted.
- `evictLoop()` — removes stale buckets every 5 minutes.

### `internal/interfaces/http/ratelimit_middleware.go` (new)
- `RateLimitMiddleware(cfg)` — returns no-op when disabled.
- Creates a default limiter and optional per-route limiters (prefix match).
- `clientIP(r)` — extracts client IP from `X-Forwarded-For`, `X-Real-Ip`, or `RemoteAddr`.
- `writeRateLimited(w)` — sets `Retry-After: 1` header, writes 429 JSON error.

### `internal/platform/apperror/apperror.go`
- Added `CodeRateLimited Code = "rate_limited"`.
- Added `RateLimited(message) *Error` constructor.

### `internal/interfaces/http/response.go`
- `StatusFromCode` maps `CodeRateLimited` → `http.StatusTooManyRequests` (429).

### `internal/platform/config/config.go`
- Added `RateLimitConfig` struct (`Enabled`, `Default`, `PerRoute`).
- Added `RateLimitRule` (`Rate`, `Burst`) and `RouteRateLimitRule` (`PathPrefix`, `Rate`, `Burst`).
- `applyEnv()` reads `SHOPANDA_RATE_LIMIT_ENABLED`, `SHOPANDA_RATE_LIMIT_DEFAULT_RATE`, `SHOPANDA_RATE_LIMIT_DEFAULT_BURST`.
- `flatten()` includes `rate_limit.enabled`, `rate_limit.default.rate`, `rate_limit.default.burst`.

### `cmd/api/main.go`
- Added `shophttp.RateLimitMiddleware(cfg.RateLimit)` after `RequestIDMiddleware` and before `LoggingMiddleware`.

## Configuration

```yaml
rate_limit:
  enabled: true
  default:
    rate: 50      # tokens per second
    burst: 100    # max bucket size
  per_route:
    - path_prefix: "/api/v1/auth"
      rate: 5
      burst: 10
```

Environment variables (override YAML):

```shell
SHOPANDA_RATE_LIMIT_ENABLED=true
SHOPANDA_RATE_LIMIT_DEFAULT_RATE=50
SHOPANDA_RATE_LIMIT_DEFAULT_BURST=100
```

## Middleware ordering

```text
Recovery → RequestID → RateLimit → Logging → Auth → Store → Language → CacheControl
```

Rate limiting runs after RequestID (so 429 responses include a request ID for debugging)
and before Auth/Store/Language (to reject excess traffic before expensive operations).

## 429 response format

```json
{
  "data": null,
  "error": {
    "code": "rate_limited",
    "message": "rate limit exceeded"
  }
}
```

Headers: `Retry-After: 1`
