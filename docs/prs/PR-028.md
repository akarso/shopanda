# PR-028 — Cart HTTP Handlers

## Summary

Adds HTTP handlers for all cart operations (create, get, add item, update
item, remove item) and wires them into the API server behind `RequireAuth`
middleware.

## Files Changed

| File | Change |
|------|--------|
| `internal/interfaces/http/cart.go` | New — `CartHandler` with five handler methods |
| `internal/interfaces/http/cart_test.go` | New — 15 handler tests |
| `cmd/api/main.go` | Wired cart repos, pricing pipeline, service, handler, and routes |

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/carts` | Create a cart (currency required) |
| GET | `/api/v1/carts/{cartId}` | Get a cart by ID |
| POST | `/api/v1/carts/{cartId}/items` | Add an item (variant_id + quantity) |
| PUT | `/api/v1/carts/{cartId}/items/{variantId}` | Update item quantity |
| DELETE | `/api/v1/carts/{cartId}/items/{variantId}` | Remove an item |

All routes require authentication (`RequireAuth` middleware).

## Design Decisions

- **Handler struct pattern** — `CartHandler` holds a `*cartApp.Service`; each
  method returns `http.HandlerFunc`, consistent with existing product handlers.
- **Response envelope** — Uses shared `JSON()`/`JSONError()` helpers for
  `{"data": …, "error": …}` envelope format.
- **`toCartResponse`** — Converts domain `cart.Cart` to a flat response DTO
  with `line_total` computed per item (unit_price × quantity).
- **Auth via context** — Handlers extract customer ID from
  `auth.IdentityFrom(ctx)` for cart ownership.
- **Route registration** — `router.Handle()` wraps each handler in
  `RequireAuth()` middleware, consistent with the existing protected route
  pattern.

## Tests

15 tests covering:

- Create: valid, invalid currency, malformed body
- Get: found, not found
- Add item: success, cart not found, missing variant ID, invalid quantity
- Update item: success, cart not found, invalid quantity
- Remove item: success, cart not found, item not found
