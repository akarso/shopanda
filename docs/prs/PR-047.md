# PR-047 — Checkout: Payment + Shipping

## Summary

Adds `select_shipping` and `initiate_payment` steps to the checkout workflow. After the order is created, the workflow now calculates a shipping rate, creates a pending shipment, then creates and initiates a payment via the configured provider.

## Changes

### New files

| File | Purpose |
|------|---------|
| `internal/application/checkout/select_shipping_step.go` | Calculates rate, creates pending shipment |
| `internal/application/checkout/initiate_payment_step.go` | Creates payment, initiates via provider |
| `internal/application/checkout/payment_shipping_steps_test.go` | 16 tests for both steps |

### Modified files

| File | Change |

## Design

### Updated workflow pipeline

```text
1. validate_cart
2. recalculate_pricing
3. reserve_inventory
4. create_order
5. select_shipping   ← NEW
6. initiate_payment  ← NEW
```

### select_shipping step

- Injects `shipping.Provider` + `shipping.ShipmentRepository`
- Calls `provider.CalculateRate(ctx, orderID, currency, itemCount)`
- Creates a pending `Shipment` via `shipping.NewShipment()`
- Persists via `repo.Create()`
- Stores `*shipping.Shipment` in meta as `"shipment"`
- Idempotency via `"shipment_selected"` marker

### initiate_payment step

- Injects `payment.Provider` + `payment.PaymentRepository`
- Creates a pending `Payment` via `payment.NewPayment()` with `order.TotalAmount`
- Persists via `repo.Create()`
- Calls `provider.Initiate()` to start payment
- On success: `payment.Complete(providerRef)` → `repo.UpdateStatus()`
- On decline: `payment.Fail()` → `repo.UpdateStatus()` → step returns error
- On system error: payment stays pending, step fails
- Stores `*payment.Payment` in meta as `"payment"`
- Idempotency via `"payment_initiated"` marker

### Wiring

- `manualPayProvider` (`manualpay.NewProvider()`) used for payment
- `flatRateProvider` (already existed) used for shipping
- `shippingRepo` (`postgres.NewShippingRepo(conn)`) added for shipment persistence
