# PR-058 — Scheduler (Cron)

## Summary

Adds an in-process cron scheduler with a domain interface, infrastructure implementation, `app scheduler` CLI subcommand, and zero external dependencies.

## Changes

### Domain (`internal/domain/scheduler/`)

- **`scheduler.go`** — `Scheduler` interface: `Register(name, spec, fn)`, `Start(ctx)`, `Stop()`. Standard 5-field cron spec (minute hour day-of-month month day-of-week).

### Infrastructure (`internal/infrastructure/cron/`)

- **`scheduler.go`** — In-process `Scheduler` implementation. Includes full 5-field cron parser supporting `*`, `*/n`, ranges (`n-m`), lists (`n,m`), and range-with-step (`n-m/s`). Aligns to minute boundaries, ticks every 60s, recovers from panics in task functions. Uses `sync.Once` for idempotent `Stop()`.
- **`cron_test.go`** — 5 cron parser/matching tests: valid specs, invalid specs, wildcard+step matching, range matching, list matching, range-with-step matching.
- **`scheduler_test.go`** — 7 scheduler lifecycle tests: Register panics (empty name, nil fn, bad spec), idempotent Stop, Start/Stop unblocking, context cancellation, tick dispatch, panic recovery.

### CLI (`cmd/api/main.go`)

- Added `scheduler` subcommand case in dispatch switch.
- `runScheduler()`: opens DB, creates `JobQueue` + `cron.Scheduler`, listens for SIGINT/SIGTERM, blocks on `sched.Start(ctx)`.
- Commented placeholder for registering scheduled jobs (e.g. `cache.cleanup`).

### Diagrams

- **`c4-code.md`** — Added `Scheduler` interface and `CronScheduler` adapter class with implements relationship.
- **`c4-component.md`** — Added `CronScheduler` component in Domain boundary, relationship to `PostgresJobQueue`.

## Design

```
scheduler.Register("cache.cleanup", "*/5 * * * *", func() {
    job, _ := jobs.NewJob(id.New(), "cache.cleanup", nil)
    _ = queue.Enqueue(ctx, job)
})
```

The scheduler does NOT execute business logic directly. It enqueues jobs → the Worker processes them.
