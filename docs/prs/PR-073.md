# PR-073 — Stock Import/Export

## Summary

Adds `StockImporter` and `StockExporter` for CSV-based stock quantity
management, plus `import:stock` and `export:stock` CLI commands.

## Changes

### Modified: `internal/domain/inventory/repository.go`

- Added `ListStock(ctx context.Context, offset, limit int) ([]StockEntry, error)`
  to the `StockRepository` interface for paginated stock enumeration

### Modified: `internal/infrastructure/postgres/stock_repo.go`

- Added `ListStock` implementation: `SELECT variant_id, quantity, updated_at
  FROM stock ORDER BY variant_id LIMIT $1 OFFSET $2`
- Validates offset >= 0 and limit > 0; caps limit at 100

### New: `internal/application/importer/stock.go`

- `StockImporter` struct using `VariantRepository` and `StockRepository`
- `Import(ctx, r io.Reader) (*StockResult, error)` — parses CSV with required
  columns `sku` and `quantity`, resolves SKU to variant via `FindBySKU`,
  validates quantity (non-negative integer), calls `SetStock`
- `StockResult` struct: `Updated`, `Skipped`, `Errors []string`
- Line-level error reporting with line numbers

### New: `internal/application/exporter/stock.go`

- `StockExporter` struct using `StockRepository` and `VariantRepository`
- `Export(ctx, w io.Writer) (*StockResult, error)` — paginates through
  `ListStock`, resolves each entry's variant via `FindByID` for SKU,
  skips orphan entries, writes CSV with `sku,quantity` columns
- `StockResult` struct: `Entries` count

### New: `internal/application/importer/stock_test.go`

9 tests:
- `TestStockImport_Basic` — happy path, 2 SKUs imported
- `TestStockImport_MissingColumns` — rejects CSV without required columns
- `TestStockImport_UnknownSKU` — unknown SKU skipped with error message
- `TestStockImport_InvalidQuantity` — non-integer quantity produces error
- `TestStockImport_NegativeQuantity` — negative quantity produces error
- `TestStockImport_EmptySKU` — blank SKU produces error
- `TestStockImport_SetStockError` — repository error propagated to result
- `TestStockImport_ZeroQuantity` — zero quantity accepted
- `TestStockImport_MixedValidAndInvalid` — mixed rows correctly counted

### New: `internal/application/exporter/stock_test.go`

5 tests:
- `TestStockExport_Basic` — 2 entries exported with correct CSV structure
- `TestStockExport_EmptyStock` — header-only output
- `TestStockExport_OrphanStockSkipped` — orphan stock entry skipped
- `TestStockExport_ListStockError` — repository error propagated
- `TestStockExport_Pagination` — 150 entries across multiple pages

### Modified: `cmd/api/main.go`

- Added `import:stock` CLI subcommand with `runImportStock` function
  - Usage: `app import:stock <file.csv>`
- Added `export:stock` CLI subcommand with `runExportStock` function
  - Usage: `app export:stock <file.csv>`

### Modified: `docs/diagrams/c4-component.md`

- Added `StockImporter` and `StockExporter` components in Application layer
- Added relationships to PostgreSQL Repositories

## Architecture

```text
CLI (main.go)
  → StockImporter (application layer)
    → VariantRepository.FindBySKU (SKU resolution)
    → StockRepository.SetStock (persistence)

CLI (main.go)
  → StockExporter (application layer)
    → StockRepository.ListStock (paginated)
    → VariantRepository.FindByID (SKU lookup)
    → CSV writer (io.Writer)
```

## CSV Formats

### Import: `import:stock`

Columns: `sku`, `quantity` (both required)

```csv
sku,quantity
SKU-001,50
SKU-002,100
```

### Export: `export:stock`

Columns: `sku`, `quantity`

One row per stock entry with a resolvable variant.
