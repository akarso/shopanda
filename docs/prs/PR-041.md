# PR-041 — Entity Admin Endpoints

## Summary

Adds admin CRUD endpoints for products (list) and orders (list + view), completing the minimal admin surface behind the `RequireRole(admin)` guard introduced in PR-040.

## Changes

### Domain — Order Repository Interface
- **`internal/domain/order/repository.go`** — Added `List(ctx, offset, limit) ([]Order, error)` for paginated all-orders retrieval.

### Infrastructure — Order Postgres Repository
- **`internal/infrastructure/postgres/order_repo.go`** — Implemented `List` with `LIMIT`/`OFFSET`, batch item loading, and cap at 100 per page.

### Interfaces — Product Admin Handler
- **`internal/interfaces/http/product_admin.go`** — Added `List()` handler (`GET /api/v1/admin/products`). Simple paginated product list without composition pipeline overhead.

### Interfaces — Order Admin Handler (new)
- **`internal/interfaces/http/order_admin.go`** — `OrderAdminHandler` with:
  - `List()` — paginated list of all orders across all customers.
  - `Get()` — view any order by ID (no ownership check).
  - Reuses `toOrderResponse()` and `orderResponse` from `order.go`.

### Wiring
- **`cmd/api/main.go`** — Three new admin routes:
  - `GET /api/v1/admin/products` → `productAdmin.List()`
  - `GET /api/v1/admin/orders` → `orderAdmin.List()`
  - `GET /api/v1/admin/orders/{orderId}` → `orderAdmin.Get()`

### Tests — 12 new tests
- **`internal/interfaces/http/product_admin_test.go`** — 3 tests:
  - `List_OK`, `List_Empty`, `List_Pagination`
- **`internal/interfaces/http/order_admin_test.go`** — 9 tests:
  - `List_OK`, `List_Empty`, `List_AllCustomers`, `List_CustomerForbidden`, `List_GuestUnauthorized`
  - `Get_OK`, `Get_AnyCustomerOrder`, `Get_NotFound`, `Get_CustomerForbidden`

### Test mock updates
- `stubOrderRepo` (order_test.go), `stubCheckoutOrderRepo` (checkout_test.go), `mockOrderRepo` (order_inventory_steps_test.go) — all updated to implement new `List` method.
- `mockAdminProductRepo` — added `listFn` callback, `newAdminRouter` now registers `GET` route.

## Routes Added

| Method | Path | Auth | Handler |
|--------|------|------|---------|
| GET | `/api/v1/admin/products` | admin | `productAdmin.List()` |
| GET | `/api/v1/admin/orders` | admin | `orderAdmin.List()` |
| GET | `/api/v1/admin/orders/{orderId}` | admin | `orderAdmin.Get()` |

## Key Decisions

- **No composition pipeline for admin product list** — Admin sees raw product data; composition pipeline is for storefront enrichment.
- **No customer ownership check on admin order endpoints** — Admins can view any order.
- **Reuses existing `orderResponse`/`toOrderResponse`** from the customer-facing order handler to avoid response type duplication.
