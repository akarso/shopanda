# PR-061 ‚Äî Media: Storage Interface

## Summary

Introduces the media storage abstraction: `Asset` entity, `Storage` interface, and a local filesystem implementation. Adds config support for media storage and wires the storage backend in `main.go`.

## Changes

### Domain (`internal/domain/media/`)

- **`asset.go`** ‚Äî `Asset` struct (ID, Path, Filename, MimeType, Size, Meta, CreatedAt). `NewAsset()` constructor with validation for all required fields.
- **`storage.go`** ‚Äî `Storage` interface: `Name()`, `Save(path, io.Reader)`, `Delete(path)`, `URL(path)`.
- **`events.go`** ‚Äî Event constants `asset.uploaded`, `asset.deleted` and `AssetEventData` struct.
- **`asset_test.go`** ‚Äî 2 tests: happy path + 6 validation subtests (empty id/path/filename/mime, zero/negative size).

### Infrastructure (`internal/infrastructure/localfs/`)

- **`storage.go`** ‚Äî `LocalStorage` implements `media.Storage` using local filesystem. `Save` creates intermediate directories, `Delete` is idempotent. Both operations include directory traversal prevention (resolved path must stay within basePath).
- **`storage_test.go`** ‚Äî 6 tests: Name, Save+URL, Delete, Delete non-existent, Save traversal attack, Delete traversal attack.

### Config (`internal/platform/config/config.go`)

- Added `MediaConfig` (Storage driver string, Local sub-config) and `LocalStorageConfig` (BasePath, BaseURL).
- Defaults: `storage=local`, `base_path=./public/media`, `base_url=/media`.
- Env overlays: `SHOPANDA_MEDIA_STORAGE`, `SHOPANDA_MEDIA_LOCAL_BASEPATH`, `SHOPANDA_MEDIA_LOCAL_BASEURL`.
- Flatten: `media.storage`, `media.local.base_path`, `media.local.base_url`.

### Wiring (`cmd/api/main.go`)

- Initializes `media.Storage` based on `cfg.Media.Storage` (currently "local" only).
- Placeholder `_ = mediaStorage` ‚Äî wired to HTTP handler in PR-062.

### Diagrams

- **`c4-component.md`** ‚Äî Added `LocalStorage` component in Infrastructure boundary.
- **`c4-code.md`** ‚Äî Added `Asset` entity, `Storage` interface, `LocalStorage` adapter with `Storage <|.. LocalStorage` relationship.

## Design

```
Path is storage-relative (never a full URL).
URL is always generated via storage.URL(path).
Local filesystem is the core implementation.
S3/CDN backends will be provided via plugins.
Directory traversal is prevented by validating resolved paths stay within basePath.
```
