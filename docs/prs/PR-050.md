## Design

### Category entity

```text
Category {
    ID        string
    ParentID  *string          — nil for root categories
    Name      string
    Slug      string           — unique
    Position  int              — sibling ordering
    Meta      map[string]any   — SEO metadata
    CreatedAt time.Time
    UpdatedAt time.Time
}
```

- Adjacency list tree via nullable `ParentID`
- `NewCategory(id, name, slug)` validates required fields, sets timestamps
- `NewCategoryFromDB(...)` hydrates from storage without validation
- `IsRoot()` returns true when `ParentID == nil`

### Repository interface

- `FindByID(ctx, id)` — single lookup, `(nil, nil)` when not found
- `FindBySlug(ctx, slug)` — slug lookup, `(nil, nil)` when not found
- `FindByParentID(ctx, *parentID)` — children ordered by position then name; nil for roots
- `Create(ctx, *Category)` — insert with unique slug constraint
- `Update(ctx, *Category)` — update with not-found and slug conflict handling

### Migration (014)

- `categories` table: TEXT id, nullable parent_id self-referencing FK, JSONB meta, unique slug
- `product_categories` join table: composite PK `(product_id, category_id)`, cascading deletes
- Indexes on `parent_id`, `slug`, and `category_id` (join table)

### PostgreSQL implementation

- `hydrateCategory` accepts a scan function (works with both `QueryRow` and `Rows`)
- `sql.NullString` for nullable `parent_id`
- JSONB marshaling for `Meta`
- `pq.Error` code `23505` mapped to `apperror.Conflict`
- `RowsAffected() == 0` mapped to `apperror.NotFound`
