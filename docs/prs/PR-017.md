# PR-017: Basic Product Import (CSV)

## Summary

Adds a CSV-based product import capability with a CLI command `import:products`.

## Changes

### New Files

- `internal/application/importer/products.go` — `ProductImporter` with `Import(ctx, reader)` method
- `internal/application/importer/products_test.go` — 8 unit tests

### Modified Files

- `cmd/api/main.go` — Added `import:products` CLI subcommand, `runImportProducts` function

## Design

### Pipeline: Parse → Validate → Write

1. **Parse** — All CSV rows are read and grouped by `slug` into `parsedRow` structs
2. **Validate** — Required fields checked, column presence verified, duplicates detected
3. **Write** — New products created atomically (product + all variants in a single transaction)

### CSV Rules

- Columns: `name`, `slug`, `sku` (required), `description`, `variant_name` (optional)
- Column order is flexible (detected from header row)
- Rows with the same `slug` are grouped into a single product with multiple variants
- Existing products (matched by slug) are reused — their variants are appended directly
- Row-level errors are collected in `Result.Errors` without aborting the import
- Skipped rows (empty required fields) are counted separately

### Transaction Support

- New products are created atomically: product + all its variants within a single DB transaction
- If any variant fails, the entire product group is rolled back (no orphaned products)
- `TxStarter` interface decouples transaction creation from `*sql.DB`
- When `TxStarter` is nil (e.g. in tests), writes proceed without transactions

### Type-Safe Transactions

- `catalog.Tx` interface (`Commit() error`, `Rollback() error`) replaces `interface{}` in `WithTx`
- Both `ProductRepository` and `VariantRepository` accept `catalog.Tx` in their `WithTx` methods
- Postgres implementations type-assert to `*sql.Tx` with explicit panic on invalid type

## CLI Usage

```
app import:products path/to/products.csv
```
