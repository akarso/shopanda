# PR-104 — Webhook signature verification

## Summary

Adds per-provider HMAC-SHA256 signature verification for payment webhooks.
The webhook handler now buffers the raw request body, verifies the
`X-Webhook-Signature` header against a per-provider secret, and rejects
unsigned or invalid requests with 401. Providers without a configured
secret skip verification (backwards-compatible for `manual` provider).

## Changes

### `internal/domain/payment/webhook.go` (new)
- `WebhookVerifier` interface (port) with `Verify(provider, signature, body) error`.
- `ErrSignatureMissing` and `ErrSignatureInvalid` sentinel errors.

### `internal/infrastructure/webhook/hmac.go` (new)
- `HMACVerifier` — adapter implementing `WebhookVerifier` using HMAC-SHA256.
- Per-provider secrets via constructor `NewHMACVerifier(secrets)`.
- Providers without a configured secret pass through (no enforcement).

### `internal/interfaces/http/payment_webhook.go`
- `NewPaymentWebhookHandler` now requires a `payment.WebhookVerifier`.
- `Handle()` reads raw body via `io.ReadAll(io.LimitReader(...))` (1 MB cap),
  verifies `X-Webhook-Signature` header before JSON decode.
- Returns 401 for missing or invalid signatures.

### `internal/platform/config/config.go`
- Added `WebhooksConfig` struct with `Secrets map[string]string`.
- Added `Secret(provider)` accessor method.
- `applyEnv()` reads `SHOPANDA_WEBHOOKS_SECRET_<PROVIDER>` env vars.
- `flatten()` includes `webhooks.secrets.<provider>` keys.

### `cmd/api/main.go`
- Creates `webhook.NewHMACVerifier(cfg.Webhooks.Secrets)`.
- Passes verifier to `NewPaymentWebhookHandler`.

## New tests

### `internal/infrastructure/webhook/hmac_test.go`
- `TestHMACVerifier_ValidSignature` — correct MAC passes.
- `TestHMACVerifier_InvalidSignature` — wrong MAC rejected.
- `TestHMACVerifier_MissingSignature` — empty sig with configured secret → error.
- `TestHMACVerifier_NoSecretForProvider` — unconfigured provider skips check.
- `TestHMACVerifier_NilSecretsMap` — nil map is safe.
- `TestHMACVerifier_TamperedBody` — body tampered after signing → rejected.

### `internal/interfaces/http/payment_webhook_test.go`

- `TestPaymentWebhook_MissingSignature` — 401 when secret configured but no header.
- `TestPaymentWebhook_InvalidSignature` — 401 when signature doesn't match.
- `TestPaymentWebhook_NoSecretConfigured_SkipsVerification` — existing behavior preserved.

### `internal/platform/config/config_test.go`

- `TestWebhooksConfig_EnvOverridesYAML` — env takes precedence.

## Configuration

```yaml
webhooks:
  secrets:
    stripe: "whsec_..."
    paypal: "pp_secret_..."
```

Environment variables (override YAML):
```
SHOPANDA_WEBHOOKS_SECRET_STRIPE=whsec_...
SHOPANDA_WEBHOOKS_SECRET_PAYPAL=pp_secret_...
```

## Route table

| Method | Path | Auth | Signature |
|--------|------|------|-----------|
| POST | `/api/v1/payments/webhook/{provider}` | Public | X-Webhook-Signature (HMAC-SHA256, per-provider) |

## Architecture

```
domain/payment/webhook.go          → WebhookVerifier port
infrastructure/webhook/hmac.go     → HMACVerifier adapter
interfaces/http/payment_webhook.go → raw-body buffering + verification
platform/config/config.go          → per-provider secret config
```

Dependency direction: `interfaces → domain ← infrastructure`.
