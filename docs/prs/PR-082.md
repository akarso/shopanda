# PR-082 — Tax Pipeline Step

## Summary

Adds a `TaxStep` to the pricing pipeline that calculates per-item tax in both
**exclusive** (tax added on top) and **inclusive** (tax extracted from price)
modes. The step is wired between `BasePriceStep` and `FinalizeStep`.

## Design

### Pipeline placement

```text
BasePriceStep → TaxStep → [plugin steps] → FinalizeStep
```

### Meta keys (on PricingContext)

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `tax_country` | `string` | yes | ISO 3166-1 alpha-2 country code |
| `tax_mode` | `string` | yes | `"exclusive"` or `"inclusive"` |
| `tax_classes` | `map[string]string` | no | Variant ID → tax class override |

### Per-item logic

1. Resolve tax class: `tax_classes[variantID]` → `defaultClass` (constructor arg, wired as `"standard"`)
2. Look up `TaxRate` via `RateRepository.FindByCountryAndClass(country, class)`
3. If rate is nil or zero → skip (zero-rated)
4. Calculate tax amount via `tax.Calculate(item.Total, rate, mode)`
5. Create `Adjustment` of type `AdjustmentTax`, code `"tax.{Country}.{Class}"`

### Exclusive vs inclusive

| Mode | item.Total | Adjustment.Included | FinalizeStep result |
|------|-----------|---------------------|---------------------|
| Exclusive | unchanged | `false` | `GrandTotal = Subtotal + Tax` |
| Inclusive | reduced to net (`Total - tax`) | `true` | `GrandTotal = net + tax = original price` |

No changes to `FinalizeStep` required — the math works naturally because
inclusive mode adjusts `item.Total` to net before finalization.

## Test coverage

- `TestTaxStep_Name` — returns `"tax"`
- `TestTaxStep_Exclusive` — 19% on 10000 → tax 1900, total unchanged
- `TestTaxStep_Inclusive` — 19% on 11900 → tax 1900, total reduced to 10000
- `TestTaxStep_MultipleItems` — two items taxed independently
- `TestTaxStep_ZeroRate` — rate=0 → no adjustment
- `TestTaxStep_NoRate` — no rate found → no adjustment
- `TestTaxStep_PerVariantClassOverride` — standard vs reduced rates via Meta
- `TestTaxStep_MissingCountryMeta` — error on missing key
- `TestTaxStep_MissingModeMeta` — error on missing key
- `TestTaxStep_InvalidMode` — error on bogus mode
- `TestTaxStep_RepoError` — propagates repo error
- `TestTaxStep_FullPipeline_Exclusive` — Tax → Finalize integration
- `TestTaxStep_FullPipeline_Inclusive` — Tax → Finalize integration
- `TestTaxStep_Inclusive_Quantity` — multi-quantity inclusive calculation
