# PR-033: Complete Auth API Surface

## Summary

Delivers the full auth API surface required for a login form — register, login, logout, me, and password reset (request + confirm). Adds token generation–based revocation, `expires_at` in auth responses, event emission for `customer.created` and `password_reset.requested`, and a `ValidatingTokenParser` that checks JWT generation against the customer record.

## API Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/api/v1/auth/register` | public | Create account, return JWT + expires_at |
| POST | `/api/v1/auth/login` | public | Authenticate, return JWT + expires_at |
| POST | `/api/v1/auth/logout` | required | Bump token generation, invalidate all JWTs |
| GET | `/api/v1/auth/me` | required | Return current customer profile |
| POST | `/api/v1/auth/password-reset/request` | public | Generate reset token, emit event |
| POST | `/api/v1/auth/password-reset/confirm` | public | Validate token, set new password |

## Design Decisions

- **Token generation over opaque tokens**: JWT remains stateless for signature verification, but `ValidatingTokenParser` checks the `gen` claim against the customer's `token_generation` on each request. This enables revocation (logout, password reset) without maintaining a session table.
- **Password reset prevents enumeration**: `RequestPasswordReset` always returns 200 regardless of whether the email exists.
- **Reset token security**: Plaintext token delivered via event; only SHA-256 hash stored in DB. Token auto-expires after configurable TTL (default: 1 hour).
- **Event-driven reset delivery**: `password_reset.requested` event carries plaintext token. A dev handler logs the token when `SHOPANDA_DEV_MODE` is set; email delivery is handled by a downstream plugin (not in scope for this PR).

## ⚠️ Breaking Changes

- **JWT `Create` signature changed**: `issuer.Create(subject, role string, gen int64)` now requires a third `gen` parameter. All callers must be updated.
- **`auth.NewService` signature changed**: Now requires `resetTTL time.Duration` as the last parameter.
- **`NewValidatingTokenParser` signature changed**: Now requires `cacheTTL time.Duration` as the third parameter (0 disables caching).
- **`CustomerRepository` interface**: Added `BumpTokenGeneration(ctx context.Context, customerID string) error` method. All implementations must be updated.

## Performance Notes

- **ValidatingTokenParser** caches each customer's `TokenGeneration` in-memory with a configurable TTL (default: 30s in production, 0/disabled in tests). Cache miss triggers a `FindByID` DB lookup. Revocation is enforced within the cache window. For immediate revocation, set `cacheTTL` to 0.
