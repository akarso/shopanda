# PR-014 — Product Write Handlers (Admin)

## Summary

Adds product Create and Update HTTP endpoints under `/api/v1/admin/products`. Separate `ProductAdminHandler` struct keeps write concerns isolated from read handlers (PR-013).

## Files

| File | Purpose |
|------|---------|
| `internal/interfaces/http/product_admin.go` | `ProductAdminHandler` — `Create()` and `Update()` handlers with request DTOs |
| `internal/interfaces/http/product_admin_test.go` | 14 tests covering create/update happy paths, validation, errors |
| `cmd/api/main.go` | Wires `ProductAdminHandler` and registers admin routes |

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/v1/admin/products` | Create a new product (draft status) |
| `PUT` | `/api/v1/admin/products/{id}` | Update an existing product (partial) |

## Design Decisions

- **Separate handler struct**: `ProductAdminHandler` only depends on `ProductRepository` — no composition pipelines needed for writes.
- **Admin route prefix**: `/api/v1/admin/` groups write endpoints. Auth guard will be added in PR-021/040.
- **Pointer fields for partial update**: `updateProductRequest` uses `*string` for optional fields so `null`/absent fields are distinguished from empty strings.
- **Validation at handler level**: Empty name/slug rejected with 422 before hitting the repo. Invalid status rejected with 422.
- **ID generation in handler**: `Create` calls `id.New()` — domain's `NewProduct` receives a pre-generated ID.
- **Duplicate slug**: Repo returns `apperror.Conflict` (from PR-011 SQLSTATE 23505 detection), handler passes it through → 409.

## Request/Response

```json
// POST /api/v1/admin/products
// Request:
{ "name": "Widget", "slug": "widget", "description": "...", "attributes": {"color": "blue"} }
// Response (201):
{ "data": { "product": {...} }, "error": null }

// PUT /api/v1/admin/products/{id}
// Request (all fields optional):
{ "name": "New Name", "slug": "new-slug", "description": "...", "status": "active", "attributes": {...} }
// Response (200):
{ "data": { "product": {...} }, "error": null }
```

## Tests

14 tests:
- Create: OK, with attributes, missing name (422), missing slug (422), invalid body (422), duplicate slug (409)
- Update: OK (name + description + status), not found (404), invalid status (422), empty name (422), empty slug (422), invalid body (422), partial update (only description changed), repo error (500)
