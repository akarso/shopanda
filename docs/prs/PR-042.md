provider interface, domain events — along with the PostgreSQL
implementation and migration.

## Domain Model

### PaymentStatus

`pending` → `completed` (via `Complete(providerRef)`)
`pending` → `failed` (via `Fail()`)
`completed` → `refunded` (via `Refund()`)

Each transition has a guard clause that rejects invalid source states.

### PaymentMethod

Currently: `manual`. Additional methods (e.g. Stripe) will be added as
plugins.

### Payment Entity

- Private `status` field with getter + `SetStatusFromDB` for persistence
  reconstruction (same pattern as `order.Order`).
- `NewPayment(id, orderID, method, amount)` validates all inputs and
  defaults to `StatusPending`.
- `Amount` uses `shared.Money` (int64 smallest-unit + ISO 4217 currency).

## Provider Interface

```go
type Provider interface {
    Method() PaymentMethod
    Initiate(ctx context.Context, p *Payment) (ProviderResult, error)
}
```

Designed for plugin implementations. The `manual` provider will ship as the
built-in default in a follow-up PR.

## Repository Interface

- `FindByID` / `FindByOrderID` — return `(nil, nil)` when not found
- `Create` — insert new payment
- `UpdateStatus` — update status, provider_ref, updated_at

## PostgreSQL Implementation

- Compile-time interface check
- `hydratePayment` scanner abstraction (same pattern as `hydrateOrder`)
- `sql.ErrNoRows` → `(nil, nil)`
- `NullString` handling for optional `provider_ref`
- Validates `PaymentMethod` from DB (defense-in-depth against corrupt values)

## Migration (012)

```sql
CREATE TABLE payments (
    id          TEXT PRIMARY KEY,
    order_id    TEXT NOT NULL REFERENCES orders(id),
    method      TEXT NOT NULL CHECK (method IN ('manual')),
    status      TEXT NOT NULL DEFAULT 'pending'
                    CHECK (status IN ('pending','completed','failed','refunded')),
    amount      BIGINT NOT NULL CHECK (amount > 0),
    currency    TEXT NOT NULL CHECK (length(currency) = 3),
    provider_ref TEXT,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    CONSTRAINT payments_order_id_unique UNIQUE (order_id)
);

CREATE INDEX IF NOT EXISTS payments_status_idx ON payments (status);
```
