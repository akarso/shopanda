# PR-024 — Inventory Reservations

## Summary

Adds the reservation model to the inventory domain: a temporary hold on stock
that is created at checkout, confirmed on successful payment, or released on
failure/expiry.

## Changes

### Domain (`internal/domain/inventory/`)

| File | Description |
|------|-------------|
| `reservation.go` | `Reservation` entity with `ID`, `VariantID`, `Quantity`, `Status`, `ExpiresAt`, `CreatedAt`. `NewReservation` validates all fields. `IsExpired(now)` helper. |
| `reservation_repository.go` | `ReservationRepository` interface — `Reserve`, `Release`, `Confirm`, `FindByID`, `ListActiveByVariantID`. |
| `reservation_test.go` | 9 unit tests covering construction validation, expiry checks, and status enum. |

### Status Lifecycle

```
active → confirmed   (payment succeeded, stock stays decremented)
active → released    (payment failed / timeout, stock restored)
```

### Infrastructure (`internal/infrastructure/postgres/`)

| File | Description |
|------|-------------|
| `reservation_repo.go` | `ReservationRepo` implements `ReservationRepository`. `Reserve` atomically decrements stock + inserts reservation in a transaction. `Release` restores stock. `Confirm` marks confirmed without restoring. |
| `reservation_repo_test.go` | 10 integration tests: reserve, insufficient stock, release, confirm, list active, nil/empty guards. |

### Migration

| File | Description |
|------|-------------|
| `migrations/005_create_reservations.sql` | `reservations` table with FK to `variants`, CHECK constraints on `quantity > 0` and status enum, partial index on `expires_at` for active reservations. |

## Design Decisions

- **Atomic reserve**: `Reserve` uses a DB transaction — decrement stock + insert reservation is all-or-nothing. If stock < requested quantity, the UPDATE matches 0 rows and we return a conflict error.
- **Confirm does not touch stock**: Stock was already decremented at reserve time, so `Confirm` only flips the status.
- **Release restores stock**: `Release` atomically marks the reservation as released and increments the stock quantity.
- **Partial index on `expires_at`**: Only indexes active reservations, enabling efficient expiration sweeps without indexing confirmed/released rows.
