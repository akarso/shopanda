# PR-037: Checkout Steps — Reserve Inventory & Create Order

## Summary

Adds the final two steps to the checkout workflow:

1. **ReserveInventoryStep** — creates time-bounded inventory reservations for each cart item.
2. **CreateOrderStep** — builds and persists an order using the pricing snapshot from the pipeline.

Both steps are idempotent (safe to retry) and follow the same `Step` interface established in PR-035.

## Design Decisions

### ReserveInventoryStep

- Uses `inventory.ReservationRepository.Reserve()` which atomically decrements stock.
- Default TTL of 15 minutes (`ReservationTTL`); reservations stored in `Meta["reservations"]` as `[]string`.
- Idempotency guard: `Meta["reserved"]` (bool, type-asserted).
- On any reservation failure the step returns an error immediately (no partial cleanup — the workflow failure handler is responsible for compensation).

### CreateOrderStep

- Reads the pricing snapshot from `Meta["pricing"]` (`*pricing.PricingContext`).
- Looks up each variant via `catalog.VariantRepository.FindByID` to capture SKU and Name at order time.
- Uses `pi.UnitPrice` from the pricing pipeline (not the cart's unit price) ensuring the order reflects any discounts/adjustments applied during pricing.
- Creates the order via `order.NewOrder()` + `orders.Save()`.
- Sets `cctx.Order` and `Meta["created_order_id"]` (string, type-asserted for idempotency).
