# PR-077 ŌĆö Price Import/Export

## Summary

Adds CSV-based price import and export commands (`import:prices`, `export:prices`). Prices reference variants by SKU and support multiple currencies per variant. Completes the data exchange milestone (products, stock, customers, attributes, categories, prices).

## CSV Format

```csv
sku,currency,amount
TSHIRT-M-RED,EUR,1999
TSHIRT-M-RED,USD,2199
HOODIE-L,EUR,3999
```

- **sku** ŌĆö required, references a variant by SKU
- **currency** ŌĆö required, ISO 4217 3-letter code (case-insensitive, normalized to uppercase)
- **amount** ŌĆö required, positive integer in smallest currency unit (e.g. cents)

## Design Decisions

1. **SKU-based reference** ŌĆö matches the stock importer pattern; more user-friendly than variant UUIDs
2. **Upsert semantics** ŌĆö uses `PriceRepository.Upsert` which does `INSERT ... ON CONFLICT (variant_id, currency) DO UPDATE`; existing prices for the same variant+currency pair are updated
3. **Created/Updated tracking** ŌĆö checks `FindByVariantAndCurrency` before upsert to distinguish creates from updates in the result summary
4. **Currency normalization** ŌĆö input is uppercased before validation via `shared.IsValidCurrency`
5. **Amount validation** ŌĆö must be a positive integer; zero and negative amounts are rejected
6. **Paginated export** ŌĆö follows the stock exporter pattern with `pageSize` batching
7. **Orphan handling** ŌĆö prices for deleted variants are silently skipped during export
8. **Formula injection protection** ŌĆö SKU field sanitized via `sanitizeCSVCell`
