# PR-010 — Product Domain

## Scope

Product entity, repository interface, event definitions.

## What Changed

### New: `internal/domain/catalog/product.go`

- **`Status`** — String type with constants: `draft`, `active`, `archived`. `IsValid()` method.
- **`Product`** — Entity struct: `ID`, `Name`, `Slug`, `Description`, `Status`, `Attributes` (map[string]interface{}), `CreatedAt`, `UpdatedAt`.
- **`NewProduct(id, name, slug)`** — Constructor. Defaults to `StatusDraft`, initialises empty `Attributes` map, sets timestamps to `time.Now().UTC()`.

### New: `internal/domain/catalog/repository.go`

- **`ProductRepository`** — Interface defining persistence operations:
  - `FindByID(ctx, id)` → `(*Product, error)`
  - `FindBySlug(ctx, slug)` → `(*Product, error)`
  - `List(ctx, offset, limit)` → `([]Product, error)`
  - `Create(ctx, *Product)` → `error`
  - `Update(ctx, *Product)` → `error`

### New: `internal/domain/catalog/events.go`

- **Event constants**: `catalog.product.created`, `catalog.product.updated`
- **`ProductCreatedData`** — Payload: `ProductID`, `Name`, `Slug`, `Status`
- **`ProductUpdatedData`** — Payload: `ProductID`, `Name`, `Slug`, `Status`

### New: `internal/domain/catalog/product_test.go`

4 tests covering:
- `NewProduct` construction (fields, defaults, timestamps)
- `Status.IsValid()` for all valid and invalid values
- Default empty description
- Event constant values

## Design Decisions

- **No database dependencies** — pure domain layer per hexagonal architecture.
- **Repository returns `*Product` (nil, nil)** for not-found — lets callers decide how to handle absence without coupling to `apperror`.
- **ID generated externally** — `NewProduct` takes `id` as parameter. Caller uses `id.New()`. Keeps domain free of platform dependencies.
- **Attributes as `map[string]interface{}`** — mirrors JSONB from the schema. Typed attribute model deferred to PR-069.
- **Events carry identifiers, not full objects** — per EVENT_SYSTEM.md payload design principles.
