# PR-222 — Media Library

## Summary

Implemented admin media management for the embedded SPA:

- Media library page (`/admin/media`)
- Asset upload with file picker and drag-and-drop
- Asset grid with preview, copy URL, and delete actions
- Reusable image picker modal for product forms
- Product featured-image selection stored in product attributes
- Backend admin media list and delete endpoints

No build step. Vanilla JS only.

## Scope

### Media Library

- Fetches media assets: `GET /api/v1/admin/media?offset=0&limit=100`
- Renders uploaded assets as thumbnail cards
- Shows filename, file size, preview, copy URL, and delete actions
- Uses thumbnail URLs when available, falls back to the original asset URL

### Upload Flow

- Uploads files with multipart form data: `POST /api/v1/admin/media`
- Existing compatibility route retained: `POST /api/v1/admin/media/upload`
- Supports file input and drag-and-drop
- Displays upload progress during transfer
- Refreshes asset grid after upload completes

### Asset Deletion

- Deletes asset metadata and stored file: `DELETE /api/v1/admin/media/{assetId}`
- Removes the primary asset file from storage
- Attempts thumbnail cleanup before deleting the asset row
- Publishes `asset.deleted` after successful delete

### Product Image Picker

- Product create/edit form now includes a `Featured Image` section
- `Choose From Library` opens a modal asset picker backed by the same media API
- Selected asset stores:
  - `attributes.image_asset_id`
  - `attributes.image_url`
- `Clear` resets the selected product image

## Auth & Permissions

Admin media endpoints require authenticated admin access and appropriate media permissions:

- `GET /api/v1/admin/media` -> media read
- `POST /api/v1/admin/media` -> media write
- `POST /api/v1/admin/media/upload` -> media write
- `DELETE /api/v1/admin/media/{assetId}` -> media write

Common failures:

- `401 Unauthorized` for missing or invalid authentication
- `403 Forbidden` for authenticated users without required permission
- `422 Unprocessable Entity` for invalid upload payloads or missing files

## Notes

- Product image selection is stored in product attributes to avoid changing the core product schema in this PR.
- The product form stores both asset ID and URL so the selected image can be previewed without an extra asset lookup.
- Thumbnail delete failures are logged as warnings; the main asset delete path still removes the asset record when the primary file delete succeeds.
