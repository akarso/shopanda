# PR-227 — Cart + Mini-Cart

## Summary

Implemented the storefront cart slice:

- added SSR cart routes for `GET /cart` plus cart mutation posts
- added header cart count and mini-cart fragments for HTMX refreshes
- wired add-to-cart from the storefront PDP using the first available variant
- kept forms progressively enhanced so cart flows still work without JavaScript

No storefront build step was introduced. The bundled theme remains server-rendered.

## Scope

### Storefront Cart Session

- Extended the storefront handler with optional cart-service and variant-repository wiring
- Added cookie-backed guest cart lookup for public storefront pages
- Reused active customer carts for authenticated storefront requests
- Allowed anonymous cart creation in the cart application service so SSR storefront routes can create guest carts without introducing a second cart implementation

### Storefront Routes

- Added `GET /cart` for the full cart page
- Added `POST /cart/add`, `POST /cart/update`, and `POST /cart/remove`
- Added fragment routes:
  - `GET /fragments/cart-count`
  - `GET /fragments/mini-cart`
  - `POST /fragments/cart/add`
  - `POST /fragments/cart/update`
  - `POST /fragments/cart/remove`

### Default Theme

- Updated the shared layout to load HTMX when cart support is enabled
- Replaced the static cart pill text with a fragment-backed cart count
- Added a small header mini-cart panel that refreshes on `cart-updated`
- Updated the default product page with an add-to-cart form
- Added `themes/default/templates/cart.html` for the full cart page
- Added cart, mini-cart, and quantity/remove styling to the bundled theme CSS

## Behavior Notes

- Guest carts are tracked with a `shopanda_storefront_cart` cookie and resolved through the existing cart service
- Authenticated users read and mutate their active customer cart instead of the guest cookie cart
- Product pages resolve the first variant for the current product and use it as the default add-to-cart target
- HTMX cart updates use full cart-page responses together with `hx-select="#cart-page"`, so the server keeps rendering one canonical page shape while the browser swaps only the cart content area
- Non-HTMX form posts still work via normal redirects, preserving progressive enhancement
