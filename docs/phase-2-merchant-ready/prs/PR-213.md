# PR-213 — Image resize + thumbnails

## Goal

Generate thumbnails on upload using configurable presets (small/medium/large).
Implements MEDIA_PROCESSING.md §2.

## Design notes

1. **ImageProcessor interface** — `Resize(io.Reader, ResizeOpts) (io.Reader, error)`.
   Domain-side contract; infra implements via `disintegration/imaging`.

2. **SetImageProcessor** — Optional dependency on Service. When nil,
   Upload works exactly as before (no thumbnails). Backward-compatible.

3. **Buffering** — Upload now buffers full file in memory (was streaming
   via countingReader). Required to feed same bytes to both storage.Save
   and processor.Resize for each preset. 10MB max still enforced.

4. **Thumbnail paths** — `uploads/{assetID}/{preset}/{filename}`.
   Stored in `Asset.Thumbnails` map (preset name → storage path).

5. **Graceful degradation** — If resize fails for a preset, a warning
   is logged and that preset is skipped. Upload succeeds with whatever
   thumbnails were generated.

6. **Fit modes** — `cover` (fill + crop), `contain` (fit within bounds,
   preserve aspect), `fill` (stretch to exact dimensions).

7. **Supported formats** — JPEG, PNG, GIF. WebP decode/encode deferred
   to PR-214.
