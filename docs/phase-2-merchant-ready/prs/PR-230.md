# PR-230 — Merchant Guide

## Summary

Added the first merchant-facing operations guide for Shopanda.

The new guide is written for non-technical store operators and focuses on the workflows that are already available in the current product:

- first admin login
- dashboard orientation
- product and variant management
- media library usage
- order review and status updates
- grouped store settings
- CSV-based bulk operations
- operational caveats where configuration is still API- or deployment-driven

## Scope

### Merchant documentation

- added `docs/guides/MERCHANT.md`
- organized the guide around task-oriented sections instead of architecture topics
- documented the seeded first-login path at `/admin`
- described the current embedded admin SPA pages:
  - dashboard
  - products
  - orders
  - media
  - settings

### Operator caveats

- documented that pricing, categories, stock, and other large catalog changes can be handled through existing CSV import/export commands
- documented that shipping-zone management exists in the platform but is currently API-backed rather than exposed in the embedded admin SPA
- documented that payment-provider setup remains deployment-level configuration
- documented that invoice handling is currently email-first, with PDF attachments rather than an admin invoice viewer

### Discoverability

- added a `Merchant Guide` link to the main `README.md` documentation section

## Files Changed

- `docs/guides/MERCHANT.md`
  - new merchant-facing guide for store operators
- `README.md`
  - added a direct link to the merchant guide

## Notes

- the guide intentionally distinguishes between workflows available inside the embedded admin SPA and workflows that still rely on CLI, API, or deployment configuration so merchants are not misled about the current surface area
- this PR stays within the documentation track only and does not change runtime behavior
