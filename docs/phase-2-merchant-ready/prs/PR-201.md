# PR-201: Dockerfile (Multi-Stage)

## Summary

Multi-stage Dockerfile producing a minimal Alpine-based image with non-root user, built-in health check, and bundled migrations/config/themes.

## Changes

### New Files

- **`Dockerfile`** — Two-stage build:
  - **Build stage**: `golang:1.25-alpine`, static binary with `-trimpath -ldflags="-s -w"`, module cache layer
  - **Runtime stage**: `alpine:3.21`, non-root `appuser` (UID 1000), `ca-certificates` + `wget` for HEALTHCHECK
  - Bundles: `migrations/`, `configs/config.example.yaml` → `configs/config.yaml`, `openapi.yaml`, `themes/`
  - `ENTRYPOINT ["shopanda"]` + `CMD ["serve"]` — override CMD for `migrate`, `seed`, etc.
  - `HEALTHCHECK` hits `GET /healthz` every 10s

- **`.dockerignore`** — Excludes `.git`, docs, editor config, dev-only files, build artifacts to keep context small and builds fast

## Usage

```bash
# Build
docker build -t shopanda .

# Run (default: serve)
docker run -p 8080:8080 --env-file .env shopanda

# Run migrations
docker run --env-file .env shopanda migrate

# Run seeder
docker run --env-file .env shopanda seed
```
