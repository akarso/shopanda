# PR-210 — Shipping zones + rate tables

## Goal

Introduce a geographic Zone entity (list of ISO countries, priority,
active flag) and weight-based RateTier entity (min/max weight, price per
zone), with full Postgres persistence and admin CRUD endpoints, as
defined in SHIPPING_ZONES.md §2.

## Scope

## Design notes

1. **Separate repository** — `ZoneRepository` is distinct from the
   existing `ShipmentRepository` — zones are configuration entities
   (admin-managed), while shipments are runtime state.

2. **TEXT[] for countries** — Postgres native `TEXT[]` is used for the
   country list, scanned/inserted via `pq.Array`. This avoids a separate
   join table for a simple list of 2-char codes.

3. **CASCADE delete** — `shipping_rate_tiers` has `ON DELETE CASCADE` on
   the `zone_id` FK, so deleting a zone automatically removes all its
   rate tiers without additional application logic.

4. **Priority-based ordering** — `ListZones` returns zones ordered by
   priority descending, matching the zone resolution spec (highest
   priority zone wins on country overlap).

5. **Weight range semantics** — `MaxWeight = 0` means unlimited
   (no upper weight bound). The domain validates `maxWeight ≥ minWeight`
   when both are positive.

6. **Default currency** — `CreateRate` defaults to `"EUR"` when no
   currency is provided in the request body. The database also defaults
   to `EUR` with a `CHECK(length(currency) = 3)` constraint.

7. **RBAC** — `ShippingRead`/`ShippingWrite` are new permissions granted
   to Admin and Manager roles. Editor and Support do not receive these
   since shipping zone configuration is an operational concern.

## Review round

| Finding | Fix |
|---|---|
| RBAC test: count-only assertion, no shipping perm check | Added `ShippingRead`/`ShippingWrite` to explicit loops in `TestHasPermission_Admin` and `TestHasPermission_Manager` |
| UpdateRate: mutates tier in place bypassing invariants | Rewrote to overlay fields then re-validate via `shipping.NewRateTier` |
| DeleteRate: ignores `zoneId`, allows cross-zone delete | Added `FindRateTierByID` lookup + `tier.ZoneID != zoneID` guard |
| Migration: no weight CHECKs | Added `min_weight >= 0`, `max_weight >= 0`, and `max_weight = 0 OR max_weight >= min_weight` |
| zone_test: no currency assertion | Added `rt.Price.Currency()` check |
| NewZone: uses caller slice, no uppercase | Now builds new slice with `strings.ToUpper(c)` normalization |
| Inefficient ListRateTiers scan for Update/Delete | Added `FindRateTierByID` to `ZoneRepository` interface + Postgres impl; both handlers use it |
| Missing handler tests | Added `UpdateRate_InvalidInvariants` (422) and `DeleteRate_RejectsMismatchedZone` (404) |
