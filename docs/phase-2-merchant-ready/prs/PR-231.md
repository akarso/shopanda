# PR-231 — Deployment Guide

## Summary

Added the deployment and operations guide for technical operators.

The guide covers the repository's current deployment surfaces instead of the older aspirational runbook flow:

- Docker quick start
- environment variable setup
- image and Compose usage
- worker and scheduler process separation
- bare metal deployment
- reverse-proxy TLS examples
- PostgreSQL backup and restore guidance
- monitoring with the real `/healthz` and `/docs` endpoints

## Scope

### Deployment documentation

- added `docs/guides/DEPLOYMENT.md`
- documented the current Dockerfile behavior:
  - multi-stage build
  - non-root runtime user
  - `shopanda` entrypoint
  - `/healthz` health check
- documented current `docker-compose.yml` behavior and profiles:
  - default `app` + `postgres`
  - optional `dev` profile for Mailpit
  - optional `search` profile for Meilisearch

### Runtime guidance

- documented the real process model for production:
  - `serve`
  - `worker`
  - `scheduler`
- documented the need to persist `/app/public/media` when local media storage is used in containers
- documented the seeded admin credential flow required by `setup` and `seed`

### Discoverability

- added a `Deployment Guide` link to the main `README.md`

## Files Changed

- `docs/guides/DEPLOYMENT.md`
  - new deployment and operations guide
- `README.md`
  - added a direct link to the deployment guide

## Notes

- the guide intentionally uses the live `/healthz` and `/docs` endpoints from the current application rather than the stale paths that appear in older operational notes
- this PR stays within the documentation track only and does not change runtime behavior
