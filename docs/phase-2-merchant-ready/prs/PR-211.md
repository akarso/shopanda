# PR-211 ‚Äî Weight-based rate calculation

## Goal

Add product weight to variant entity and implement weight-based shipping
rate calculation using zones and rate tiers from PR-210, as defined in
SHIPPING_ZONES.md ¬ß3.

## Design notes

1. **Weight on Variant** ‚Äî Stored as `float64` (kg). Defaults to 0.
   Not part of constructor since valid zero value. Set directly or
   via admin handler.

2. **ZoneRateCalculator** ‚Äî Domain service, not a Provider impl.
   Takes `RateRequest` (country, currency, items with weight/qty),
   returns `ShippingRate`. Clean separation from existing Provider
   interface avoids breaking flat rate flow.

3. **Zone resolution** ‚Äî Iterates all active zones, matches country,
   picks highest priority. Spec ¬ß2.3 ordering.

4. **Tier matching** ‚Äî `min_weight <= totalWeight AND (max_weight == 0
   OR max_weight >= totalWeight)`. When multiple tiers match, most
   specific wins (highest min_weight).

5. **Weight calculation** ‚Äî `sum(item.weight * item.quantity)` per
   spec ¬ß3.3.

6. **Currency guard** ‚Äî Returns error if tier currency doesn't match
   requested currency. Prevents silent mismatch.

7. **MethodWeightBased** ‚Äî New shipping method constant. `IsValid()`
   updated to accept both `flat_rate` and `weight_based`.

- `TestZoneRateCalculator_InactiveZoneSkipped` ‚Äî inactive zone ignored despite higher priority
- `TestZoneRateCalculator_NoZone` ‚Äî unmatched country returns error
- `TestZoneRateCalculator_NoMatchingTier` ‚Äî weight exceeds all tiers returns error
- `TestZoneRateCalculator_EmptyItems` ‚Äî empty cart (weight 0) still matches catch-all tier
- `TestZoneRateCalculator_CurrencyMismatch` ‚Äî EUR tier + USD request returns error
- `TestZoneRateCalculator_MissingCountry` ‚Äî empty country validation
- `TestZoneRateCalculator_MissingCurrency` ‚Äî empty currency validation
- `TestZoneRateCalculator_MostSpecificTier` ‚Äî overlapping tiers, most specific (highest min) wins
