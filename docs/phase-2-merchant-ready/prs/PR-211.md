# PR-211 — Weight-based rate calculation

## Goal

Add product weight to variant entity and implement weight-based shipping
rate calculation using zones and rate tiers from PR-210, as defined in
SHIPPING_ZONES.md §3.

## Scope

| Area | Change |
|---|---|
| `internal/domain/catalog/variant.go` | Added `Weight float64` field (kg, 0 = unset) to `Variant` struct |
| `migrations/037_add_variant_weight.sql` | `ALTER TABLE variants ADD COLUMN weight NUMERIC NOT NULL DEFAULT 0 CHECK (weight >= 0)` |
| `internal/infrastructure/postgres/variant_repo.go` | Updated all SELECT/INSERT/UPDATE queries and `scanVariant` to include `weight` column |
| `internal/interfaces/http/variant.go` | Added `Weight *float64` to create/update request structs; validates non-negative; sets on variant |
| `internal/domain/shipping/shipment.go` | Added `MethodWeightBased ShippingMethod = "weight_based"`, updated `IsValid()` |
| `internal/domain/shipping/zone_rate.go` | New file: `RateRequest`, `RateRequestItem`, `ZoneRateCalculator` with `CalculateRate` method — resolves zone by country, sums item weights, matches tier |
| `internal/domain/shipping/zone_rate_test.go` | 12 tests covering zone resolution, tier matching, priority, inactive zones, currency mismatch, edge cases |

## Design notes

1. **Weight on Variant** — Stored as `float64` (kg). Defaults to 0.
   Not part of constructor since valid zero value. Set directly or
   via admin handler.

2. **ZoneRateCalculator** — Domain service, not a Provider impl.
   Takes `RateRequest` (country, currency, items with weight/qty),
   returns `ShippingRate`. Clean separation from existing Provider
   interface avoids breaking flat rate flow.

3. **Zone resolution** — Iterates all active zones, matches country,
   picks highest priority. Spec §2.3 ordering.

4. **Tier matching** — `min_weight <= totalWeight AND (max_weight == 0
   OR max_weight >= totalWeight)`. When multiple tiers match, most
   specific wins (highest min_weight).

5. **Weight calculation** — `sum(item.weight * item.quantity)` per
   spec §3.3.

6. **Currency guard** — Returns error if tier currency doesn't match
   requested currency. Prevents silent mismatch.

7. **MethodWeightBased** — New shipping method constant. `IsValid()`
   updated to accept both `flat_rate` and `weight_based`.

## Tests

### Domain — zone_rate_test.go (14 tests)

- `TestZoneRateCalculator_HappyPath` — light cart hits first tier
- `TestZoneRateCalculator_HigherWeightTier` — heavier cart hits second tier
- `TestZoneRateCalculator_UnlimitedMaxWeight` — max=0 tier catches any weight
- `TestZoneRateCalculator_MultiZonePriority` — DE matches both EU and Domestic, higher priority wins
- `TestZoneRateCalculator_InactiveZoneSkipped` — inactive zone ignored despite higher priority
- `TestZoneRateCalculator_NoZone` — unmatched country returns error
- `TestZoneRateCalculator_NoMatchingTier` — weight exceeds all tiers returns error
- `TestZoneRateCalculator_EmptyItems` — empty cart (weight 0) still matches catch-all tier
- `TestZoneRateCalculator_CurrencyMismatch` — EUR tier + USD request returns error
- `TestZoneRateCalculator_MissingCountry` — empty country validation
- `TestZoneRateCalculator_MissingCurrency` — empty currency validation
- `TestZoneRateCalculator_MostSpecificTier` — overlapping tiers, most specific (highest min) wins
- `TestNewZoneRateCalculator_NilRepo` — nil repo returns error
- `TestZoneRateCalculator_MixedCaseCountryAndCurrency` — "de"/"eur" normalized to uppercase

## Review round

| Finding | Fix |
|---|---|
| `ZeroRate` dead code | Removed function and unused `shared` import |
| Case-sensitive country/currency matching | Normalize `req.Country`/`req.Currency` to uppercase via `strings.ToUpper` at entry; use `strings.EqualFold` for currency comparison |
| NaN/Inf weight accepted in variant handler | Added `math.IsNaN`/`math.IsInf` checks in both Create and Update handlers |
| Constructor panics on nil repo | Changed `NewZoneRateCalculator` to return `(*ZoneRateCalculator, error)` |
| `errors.New` + `fmt.Sprintf` concatenation | Replaced with single `fmt.Errorf` call |
| Unbounded NUMERIC for weight column | Changed to `NUMERIC(10,3)` |
| Missing tests for new behavior | Added `TestNewZoneRateCalculator_NilRepo` and `TestZoneRateCalculator_MixedCaseCountryAndCurrency` |
