# PR-218 — Admin SPA Scaffold

## Summary

Embedded admin SPA served at `/admin` via Go `embed`. Includes login page, layout shell (header + sidebar + content area), client-side routing, and SPA fallback.

## Changes

### New Files

| File | Purpose |
|------|---------|
| `internal/interfaces/http/admin_handler.go` | Go handler: embeds `admin/dist/*`, serves static files at `/admin`, SPA fallback to `index.html` |
| `internal/interfaces/http/admin_handler_test.go` | 5 tests: index, CSS, JS, SPA fallback (4 routes), trailing slash |
| `internal/interfaces/http/admin/dist/index.html` | SPA entry — layout shell with header, sidebar nav, content area |
| `internal/interfaces/http/admin/dist/admin.css` | Admin styles on top of Pico CSS (sidebar, header, login, layout) |
| `internal/interfaces/http/admin/dist/admin.js` | Vanilla JS: auth (JWT localStorage), client-side routing, login form, logout |

### Modified Files

| File | Change |
|------|--------|
| `cmd/api/main.go` | Register admin handler at `GET /admin` and `GET /admin/{path...}` |

## Architecture

```
Browser → GET /admin/*
           ↓
    AdminHandler.ServeHTTP()
           ↓
    Known static file? → serve from embed.FS
           ↓ (no)
    SPA fallback → index.html
           ↓
    Client-side JS router handles path
           ↓
    Auth check → /api/v1/auth/login (JWT)
```
