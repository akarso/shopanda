## Design Notes

1. **Interface compliance**: `meili.Engine` implements `search.SearchEngine` (Name/IndexProduct/RemoveProduct/Search) — drop-in replacement for Postgres FTS. Compile-time check via `var _ search.SearchEngine = (*Engine)(nil)`.

2. **Zero external dependencies**: Uses a hand-rolled HTTP client (`httpClient`) against the Meilisearch REST API. No SDK import needed — keeps `go.mod` clean and avoids version-pinning headaches.

3. **Testable via interface**: Internal `meiliAPI` interface extracts `addDocuments`, `deleteDocument`, `search`, and `updateSettings`. Tests mock this interface — no Meilisearch server needed.

4. **Index settings applied on startup**: `New()` calls `PATCH /indexes/{uid}/settings` with searchable, filterable, sortable, and displayed attributes per the spec schema.

5. **Faceted search**: Search requests include `facets: ["category_id"]`. Response facet distribution mapped to `SearchResult.Facets`.

6. **Filter mapping**: Query `Filters` are translated to Meilisearch filter syntax — `category` → `category_id = "..."`, `price_min`/`price_max` → range filters, `in_stock` → boolean filter.

7. **Sort mapping**: `-price` → `price:desc`, `name` → `name:asc`.

8. **Event-driven sync**: `catalog.product.created` and `catalog.product.updated` wired via `bus.OnAsync()` → `searchEngine.IndexProduct()`.

9. **Engine switch**: `cfg.Search.Engine` selects backend. Default remains `"postgres"`. Meilisearch is opt-in via `search.engine: meilisearch`.

10. **`search:reindex` updated**: Uses the same engine switch — reindex works with whichever engine is configured.

## Config

```yaml
search:
  engine: "postgres"         # or "meilisearch"
  meilisearch:
    host: "http://localhost:7700"
    api_key: ""              # SHOPANDA_SEARCH_MEILI_KEY
    index: "products"        # SHOPANDA_SEARCH_MEILI_INDEX
```

Env overlays: `SHOPANDA_SEARCH_ENGINE`, `SHOPANDA_SEARCH_MEILI_HOST`, `SHOPANDA_SEARCH_MEILI_KEY`, `SHOPANDA_SEARCH_MEILI_INDEX`.
