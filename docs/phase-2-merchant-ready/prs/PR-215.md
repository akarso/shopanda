## Design Notes

1. **Interface compliance**: `s3store.Storage` implements `media.Storage` (Name/Save/Delete/URL) — drop-in replacement for `localfs.Storage`. Compile-time check via `var _ media.Storage = (*Storage)(nil)`.

2. **S3-compatible**: Works with AWS S3, MinIO, DigitalOcean Spaces, Backblaze B2, and any S3-compatible backend. Uses `UsePathStyle = true` when a custom endpoint is set (required for MinIO).

3. **Testable via interface**: Internal `s3API` interface extracts `PutObject` and `DeleteObject` from the real S3 client. Tests use a mock implementing this interface — no network calls, no real S3 bucket needed.

4. **Upload details**: Save sets `Content-Type` (guessed from extension) and `Cache-Control: public, max-age=31536000`. ACL is only set when `public_acl: true` (opt-in for legacy ACL buckets); omitted by default for "Bucket owner enforced" compatibility.

5. **URL resolution**: When `base_url` is configured (CDN), URLs use that prefix. Otherwise, falls back to path-style `{endpoint}/{bucket}/{key}` or AWS default `https://s3.{region}.amazonaws.com/{bucket}/{key}`.

6. **Security**: `validateKey()` rejects keys with `..`, `.`, or empty segments (returns error instead of silently normalizing). Static credentials are only set when both `access_key` and `secret_key` are non-empty — otherwise the SDK default credential chain is used (IAM role, shared credentials file, etc.). Config example uses empty strings for credentials with comments showing env var names.

7. **Config**: Full env overlay support: `SHOPANDA_MEDIA_S3_ENDPOINT`, `_BUCKET`, `_REGION`, `_ACCESS_KEY`, `_SECRET_KEY`, `_BASE_URL`, `_PUBLIC_ACL`. Flatten entries for `Get()`/`GetInt()` access.

8. **Backward-compatible**: Default storage remains `"local"`. S3 is opt-in via `media.storage: s3`.

## Review Round

5 findings — all verified valid, all fixed.

| # | Finding | Fix |
|---|---------|-----|
| 1 | `config.example.yaml` had literal `${...}` placeholders for S3 credentials that `yaml.Unmarshal` would parse as strings | Replaced with empty strings + comments showing env var names and SDK default chain fallback |
| 2 | `TestURL_TrailingSlashStripped` bypassed trim logic via manual `st.baseURL` override | `newWithClient` now trims trailing slashes; test no longer overrides field |
| 3 | `cleanPath` silently normalized traversal paths instead of rejecting | Replaced with `validateKey()` returning `(string, error)` — rejects `..`, `.`, empty segments; Save/Delete return wrapped error and skip S3 call |
| 4 | `PutObject` hardcoded `ACL: public-read`, fails on "Bucket owner enforced" buckets | ACL omitted by default; only set when `PublicACL: true` config flag is enabled |
| 5 | Static credentials always set even with empty keys, overriding SDK default chain | `NewStaticCredentialsProvider` only assigned when both `AccessKey` and `SecretKey` are non-empty |
