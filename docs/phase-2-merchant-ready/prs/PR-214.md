## Design Notes

1. **Interface extension**: `ImageProcessor.Format(io.Reader, string, int) (io.Reader, error)` — decodes an encoded image and re-encodes to the target MIME type. Separates resize from format conversion.

2. **WebP encoder**: Uses `github.com/chai2010/webp` (CGO, bundled libwebp sources — no system library install needed). Chosen because no pure-Go WebP encoder exists in the Go ecosystem.

3. **Refactored encode**: Introduced `encodeImage()` helper and `supportedMime()` replacing the old `mimeToFormat()`. WebP is now a first-class format for both `Resize` and `Format`.

4. **Conversion flow**: For JPEG/PNG uploads with WebP enabled, after saving each original-format thumbnail, the service calls `processor.Format()` to produce a WebP variant stored at `{dir}/{preset}/{filename}.webp`. WebP variants appear in the response as `{preset}_webp` keys.

5. **Graceful degradation**: WebP conversion failures are logged as warnings and skipped — the original-format thumbnail is kept. WebP is entirely independent of the original thumbnail save.

6. **Config**: `media.webp.enabled` (default: true) + `media.webp.quality` (default: 80). Config values flattened for `Get/GetInt` access.

7. **Backward-compatible**: When `webp.enabled: false` or no WebP config, behavior is identical to PR-213. The `SetWebPConfig` call is optional.

## Review Round

### Fixes applied

| # | Finding | Fix |
|---|---------|-----|
| 1 | CGO requirement from `chai2010/webp` — default `enabled: true` breaks non-CGO builds | Changed default to `Enabled: false` in config.go; documented CGO requirement in config.example.yaml |
| 2 | WebP filename collision when source already ends in `.webp` — `webpFilename("photo.webp")` returns same name, overwriting the original thumbnail | Added collision guard: skip WebP variant when `webpFilename(safeFilename) == safeFilename` |
| 3 | `main.go` passes literal `true` instead of `cfg.Media.WebP.Enabled` | Changed to pass `cfg.Media.WebP.Enabled`; added warning log when WebP enabled but no thumbnail presets configured |
| 4 | `SetWebPConfig` accepts any quality value without validation | Clamp quality: `<= 0 → 80`, `> 100 → 100` |
| 5 | Double lossy compression — WebP variant was generated from already-encoded JPEG bytes via `Format()` | Changed to call `processor.Resize()` with `MimeType: "image/webp"` directly from the original `fileBytes`, eliminating the decode-JPEG-re-encode-WebP path |
| 6 | `processor.go` quality not clamped above 100 | Added `> 100 → 100` clamp in both `Resize` and `Format` |
| 7 | `service_test.go` mock didn't capture resize MimeType | Added `resizeMimes []string` + `webpErr` to mock; tests now assert WebP resize call |
| 8 | `processor_test.go` Format tests ignored `io.Copy` error | Added error checks in `TestFormat_JPEGToWebP` and `TestFormat_PNGToWebP` |
