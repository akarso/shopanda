# PR-204 ‚Äî File-Based Template Loader

## Summary

Introduces a file-based email template system that loads HTML templates from
`templates/emails/`, supports a shared layout wrapper (`_layout.html`), and
integrates with the existing `mail.Templates` registry.

## Architecture

```text
templates/emails/
  _layout.html            ‚Üê shared wrapper
  order_confirmed.html    ‚Üê (added in PR-205)
  ...

internal/domain/mail/
  loader.go               ‚Üê TemplateLoader interface + EmailData
  mail.go                 ‚Üê ErrTemplateNotFound

internal/infrastructure/email/
  file_loader.go          ‚Üê FileLoader (implements TemplateLoader)
  layout.go               ‚Üê LayoutRenderer (_layout.html ‚Üí wrapped body)
  email.go                ‚Üê LoadDir, RenderEmail, Setup
```

### Flow

```text
1. Setup(dir) ‚Üí FileLoader + LayoutRenderer
2. LoadDir(registry, loader, dir) ‚Üí reads *.html, calls registry.Register()
3. RenderEmail(registry, lr, name, to, data) ‚Üí Render + Wrap
```

### Resolution Order (design)

```text
1. DB template (future ‚Äî not in this PR)
2. File template (templates/emails/{name}.html)
3. Fallback: built-in hardcoded (RegisterTemplates in notification service)
```

File templates registered via `LoadDir` silently overwrite hardcoded templates
already in the registry, achieving file ‚Üí hardcoded fallback ordering.

## Template File Format

```html
<!-- Subject: Order {{.Data.OrderID}} ‚Äî Confirmation -->
<h1>Thank you, {{.Data.FirstName}}!</h1>
<p>Your order has been confirmed.</p>
```

- First line: `<!-- Subject: ... -->` (required, supports Go template expressions)
- Remaining lines: body HTML (inner content before layout wrapping)

## Key Design Decisions

1. **Subject in HTML comment** ‚Äî keeps templates as single files, parseable by
   editors and email preview tools.
2. **LayoutRenderer is optional** ‚Äî when `_layout.html` is absent, `Wrap()`
   returns the body unchanged, enabling layout-free testing and plain emails.
3. **LoadDir skips `_` prefix** ‚Äî `_layout.html` and future partials are not
   registered as standalone templates.
4. **Domain stays clean** ‚Äî `TemplateLoader` is a domain interface; file I/O
   lives entirely in `internal/infrastructure/email/`.
