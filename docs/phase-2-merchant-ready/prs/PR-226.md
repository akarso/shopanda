# PR-226 — Category Navigation

## Summary

Implemented the storefront category-navigation slice:

- added SSR routes for `GET /categories` and `GET /categories/{slug}`
- rendered the catalog category tree in the shared storefront header
- added category landing pages with breadcrumbs and sub-category links
- reused the existing search-backed product grid for category-filtered listings

No frontend build step. Templates and CSS remain bundled and server-rendered.

## Scope

### Storefront Routes

- Added `GET /categories` as a root category landing page
- Added `GET /categories/{slug}` for category-specific browsing
- Kept `GET /products` and `GET /search` as the generic catalog and search entry points

### Category Navigation

- Extended the storefront handler to accept the existing category repository
- Header layout now loads the category tree from `CategoryRepository.FindAll`
- Root categories render in the shared layout, with first-level children shown inline below each root item

### Category Page

- Added category page view data for:
  - selected category metadata
  - breadcrumb path from the category tree
  - child-category links
  - search-backed product results filtered by category ID
- Category pages use the same PLP composition pipeline as generic listing pages, so blocks and metadata remain consistent across `/products`, `/search`, and `/categories/{slug}`

### Default Theme

- Added `themes/default/templates/category.html`
- Updated the shared layout to render the category tree in the header
- Added breadcrumb and sub-category affordances to the default theme styling

## Behavior Notes

- Category product lists are driven by the existing search engine, filtered with the resolved category ID from the slug route
- Breadcrumbs are reconstructed from the in-memory category tree for the current request, rather than requiring a separate query path
- `GET /categories` acts as a browsable directory for root categories and still shows the shared product listing grid underneath

## Files Changed

- `internal/interfaces/http/storefront.go`
  - added category routes and page data
  - added category tree, breadcrumb, and sub-category helpers
  - wired category-filtered search-backed listings
- `internal/interfaces/http/storefront_test.go`
  - added category route coverage
  - updated storefront test fixture templates and constructor wiring
- `cmd/api/main.go`
  - wired the new storefront category routes
  - passed the category repository into the storefront handler
- `themes/default/templates/layout.html`
  - rendered the category tree in the shared header
- `themes/default/templates/category.html`
  - added the category landing-page template
- `themes/default/static/css/style.css`
  - added category-navigation, breadcrumb, and sub-category styling

## Notes

- This PR keeps category browsing SSR-first and reuses the existing search and composition contracts rather than introducing a separate catalog data path.
- Header category rendering currently shows the root categories plus one nested child level, which covers the bundled default theme without introducing a more complex mega-menu interaction model.
