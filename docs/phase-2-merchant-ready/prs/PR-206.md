# PR-206 — Invoice email + attachment

## Goal

Send an invoice email with an attached PDF when the `invoice.created` event
fires, extending the notification system with attachment support end-to-end.

## Scope

| Area | Change |
|---|---|
| `internal/domain/mail/mail.go` | Added `Attachment` struct (`Filename`, `ContentType`, `Data`); added `Attachments []Attachment` to `Message` |
| `internal/application/notification/service.go` | Added optional `invoice.InvoiceRepository` + `invoice.PDFRenderer` via `WithInvoices` / `WithPDFRenderer` options; added `HandleInvoiceCreated` handler; registered `invoice_created` fallback template; extended `enqueueEmail` to base64-serialize attachments in job payload |
| `internal/application/notification/handler.go` | Extended `EmailSendHandler.Handle` to deserialize base64-encoded attachments from job payload into `mail.Message.Attachments` |
| `internal/infrastructure/smtp/mailer.go` | Updated `Send` to emit MIME multipart/mixed when attachments present (boundary-delimited HTML body + base64-encoded attachment parts) |
| `templates/emails/invoice_created.html` | New responsive HTML template — invoice number, order summary table, "View Order" CTA, PDF attachment note |
| `cmd/api/main.go` | Wire `postgres.InvoiceRepo` + `invoicepdf.Renderer` into notification service via options; subscribe `invoice.created` → `HandleInvoiceCreated` |

## Design notes

1. **Attachment type in mail domain** — `Attachment{Filename, ContentType, Data}`
   is a simple value object. Placing it alongside `Message` keeps the domain
   model self-contained.

2. **Invoice deps are optional** — `WithInvoices` / `WithPDFRenderer` options
   mean deployments that don't wire the invoice system can still use the
   notification service. `HandleInvoiceCreated` logs a warning and returns nil
   when deps are absent.

3. **PDF is re-rendered, not read from storage** — the `media.Storage` interface
   has no `Read` method. Re-rendering via the `PDFRenderer` port is clean and
   deterministic; the invoice data is immutable after creation.

4. **Attachment serialization through job queue** — PDF bytes are base64-encoded
   in the job payload. `EmailSendHandler` decodes them back. This keeps the
   existing `map[string]interface{}` payload shape and works with JSON-backed
   job storage.

5. **SMTP multipart** — when `msg.Attachments` is non-empty, the mailer
   switches from a simple `text/html` message to `multipart/mixed` with a
   boundary-delimited structure. Filenames are RFC 2047 Q-encoded.
