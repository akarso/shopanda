# PR-200: `.env.example` + Config Documentation

## Summary

Annotated environment variable template and fully commented YAML config reference so new users can configure Shopanda without reading source code.

## Changes

### New Files

- **`.env.example`** — All 33 environment variables grouped by section (server, database, log, auth, mail, media, cache, frontend, CDN, webhooks, rate limiting, seeding, dev/test). Includes generation hints for secrets and notes on required vs optional values.

- **`configs/config.example.yaml`** — Complete YAML reference with inline comments for every section. Shows defaults, per-route rate limit examples, trusted proxy CIDRs, and cross-references to env var names.

### Modified Files

- **`docs/phase-2-merchant-ready/ROADMAP.md`** — Added `TEMPLATING_GUIDELINES.md` reference to the Rules section, ensuring all frontend PRs follow SSR-first/no-build-step constraints.

## How to Use

```bash
# Quick start
cp .env.example .env
# Edit .env — at minimum set SHOPANDA_AUTH_JWT_SECRET
openssl rand -hex 32  # generate a secret

# Or use YAML config
cp configs/config.example.yaml configs/config.yaml
```

## Tests

No code changes — build passes, all 73 test packages green.
