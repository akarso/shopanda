# PR-219 — Dashboard Overview

## Summary

Admin dashboard overview with stats API endpoint and live UI.

## Scope

- **New endpoint**: `GET /api/v1/admin/stats/overview` (behind `OrdersRead` permission)
- **Stats cards**: orders today, revenue today, total products, low stock count
- **Recent orders table**: last 10 orders with ID, customer, total, status, date
- **Admin SPA**: replaced placeholder with live dashboard fetching stats API

## Files Changed

| File | Change |
|------|--------|
| `internal/domain/admin/stats.go` | `StatsRepository` interface + `DashboardStats`/`RecentOrder` types |
| `internal/infrastructure/postgres/stats_repo.go` | Postgres implementation — 4 queries (orders today, product count, low stock, recent orders) |
| `internal/interfaces/http/stats_admin.go` | `StatsAdminHandler` with `Overview()` handler |
| `cmd/api/main.go` | Wire `StatsRepo` → `StatsAdminHandler` → route |
| `internal/interfaces/http/admin/dist/admin.js` | Dashboard renders stat cards + recent orders table |
| `internal/interfaces/http/admin/dist/admin.css` | Stat card grid, order status badges |

## Architecture

```text
StatsRepository (domain/admin)
    ↑ implements
StatsRepo (infrastructure/postgres)
    ↑ injected into
StatsAdminHandler (interfaces/http)
    ↑ wired by
main.go (cmd/api)
```

## Design Decisions

- **Dedicated read-model repo** rather than reusing order/product/stock repos — single optimised query path, no need to hydrate full aggregate objects
- **Low stock threshold = 10** (hardcoded constant in handler) — configurable later via settings
- **XSS-safe**: JS uses `document.createTextNode` for escaping user data in DOM
- **Graceful degradation**: API failure shows error message, no blank page
