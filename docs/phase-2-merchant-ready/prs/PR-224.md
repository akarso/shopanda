# PR-224 — Base Layout + CSS

## Summary

Implemented the default SSR storefront shell for the bundled theme:

- shared `layout.html` with header, search, nav, main content, and footer
- bundled storefront CSS under the default theme static directory
- static asset serving at `/static/...` from the configured theme path
- storefront home page at `/`
- product detail pages now render inside the shared storefront shell

No build step. Server-rendered templates and plain CSS only.

## Scope

### Base Layout

- Added a shared layout template in `themes/default/templates/layout.html`
- Layout includes:
  - site branding
  - search form
  - primary navigation
  - cart link placeholder
  - footer with basic store/legal links
- Product pages now inherit the shared shell instead of rendering bare page content

### Default Theme Pages

- Added `themes/default/templates/home.html`
- Updated `themes/default/templates/product.html` to use the new shared layout
- Added lightweight storefront view data in the HTTP adapter so templates can access:
  - site name
  - nav links
  - cart label
  - search action
  - current year

### Asset Pipeline

- Added bundled theme CSS files:
  - `themes/default/static/css/style.css`
  - `themes/default/static/css/custom.css`
- Added static file serving at `/static/{path...}` using the configured frontend theme path
- CSS is local to the repository and does not depend on a CDN or npm pipeline

### Storefront Routes

- Added `GET /` for the SSR storefront landing page
- Kept `GET /products/{slug}` for SSR product detail rendering
- Storefront routing remains gated by `frontend.enabled`

## Auth & Behavior

- Storefront pages remain public routes
- If the configured theme does not include `home.html`, the home handler returns `404 Not Found`
- Static asset requests use the configured theme directory, so switching the theme path automatically switches storefront assets too

## Files Changed

- `internal/interfaces/http/storefront.go`
  - added home page handler
  - added shared storefront layout view data
  - factored common render helper
- `internal/interfaces/http/storefront_test.go`
  - added home route coverage
  - updated test theme fixtures to include a home template
- `cmd/api/main.go`
  - wired storefront home route
  - wired theme static asset serving from the configured theme path
- `themes/default/templates/layout.html`
  - added the shared storefront shell
- `themes/default/templates/home.html`
  - added SSR storefront landing page
- `themes/default/templates/product.html`
  - updated product page markup to render inside the new shell
- `themes/default/static/css/style.css`
  - added bundled base storefront styling
- `themes/default/static/css/custom.css`
  - added override hook for later storefront refinements

## Notes

- Navigation and cart affordances are present in the base shell even though category, cart, and account pages land in later PRs.
- The base layout is intentionally simple and SSR-first so later storefront pages can plug into the same shell without introducing a frontend build step.
- Static assets are served from `cfg.Frontend.ThemePath`, not a hardcoded theme directory, so alternate themes can reuse the same routing contract.
