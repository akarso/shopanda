# 📧 Email Templates — Specification

## 1. Overview

Replaces hardcoded email string literals with:

* file-based HTML templates
* shared layout with header/footer
* responsive email design
* override resolution (DB → file)

Design goals:

* developer-friendly (edit files, commit to git)
* merchant-customizable (DB override later)
* responsive across email clients
* event-driven triggers (existing system)

---

## 2. Template Loader (PR-204)

---

### 2.1 File Structure

```text
templates/emails/
  _layout.html          ← shared wrapper
  order_confirmed.html
  order_shipped.html
  password_reset.html
  invoice_created.html
```

---

### 2.2 Loader Interface

```go
type TemplateLoader interface {
    Load(name string) (*Template, error)
}
```

---

### 2.3 Resolution Order

```text
1. DB template (if exists for name + locale)
2. File template (templates/emails/{name}.html)
3. Fallback: built-in default (existing hardcoded strings)
```

---

### 2.4 Layout System

Templates are wrapped in `_layout.html`:

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width">
  <style>{{ .LayoutCSS }}</style>
</head>
<body>
  <div class="container">
    <div class="header">
      <img src="{{ .LogoURL }}" alt="{{ .StoreName }}">
    </div>
    <div class="content">
      {{ .Body }}
    </div>
    <div class="footer">
      <p>{{ .StoreName }} — {{ .StoreAddress }}</p>
    </div>
  </div>
</body>
</html>
```

---

### 2.5 Template Data

Each template receives:

```go
type EmailData struct {
    StoreName    string
    StoreURL     string
    LogoURL      string
    StoreAddress string

    // Template-specific data
    Data map[string]interface{}
}
```

## 3. Core Templates (PR-205)

---

### 3.1 Order Confirmation

**Trigger:** `order.paid` event

**Data:** order ID, items, totals, shipping address, payment method

**Content:**
* Thank you message
* Order summary table (product, qty, price)
* Shipping address
* Payment summary
* "View Order" link

---

### 3.2 Password Reset

**Trigger:** `POST /auth/forgot-password`

**Data:** reset token, expiry time, customer name

**Content:**
* Reset instructions
* Reset link (time-limited)
* Security note ("if you didn't request this...")

---

### 3.3 Order Shipped

**Trigger:** `order.shipped` event (admin status change)

**Data:** order ID, tracking number (optional), carrier name

**Content:**
* Shipping confirmation
* Tracking information (if available)
* Estimated delivery (if available)
* "View Order" link

---

## 4. Invoice Email (PR-206)

---

### Trigger

`invoice.created` event

---

### Content

* Invoice number reference
* Order summary
* PDF attachment (generated by existing invoice system)

---

### Attachment Flow

```text
invoice.created event
  → notification handler
    → render email template
    → fetch PDF from invoice service
    → attach to email
    → send via mailer
```

---

### Mailer Extension

```go
type MailMessage struct {
    To          string
    Subject     string
    HTML        string
    Attachments []Attachment
}

type Attachment struct {
    Filename    string
    ContentType string
    Data        []byte
}
```

---

## 5. Responsive Design

---

### Requirements

* Max width 600px
* Inline CSS (email client compatibility)
* Table-based layout for Outlook
* Dark mode support via `@media (prefers-color-scheme: dark)`
* Tested in: Gmail, Outlook, Apple Mail

---

## 6. Non-Goals (v0)

* No template editor UI (admin can edit DB records via API)
* No A/B testing
* No marketing emails / newsletters
* No SMS or push notifications
* No MJML or external email framework
