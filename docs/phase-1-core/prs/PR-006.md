# PR-006 — Database Connection + Migrations

## Summary

Adds Postgres connectivity via `database/sql` + `lib/pq`, a file-based SQL migration runner, and `app migrate` CLI subcommand.

## Files

| File | Change |
|------|--------|
| `internal/platform/db/db.go` | New — `Open(dsn)` returns `*sql.DB` with ping verification |
| `internal/platform/db/db_test.go` | New — 1 test (invalid DSN returns error) |
| `internal/platform/migrate/migrate.go` | New — `Run(db, dir)` applies pending `.sql` files in order |
| `internal/platform/migrate/migrate_test.go` | New — 3 tests (sorted listing, empty dir, missing dir) |
| `cmd/api/main.go` | Updated — subcommand dispatch, `migrate` command |

## Public API

```go
// db package
func Open(dsn string) (*sql.DB, error)

// migrate package
func Run(db *sql.DB, dir string) (applied int, err error)
```

## Design Decisions

- **`lib/pq`** — minimal, stable Postgres driver compatible with `database/sql`. Can be swapped for `pgx` later if needed.
- **File-based migrations** — plain `.sql` files in `migrations/`, sorted by filename (e.g. `001_create_products.sql`). No code generation, no embed.
- **`schema_migrations` table** — auto-created by the runner. Tracks applied migrations by filename + timestamp.
- **Transactional migrations** — each migration runs in a transaction; if it fails, the transaction rolls back and the migration is not recorded.
- **Up-only** — no rollback support. Matches the project philosophy of forward-only changes; rollbacks can be added later if needed.
- **Subcommand dispatch** — simple `os.Args[1]` switch in `main.go`. Enough for Phase 0; will be replaced with a proper CLI framework in PR-074.

## CLI Usage

```bash
# Apply all pending migrations
go run ./cmd/api/ migrate

# Default run (no subcommand) — logs config and exits
go run ./cmd/api/
```
