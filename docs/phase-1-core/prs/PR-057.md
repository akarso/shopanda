# PR-057 — Jobs: Exponential Backoff

## Summary

Replaces the fixed 10-second retry delay in `JobQueue.Fail` with exponential backoff plus jitter.

## Changes

### Infrastructure (`internal/infrastructure/postgres/`)

- **`job_queue.go`** — Removed `retryDelay` constant. Added `backoffBase` (5s), `backoffMax` (5min), `jitterRatio` (±25%) constants and `retryDelay(attempt int) time.Duration` function computing `base × 2^attempt`, capped at max, with randomized jitter. `Fail` now reads the job's attempt count (guarded by `AND status = 'processing'`) to compute the delay before the atomic retry UPDATE.
- **`backoff_test.go`** — 3 unit tests: exponential growth across attempts, cap enforcement at high attempt numbers, first-attempt-near-base range validation.
- **`job_queue_test.go`** — `TestJobQueue_Fail_Retry` updated to verify `run_at` is in the future after retry (confirms backoff delay is applied).
