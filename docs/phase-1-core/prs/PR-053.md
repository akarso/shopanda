# PR-053 — Search Engine Interface

## Summary

Domain types and port interface for product search. Defines `SearchQuery`, `SearchResult`, `FacetValue`, a search-local `Product` type, and the `SearchEngine` interface. No implementation yet (that's PR-054).

## Changes

### Domain — Types

- **`internal/domain/search/search.go`**:
  - `SearchQuery` — `Text`, `Filters` (map), `Sort`, `Limit`, `Offset`. `Validate()` rejects negative offset/limit. `EffectiveLimit()` clamps to [1, 100], defaults to 20.
  - `SearchResult` — `Products []Product`, `Total int`, `Facets map[string][]FacetValue`.
  - `FacetValue` — `Value string`, `Count int`.
  - `Product` — lightweight search-local type (`ID`, `Name`, `Slug`, `Description`, `Attributes`). Avoids importing the `catalog` package to respect hexagonal layer isolation.

### Domain — Interface (Port)

- **`SearchEngine`** interface:
  - `Name() string` — engine identifier (e.g. "postgres").
  - `IndexProduct(ctx, p Product) error` — add/update product in search index.
  - `RemoveProduct(ctx, productID string) error` — remove product from search index.
  - `Search(ctx, query SearchQuery) (SearchResult, error)` — execute search query.

### Tests

- **`internal/domain/search/search_test.go`** — 5 tests (13 sub-cases):
  - `TestSearchQuery_Validate` — valid empty, valid full, negative offset, negative limit.
  - `TestSearchQuery_EffectiveLimit` — zero/negative → default 20, within range, at max, above max clamped.
  - `TestFacetValue` — field access.
  - `TestProduct` — field access including attributes map.
  - `TestSearchResult` — products slice, total, facets map.

### C4 Diagrams

- **`docs/diagrams/c4-code.md`** — added `SearchEngine` interface, `SearchQuery`, `SearchResult`, `FacetValue`, `SearchProduct` to domain entities and ports sections.
