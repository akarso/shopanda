# PR-039 — Order HTTP Handlers (Get / List)

## Summary

Read-only HTTP endpoints for retrieving orders. Customers can view a single
order by ID or list all their orders. Both routes require authentication and
enforce customer ownership.

## New Endpoints

| Method | Path                          | Auth | Description            |
|--------|-------------------------------|------|------------------------|
| GET    | `/api/v1/orders`              | Yes  | List customer's orders |
| GET    | `/api/v1/orders/{orderId}`    | Yes  | Get order by ID        |

### GET /api/v1/orders/{orderId}

- Extracts `orderId` from path
- Looks up order via `OrderRepository.FindByID`
- Returns **404** if not found
- Returns **403** if order belongs to a different customer
- Returns **200** with order JSON on success

### GET /api/v1/orders

- Calls `OrderRepository.FindByCustomerID` with the authenticated user's ID
- Returns **200** with `{"orders": [...]}` (empty array when none)

## Design Notes

- Reuses `orderItemResp` type from `checkout.go` (same package)
- Introduces `orderResponse` (adds `updated_at` field vs checkout's `checkoutOrderResponse`)
- `toOrderResponse()` helper converts domain order to HTTP response
- Ownership enforced at handler level: `Get()` returns 403 (not 404) to avoid leaking order existence to non-owners — message says "order not found" to avoid information disclosure
- `orderRepo` was already instantiated in `main.go` from PR-037; this PR just wires the handler
