## Design

### Price Entity

Each `Price` ties a `shared.Money` value (amount + currency) to a variant ID. One variant can have multiple prices (one per currency). Constructor validates non-empty ID and variant ID.

### Storage

- `BIGINT` column stores amount in smallest currency unit (cents), matching the domain `Money` type
- `UNIQUE (variant_id, currency)` constraint enforces one price per variant per currency
- `Upsert` uses `ON CONFLICT ... DO UPDATE` for idempotent price setting

### Repository Interface

- `FindByVariantAndCurrency(ctx, variantID, currency)` — returns `(*Price, nil)` or `(nil, nil)` when not found
- `ListByVariantID(ctx, variantID)` — all prices for a variant, ordered by currency
- `Upsert(ctx, *Price)` — insert or update; nil guard
