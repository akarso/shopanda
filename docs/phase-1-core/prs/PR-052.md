# PR-052 — Collections Domain + Storage

## Summary

Manual and dynamic collection entities with full repository interface and PostgreSQL implementation. Collections provide curated or rule-based product groupings alongside the existing category hierarchy.

## Changes

### Domain — Entity

- **`internal/domain/catalog/collection.go`** — `Collection` entity with `ID`, `Name`, `Slug`, `Type` (manual/dynamic), `Rules` (JSONB for dynamic), `Meta` (JSONB), timestamps.
  - `CollectionType` string type with `CollectionManual` and `CollectionDynamic` constants + `IsValid()`.
  - `NewCollection(id, name, slug, collType)` — validates all fields including type, trims whitespace.
  - `NewCollectionFromDB(...)` — hydration factory, initializes nil maps.
  - `IsManual()`, `IsDynamic()` — type helpers.

### Domain — Repository Interface

- **`internal/domain/catalog/collection_repository.go`** — `CollectionRepository` with:
  - `FindByID`, `FindBySlug`, `List` — read operations.
  - `Create`, `Update` — write operations.
  - `AddProduct`, `RemoveProduct`, `ListProductIDs` — manual collection product assignment.

### Infrastructure — PostgreSQL

- **`internal/infrastructure/postgres/collection_repo.go`** — full implementation:
  - `hydrateCollection` scan function (works with Row and Rows).
  - Constraint-name switch for 23505 errors on Create/Update.
  - Deferred `UpdatedAt` mutation (only after successful DB update).
  - `AddProduct` — idempotency via conflict detection.
  - `RemoveProduct` — `RowsAffected() == 0` → NotFound.
  - `ListProductIDs` — ordered by product_id asc.

### Migration

- **`migrations/015_create_collections.sql`**:
  - `collections` table — TEXT PK, UNIQUE slug, CHECK constraint on type, JSONB rules/meta.
  - `collection_products` join table — composite PK, cascading deletes, index on product_id.

### Wiring

- **`cmd/api/main.go`** — `CollectionRepo` instantiated (unused pending HTTP handlers PR).

### Tests

- **`internal/domain/catalog/collection_test.go`** — 8 tests (15 sub-cases): constructor fields, dynamic type, validation (empty/whitespace × 3 + invalid type), type validity, IsManual, IsDynamic, FromDB, FromDB nil maps.
