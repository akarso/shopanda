# PR-016 — Variant HTTP Handlers

## Scope

CRUD endpoints for variants nested under `/products/{id}/variants`.

## Files Created

- `internal/interfaces/http/variant.go` — `VariantHandler` with `List`, `Get`, `Create`, `Update`
- `internal/interfaces/http/variant_test.go` — 21 tests

## Files Modified

- `cmd/api/main.go` — wired `VariantRepo`, `VariantHandler`, and 4 new routes

## Routes

| Method | Path                                              | Purpose        |
| ------ | ------------------------------------------------- | -------------- |
| GET    | `/api/v1/products/{id}/variants`                  | List variants  |
| GET    | `/api/v1/products/{id}/variants/{variantId}`      | Get variant    |
| POST   | `/api/v1/admin/products/{id}/variants`            | Create variant |
| PUT    | `/api/v1/admin/products/{id}/variants/{variantId}`| Update variant |

## Design Decisions

- Single `VariantHandler` struct for read + write — all ops require parent product verification via shared `requireProduct` helper
- `Get` and `Update` enforce `v.ProductID == p.ID` to prevent cross-product access
- Partial update via `*string` pointer fields (consistent with `ProductAdminHandler`)
- No composition pipeline for variants (none defined yet)

## Tests (21)

- **Create:** OK, with attributes, missing SKU, invalid body, duplicate SKU, product not found
