# PR-032: Registration + Login

## Summary

Implements customer registration, login, and "me" endpoints with bcrypt password hashing and HMAC-SHA256 JWT authentication.

## Changes

### New Packages

| Package | Purpose |
|---------|---------|
| `internal/platform/password` | bcrypt wrapper — `Hash(plain)`, `Compare(hash, plain)` |
| `internal/platform/jwt` | HMAC-SHA256 JWT — `Issuer.Create(sub, role)`, `Issuer.Parse(token)`, `TokenParser` adapter for `auth.TokenParser` interface |
| `internal/application/auth` | Auth service — `Register`, `Login`, `Me` use cases |

### New Files

- `internal/platform/password/password.go` — bcrypt hash/compare (cost 12)
- `internal/platform/password/password_test.go` — 3 tests
- `internal/platform/jwt/jwt.go` — JWT issuer with create/parse, HMAC-SHA256 signing
- `internal/platform/jwt/jwt_test.go` — 7 tests (round-trip, expired, tampered, malformed, validation)
- `internal/platform/jwt/parser.go` — `TokenParser` adapter implementing `auth.TokenParser`
- `internal/platform/jwt/parser_test.go` — 2 tests
- `internal/application/auth/service.go` — Auth application service
- `internal/application/auth/service_test.go` — 14 tests (register, login, me, validation, edge cases)
- `internal/interfaces/http/auth.go` — `AuthHandler` with Register/Login/Me handlers
- `internal/interfaces/http/auth_test.go` — 6 HTTP handler tests

### Modified Files

- `internal/platform/config/config.go` — Added `AuthConfig` with `jwt_secret` and `jwt_ttl` fields, env overlay (`SHOPANDA_AUTH_JWT_SECRET`, `SHOPANDA_AUTH_JWT_TTL`), defaults (`jwt_ttl: "24h"`)
- `cmd/api/main.go` — Wired JWT issuer, token parser (replaces devauth), customer repo, auth service, auth handler, and auth routes

### Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/api/v1/auth/register` | Public | Create account, returns JWT |
| POST | `/api/v1/auth/login` | Public | Authenticate, returns JWT |
| GET | `/api/v1/auth/me` | Bearer JWT | Returns authenticated customer profile |

### Configuration

| Key | Env Var | Default | Description |
|-----|---------|---------|-------------|
| `auth.jwt_secret` | `SHOPANDA_AUTH_JWT_SECRET` | (empty — must be set) | HMAC-SHA256 signing key |
| `auth.jwt_ttl` | `SHOPANDA_AUTH_JWT_TTL` | `24h` | Token expiration duration |

### Architecture

- **Domain**: Customer entity (PR-031) unchanged
- **Application**: `auth.Service` orchestrates register/login using customer repo + password hasher + JWT issuer
- **Platform**: `password` and `jwt` packages are pure library code with no domain dependencies
- **Interfaces**: `AuthHandler` maps HTTP requests to service calls
- **Wiring**: `cmd/api/main.go` creates JWT issuer from config, replaces `devauth.Parser` with `jwt.TokenParser`

## Test Results

All 27 packages pass (3 new: password, jwt, application/auth).
