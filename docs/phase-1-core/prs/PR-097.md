# PR-097 — Content Translations

## Summary

Adds entity-level content translations so that translatable fields (product name/description, page title/content) can be served in the visitor's resolved language.

## What Changed

### Domain Layer

- **`internal/domain/translation/content_translation.go`** — `ContentTranslation` entity (`EntityID`, `Language`, `Field`, `Value`) with `NewContentTranslation` constructor that validates all fields, normalises BCP 47 language tags via `golang.org/x/text/language`, and lowercases field names for canonical storage.
- **`internal/domain/translation/content_repository.go`** — `ContentTranslationRepository` interface: `FindByEntityAndLanguage`, `FindFieldValue`, `Upsert`, `DeleteByEntity`.
- **`internal/domain/translation/content_translator.go`** — `ContentTranslator` resolves the request language from context, fetches translations from the repository, and returns a `map[string]string` of field→value. Accepts an optional `ContentTranslatorLogger` to warn on repository errors.
- **`internal/domain/translation/content_translation_test.go`** — Constructor validation, language normalisation, and translator overlay tests. Mocks return empty slices (not nil) to honour repository contract.

### Infrastructure Layer

- **`internal/infrastructure/postgres/content_translation_repo.go`** — PostgreSQL implementation of `ContentTranslationRepository` with `ON CONFLICT … DO UPDATE` upsert, compile-time interface check, and validation-on-write using normalized values from `NewContentTranslation`.
- **`migrations/032_create_content_translations.sql`** — `content_translations` table with composite primary key `(entity_id, language, field)` and a covering index on `(entity_id, language)`.

### Interface Layer

- **`internal/interfaces/http/page.go`** — `PageHandler` now accepts an optional `*ContentTranslator`; `Get()` overlays `title` and `content` from content translations.
- **`internal/interfaces/http/product.go`** — `ProductHandler` now accepts an optional `*ContentTranslator`; `Get()` and `List()` overlay `name` and `description` via `overlayProductTranslation`.
- **`internal/interfaces/http/page_test.go`** — Updated constructor calls; added `TestPageHandler_Get_WithContentTranslation`.
- **`internal/interfaces/http/product_test.go`** — Updated constructor calls; added `TestProductHandler_Get_WithContentTranslation`.

### Wiring

- **`cmd/api/main.go`** — Creates `contentTranslationRepo` and `contentTranslator`, passes to `NewPageHandler` and `NewProductHandler`.

## Design Decisions

- The `ContentTranslator` is **optional** (`nil`-safe) so existing code runs unchanged when no translator is wired.
- Translations overlay fields **after** composition pipelines execute, keeping pipeline logic language-agnostic.
- Field keys (`name`, `description`, `title`, `content`) are stored lowercase to match a consistent convention in the `content_translations` table.

## Migration

```sql
-- 032_create_content_translations.sql
CREATE TABLE content_translations (
    entity_id TEXT NOT NULL,
    language  TEXT NOT NULL,
    field     TEXT NOT NULL,
    value     TEXT NOT NULL,
    PRIMARY KEY (entity_id, language, field)
);
CREATE INDEX idx_content_translations_entity_language
    ON content_translations (entity_id, language);
```
