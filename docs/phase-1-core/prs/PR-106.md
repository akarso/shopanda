# PR-106 — OpenAPI spec + docs endpoint

## Summary

Adds a hand-written OpenAPI 3.0.3 specification covering all 48 public and
admin endpoints, plus a `GET /docs` route that serves Swagger UI pointing
at the spec.

## Changes

### `openapi.yaml` (new)
- OpenAPI 3.0.3 spec with all public, authenticated, and admin endpoints.
- Grouped into tags: Health, Auth, Products, Variants, Categories, Search,
  Pages, Cart, Checkout, Orders, Account, Shipping, Payments, Admin: Products,
  Admin: Orders, Admin: Media, Admin: Pages, Admin: Stores, Admin: Schemas, SEO.
- Reusable components: `Envelope`, `ErrorBody`, `Cart`, `CartItem`, request
  schemas (`RegisterRequest`, `LoginRequest`, `CreateProductRequest`, etc.),
  shared parameters (`Offset`, `Limit`, `ProductId`, `CartId`, etc.),
  standard error responses (`Unauthorized`, `Forbidden`, `NotFound`,
  `Validation`, `RateLimited`).
- Bearer JWT security scheme.

### `internal/interfaces/http/docs.go` (new)
- `DocsHandler` struct holding pre-loaded spec bytes.
- `NewDocsHandler(specBytes)` constructor.
- `Spec()` — serves `GET /docs/openapi.yaml` with `Content-Type: application/yaml`.
- `UI()` — serves `GET /docs` as an HTML page that loads Swagger UI from the
  unpkg CDN and points it at `/docs/openapi.yaml`.

### `cmd/api/main.go`
- Reads `openapi.yaml` from the project root at startup.
- Registers `GET /docs` and `GET /docs/openapi.yaml` alongside the other
  infrastructure routes (healthz, sitemap, robots).

## New tests

### `internal/interfaces/http/docs_test.go`
- `TestDocsHandler_UI` — verifies 200, `text/html` content type, body
  contains `swagger-ui` and `/docs/openapi.yaml` references.
- `TestDocsHandler_Spec` — verifies 200, `application/yaml` content type,
  body matches the provided spec bytes exactly.
- `TestDocsHandler_EmptySpec` — verifies handler works gracefully when
  constructed with nil spec bytes (empty response body, still 200).

## Accessing the docs

```
GET /docs              → Swagger UI (HTML)
GET /docs/openapi.yaml → raw OpenAPI spec (YAML)
```

The Swagger UI loads the spec client-side from `/docs/openapi.yaml` and
renders an interactive explorer for all API endpoints.
