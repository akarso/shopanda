# PR-080 — CLI: Full Command Set

## Summary

Completes the CLI command set by adding the three missing commands from the
specification: `help`, `worker`, and `search:reindex`. Also reorganises the
switch dispatch for readability (grouped by category) and improves the
unknown-command error to suggest `help`.

## Changes

### Modified files

| File | Change |
|------|--------|
| `cmd/api/main.go` | Added `help`, `worker`, `search:reindex` cases; reorganised switch order; improved unknown-command message |

## New Commands

### `help`

Prints all available commands with one-line descriptions. No database or
config access required beyond initial config load.

```text
Usage: app <command> [arguments]

Commands:
  serve                Start the HTTP server (default)
  worker               Start the background job worker
  scheduler            Start the cron scheduler
  migrate              Run database migrations
  seed                 Seed the database with initial data
  search:reindex       Re-index all products in the search engine
  config:export        Export configuration to stdout (YAML)
  config:import <file> Import configuration from a YAML file
  import:products <f>  Import products from a CSV file
  ...
```

### `worker`

Starts a standalone background job worker, identical to the embedded worker
in `serve` but running as its own process. Registers the same handlers:

- `notification.EmailSendHandler` — processes `email.send` jobs
- `cacheApp.CleanupHandler` — processes `cache.cleanup` jobs

Blocks until `SIGINT` / `SIGTERM`, then exits.

### `search:reindex`

Iterates all products via `productRepo.List` in batches of 100 and calls
`searchEngine.IndexProduct` for each one. Logs total indexed count on
completion.

Mapping: `catalog.Product` → `search.Product` (ID, Name, Slug, Description,
Attributes).
