# PR-009 — Money Type

## Scope

`domain/shared/money.go` — `Money` struct + helpers.

## What Changed

### New: `internal/domain/shared/money.go`

- **`Money`** — Value type with `Amount int64` (smallest currency unit, e.g. cents) and `Currency string` (ISO 4217).
- **`NewMoney(amount, currency)`** — Constructor.
- **`Zero(currency)`** — Zero-value constructor.
- **Arithmetic**: `Add()`, `Sub()`, `Mul(qty)` — currency-safe (panics on mismatch).
- **Predicates**: `IsZero()`, `IsPositive()`, `IsNegative()`, `Equal()`.
- **`String()`** — Human-readable format (`"1299 EUR"`).

### New: `internal/domain/shared/money_test.go`

12 tests covering:
- Construction (`NewMoney`, `Zero`)
- Arithmetic (`Add`, `Sub`, `Mul`)
- Predicates (`IsZero`, `IsPositive`, `IsNegative`)
- Equality (same, different currency, different amount)
- String formatting
- Currency mismatch panics (Add, Sub)

## Design Decisions

- **int64 cents** — avoids floating-point errors per `DOMAIN_MODEL.md` spec.
- **Panic on currency mismatch** — programming error, not a runtime condition. Catches bugs at system boundaries before they propagate.
- **Value type (not pointer)** — immutable semantics, all methods return new values.
