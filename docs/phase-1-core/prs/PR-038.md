# PR-038: Checkout HTTP Endpoint

## Summary

Adds `POST /api/v1/checkout` — the full checkout flow exposed as a single authenticated endpoint. Accepts a cart ID, runs the checkout workflow (validate → price → reserve → create order), and returns the created order.

## Files Changed

| File | Change |
|------|--------|
| `internal/interfaces/http/checkout.go` | New — `CheckoutHandler` with `StartCheckout()` handler |
| `internal/interfaces/http/checkout_test.go` | New — 7 integration tests |
| `cmd/api/main.go` | Wire `CheckoutHandler`, register `POST /api/v1/checkout` route |

## API

```text
POST /api/v1/checkout
Authorization: Bearer <token>
Content-Type: application/json

{
  "cart_id": "uuid"
}
```

### Success Response (201 Created)

```json
{
  "data": {
    "order": {
      "id": "uuid",
      "customer_id": "uuid",
      "status": "pending",
      "currency": "EUR",
      "items": [
        {
          "variant_id": "uuid",
          "sku": "SKU-001",
          "name": "Widget A",
          "quantity": 2,
          "unit_price": 1500,
          "currency": "EUR"
        }
      ],
      "total_amount": 3000,
      "created_at": "2026-04-04T12:00:00Z"
    }
  },
  "error": null
}
```

### Error Responses

| Condition | Status | Code |
|-----------|--------|------|
| Missing/invalid JSON body | 422 | `validation` |
| Missing `cart_id` | 422 | `validation` |
| Not authenticated | 401 | `unauthorized` |
| Cart not found | 404 | `not_found` |
| Cart belongs to another customer | 403 | `forbidden` |
| Cart is empty | 422 | `validation` |

## Design Decisions

- Follows the established handler pattern: struct with service, method returning `http.HandlerFunc`.
- Customer ID extracted from auth context (`auth.IdentityFrom`), not from request body — prevents spoofing.
- All checkout validation (cart ownership, emptiness, active status) handled by `checkout.Service.StartCheckout`, keeping the handler thin.
- Response includes full order snapshot (items with SKU, name, unit price, currency) so the client has everything needed to display a confirmation.
- Route placed behind `RequireAuth` middleware.
