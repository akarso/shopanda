## Design

### Pipeline Executor

`Pipeline` holds an ordered list of `PricingStep` implementations. `Execute` runs each step sequentially against a `*PricingContext`. An error from any step halts the pipeline and is wrapped with the step name.

### Base Price Step (application layer)

Looks up each item's price via `PriceRepository.FindByVariantAndCurrency`, then sets `UnitPrice` and `Total` (via `MulChecked`). Returns an error if a price is missing or the repository fails. Lives in `internal/application/pricing` because it coordinates a domain interface with infrastructure.

### Finalize Step (domain layer)

Pure computation — no external dependencies:

1. Sums item `Total` values into `Subtotal`
2. Aggregates item-level and context-level adjustments by type (discount/tax/fee)
3. Computes `GrandTotal = Subtotal - DiscountsTotal + TaxTotal + FeesTotal`
