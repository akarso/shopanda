# PR-027 — Cart Application Service

## Summary

Adds the cart application service that orchestrates cart use cases: creating
carts, adding/updating/removing items, and recalculating prices via the
pricing pipeline after every mutation.

## Files Changed

| File | Change |
|---|---|
| `internal/application/cart/service.go` | `Service` with cart CRUD + pricing recalculation |
| `internal/application/cart/service_test.go` | 18 unit tests with stub repositories |

## Design

### Dependencies

```text
Service
  ├── cart.CartRepository     (persistence)
  ├── pricing.PriceRepository (price lookup for AddItem)
  ├── pricing.Pipeline        (recalculate after mutations)
  └── logger.Logger           (structured logging)
```

### Operations

| Method | Flow |
|---|---|
| `CreateCart` | `NewCart` → `Save` |
| `GetCart` / `GetActiveCartByCustomer` | `FindByID` / `FindActiveByCustomerID` → not-found check |
| `AddItem` | `FindByID` → lookup base price → `Cart.AddItem` → recalculate → `Save` |
| `UpdateItemQuantity` | `FindByID` → `Cart.UpdateItemQuantity` → recalculate → `Save` |
| `RemoveItem` | `FindByID` → `Cart.RemoveItem` → recalculate → `Save` |

### Recalculate via pipeline

After every cart mutation, `recalculate` builds a `PricingContext` from the
cart items, runs the pricing pipeline (BasePriceStep + FinalizeStep), and
writes computed `UnitPrice` values back onto cart items. This ensures prices
stay current and extensible (discounts, taxes can be added as pipeline steps).

### Error mapping

- Cart/item not found → `apperror.NotFound`
- Domain validation failures → `apperror.Validation`
- Infrastructure errors → wrapped with `fmt.Errorf`

## Tests

18 unit tests using in-memory stub repositories:
- CreateCart (valid + invalid currency)
- GetCart (found + not found)
- GetActiveCartByCustomer (found + not found)
- AddItem (success, not found, no price, merge quantity, save error)
- UpdateItemQuantity (success, cart not found, item not found)
- RemoveItem (success, cart not found, item not found)
- Recalculate verifies pipeline sets correct prices on multi-item cart
