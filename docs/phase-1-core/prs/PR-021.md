# PR-021 — Identity Middleware + Roles

## Summary

Introduces the minimal auth foundation: an `Identity` domain type with role enumeration, context helpers for request-scoped identity, a `TokenParser` interface (port), and three composable HTTP middleware functions.

## Scope

Phase 4 — Minimal Auth (first PR).

## Changes

### Domain: `internal/domain/identity/`

| File | Description |
|------|-------------|
| `doc.go` | Package documentation |
| `identity.go` | `Identity` struct (`UserID`, `Role`), `Role` enum (`guest`, `customer`, `admin`), constructors (`NewIdentity`, `Guest`), query methods (`IsGuest`, `IsAuthenticated`, `HasRole`) |
| `identity_test.go` | 7 tests: role validation, identity construction, guest identity, authentication checks, role matching |

### Platform: `internal/platform/auth/`

| File | Description |
|------|-------------|
| `context.go` | `WithIdentity` / `IdentityFrom` — store and retrieve `Identity` from `context.Context` (mirrors `requestctx` pattern) |
| `token.go` | `TokenParser` interface — `Parse(ctx, token) (Identity, error)` port for pluggable token strategies |
| `context_test.go` | 2 tests: round-trip storage, empty context returns guest |

### Interfaces: `internal/interfaces/http/`

| File | Description |
|------|-------------|
| `auth_middleware.go` | `AuthMiddleware(parser)` — parses `Authorization: Bearer <token>`, injects identity into context; `RequireAuth()` — rejects guests (401); `RequireRole(role)` — rejects wrong role (403) or guests (401) |
| `auth_middleware_test.go` | 9 tests: no header → guest, valid token, invalid token → 401, malformed header → 401, authenticated pass-through, guest rejection, role match, role mismatch → 403, guest on role check → 401 |

## Architecture

```text
domain/identity  ←  platform/auth  ←  interfaces/http
   (types)         (context, port)     (middleware)
```

Dependency direction follows hexagonal rules: interfaces → platform → domain.

## Error Responses

| Scenario | Code | Status | Message |
|----------|------|--------|---------|
| No/invalid `Bearer` prefix | `unauthorized` | 401 | invalid authorization header |
| Token parse failure | `unauthorized` | 401 | invalid or expired token |
| Guest accessing auth-required route | `unauthorized` | 401 | authentication required |
| Wrong role | `forbidden` | 403 | insufficient permissions |

Uses existing `apperror.CodeUnauthorized` and `apperror.CodeForbidden` — no new error codes needed.
