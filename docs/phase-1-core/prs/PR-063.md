# PR-063 — Caching Layer

## Summary

Introduces the `Cache` domain port and a zero-dependency PostgreSQL UNLOGGED table implementation. Adds config, migration, wiring, and tests.

## Changes

### Domain (`internal/domain/cache/`)

- **`cache.go`** — `Cache` interface: `Get(key, dest) (bool, error)`, `Set(key, value, ttl) error`, `Delete(key) error`.

### Migration (`migrations/`)

- **`019_create_cache.sql`** — `cache` UNLOGGED table (key TEXT PK, value JSONB, expires_at TIMESTAMPTZ, created_at). Partial index on `expires_at` for cleanup queries.

### Infrastructure (`internal/infrastructure/postgres/`)

- **`cache_store.go`** — `CacheStore` implementing `cache.Cache`. Uses `INSERT ... ON CONFLICT DO UPDATE` for upsert, application-level TTL check in `Get`, and a `DeleteExpired()` helper for the cleanup job (PR-065).
- **`cache_store_test.go`** — 7 tests via in-memory stub: hit, miss, delete, delete-missing, overwrite, expired entry, no-TTL. Compile-time interface check for both `CacheStore` and stub.

### Config (`internal/platform/config/`)

- **`config.go`** — Added `CacheConfig` struct with `Driver` field. Default: `"postgres"`. Env overlay: `SHOPANDA_CACHE_DRIVER`. Added to `flatten()` and `String()`.

### Config file (`configs/`)

- **`config.yaml`** — Added `cache.driver: "postgres"`.

### Wiring (`cmd/api/main.go`)

- Creates `CacheStore` based on `cfg.Cache.Driver` (currently only `"postgres"`).
- Placeholder `_ = appCache` — consumers wired in upcoming PRs.

### Diagrams

- **`c4-component.md`** — Added `PostgresCacheStore` in infrastructure layer.
- **`c4-code.md`** — Added `Cache` interface and `PostgresCacheStore` adapter with `implements` relationship.

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| UNLOGGED table | Acceptable for cache data — survives restarts but not crashes. Faster writes. |
| Application-level expiry check | Ensures stale entries aren't returned even before cleanup job runs. |
| JSONB value column | Allows caching any serialisable Go value. |
| Partial index on `expires_at` | Speeds up `DELETE FROM cache WHERE expires_at < now()` without bloating the index for no-TTL entries. |
| `DeleteExpired()` on adapter | Not part of the `Cache` interface — only needed by the cleanup job, keeps the port minimal. |
