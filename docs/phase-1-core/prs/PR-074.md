# PR-074 — Customer Import/Export

## Summary

Adds `CustomerImporter` and `CustomerExporter` for CSV-based customer data
management, plus `import:customers` and `export:customers` CLI commands.

## Changes

### Modified: `internal/domain/customer/repository.go`

- Added `ListCustomers(ctx context.Context, offset, limit int) ([]Customer, error)`
  to the `CustomerRepository` interface for paginated customer enumeration

### Modified: `internal/infrastructure/postgres/customer_repo.go`

- Added `ListCustomers` implementation: `SELECT ... FROM customers ORDER BY
  email LIMIT $1 OFFSET $2`
- Validates offset >= 0 and limit > 0; caps limit at 100
- Added `query` helper method (previously only `queryRow` and `exec` existed)

### New: `internal/application/importer/customers.go`

- `CustomerImporter` struct using `CustomerRepository`
- `Import(ctx, r io.Reader) (*CustomerResult, error)` — parses CSV with
  required column `email`, optional columns `first_name`, `last_name`, `role`,
  `status`, `password`
- Creates customers via `NewCustomer(id.New(), email)`, sets optional fields,
  validates role/status via `IsValid()`, hashes password via `password.Hash()`
- Duplicate email collisions handled as row-level errors (not fatal)
- `CustomerResult` struct: `Created`, `Skipped`, `Errors []string`

### New: `internal/application/exporter/customers.go`

- `CustomerExporter` struct using `CustomerRepository`
- `Export(ctx, w io.Writer) (*CustomerResult, error)` — paginates through
  `ListCustomers`, writes CSV with columns `email`, `first_name`, `last_name`,
  `role`, `status`
- Password hash is **never** included in export output
- Applies `sanitizeCSVCell()` to email, first_name, last_name fields
- `CustomerResult` struct: `Entries` count

### New: `internal/application/importer/customers_test.go`

9 tests:
- `TestCustomerImport_Basic` — happy path, 2 customers with all fields
- `TestCustomerImport_MissingEmailColumn` — rejects CSV without email column
- `TestCustomerImport_EmptyEmail` — blank email skipped with error
- `TestCustomerImport_InvalidRole` — invalid role produces error
- `TestCustomerImport_InvalidStatus` — invalid status produces error
- `TestCustomerImport_DuplicateEmail` — second row with same email skipped
- `TestCustomerImport_PasswordHashed` — password stored as bcrypt hash
- `TestCustomerImport_EmailOnly` — email-only CSV uses role/status defaults
- `TestCustomerImport_MixedValidAndInvalid` — mixed rows correctly counted

### New: `internal/application/exporter/customers_test.go`

5 tests:
- `TestCustomerExport_Basic` — 2 customers exported with correct CSV structure
- `TestCustomerExport_Empty` — header-only output
- `TestCustomerExport_ListError` — repository error propagated
- `TestCustomerExport_Pagination` — 150 customers across multiple pages
- `TestCustomerExport_NoPasswordInOutput` — password hash never appears in CSV

### Modified: `cmd/api/main.go`

- Added `case "import:customers"` and `case "export:customers"` CLI commands
- `runImportCustomers`: opens file, creates importer, logs summary + row errors
- `runExportCustomers`: atomic write via temp file + `os.Rename`

### Modified: `internal/interfaces/http/auth_test.go`

- Added `ListCustomers` stub to `authMockCustomerRepo` mock

### Modified: `internal/application/auth/service_test.go`

- Added `ListCustomers` stub to `mockCustomerRepo` mock

### Modified: `docs/diagrams/c4-component.md`

- Added `CustomerImporter` and `CustomerExporter` components to application layer
- Added relationships to `postgresRepos`

## Test Results

All 50 packages pass (`go test ./... -count=1 -short -timeout 180s`).

14 new tests total (9 importer + 5 exporter).

## Security

- Passwords are hashed via bcrypt (`password.Hash()`) during import — never
  stored in plaintext
- Password hashes are excluded from CSV export output
- CSV formula injection protection via `sanitizeCSVCell()` on user-supplied fields
