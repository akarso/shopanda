# PR-064 — Configuration: DB Layer

## Summary

Adds a database-backed configuration table with a domain repository interface, PostgreSQL adapter, and two CLI commands (`config:export`, `config:import`) for dumping/loading config entries.

## Changes

### Domain Layer
- `internal/domain/config/repository.go` — `Entry` struct (Key, Value) and `Repository` interface: `Get`, `Set`, `Delete`, `All`

### Infrastructure Layer
- `internal/infrastructure/postgres/config_repo.go` — `ConfigRepo` implementing `config.Repository` with PostgreSQL JSONB storage. Accepts `configDB` interface (satisfied by both `*sql.DB` and `*sql.Tx`) for transactional use. Uses `INSERT ... ON CONFLICT` for upsert, returns `(nil, nil)` on miss, rejects nil values, idempotent delete.

### Migration
- `migrations/020_create_config.sql` — Creates `config` table (`key TEXT PRIMARY KEY`, `value JSONB NOT NULL`)

### CLI Commands (wired in `cmd/api/main.go`)
- `config:export` — Reads all DB config entries via `ConfigRepo.All()`, writes flat key-value YAML to stdout (dot-notation keys preserved as-is, map-valued leaves not expanded)
- `config:import <file.yaml>` — Reads a flat YAML file inside a single DB transaction; all entries succeed or the import is rolled back

### Tests
- `internal/infrastructure/postgres/config_repo_test.go` — 9 integration tests (`postgres_test` package): SetAndGet, GetMiss, Upsert, Delete, DeleteMissing, All, AllEmpty, SetNilValue, ComplexValue. Uses `setupConfigRepo(t)` helper with `testDB(t)` + `migrate.Run`.

### Diagrams
- `docs/diagrams/c4-component.md` — Added `pgConfigRepo` component and DB relationship
- `docs/diagrams/c4-code.md` — Added `ConfigRepository` interface, `PostgresConfigRepo` adapter, and implements relationship

## Architecture

```text
domain/config/Repository  ←  infrastructure/postgres/ConfigRepo  ←  cmd/api (CLI commands)
                                        ↓
                                   PostgreSQL (config table)
```
