# PR-036: Checkout Steps — Validate Cart + Recalculate Pricing

## Summary

Introduces the first two concrete checkout steps that plug into the workflow engine from PR-035: `validate_cart` (verifies each cart item's variant still exists in the catalog) and `recalculate_pricing` (runs the pricing pipeline against current cart items and stores the result in checkout context metadata). Both steps are idempotent via `Meta` markers.

## Changes

### Application Layer

- **`internal/application/checkout/validate_cart_step.go`** *(new)* — `ValidateCartStep` implementing `checkout.Step`. Takes `catalog.VariantRepository`. Checks each cart item's variant exists via `FindByID`. Skips re-validation if `Meta["validated"]` is already set. Guards against nil `Cart`.
- **`internal/application/checkout/recalculate_pricing_step.go`** *(new)* — `RecalculatePricingStep` implementing `checkout.Step`. Takes `pricing.Pipeline`. Builds `PricingContext` from cart currency, converts `cart.Item`s to `pricing.PricingItem`s, runs pipeline, stores `*PricingContext` in `Meta["pricing"]`. Skips if `Meta["priced"]` is already set. Guards against nil `Cart`.
- **`internal/application/checkout/steps_test.go`** *(new)* — 12 tests covering both steps.

### Wiring

- **`cmd/api/main.go`** — `validateCartStep` and `recalculatePricingStep` instantiated and registered as ordered steps in `checkoutWorkflow`. Replaces the previous `nil` step slice.

## Design Decisions

- **Idempotency via Meta**: Both steps check a Meta flag before executing, making them safe for retry scenarios.
- **Catalog validation in step, not service**: The service validates basic cart state (active, owned, non-empty). The step validates deeper concerns (variants still exist) that belong to the orchestrated workflow where they can have trace/event observability.
- **Pricing result in Meta**: The `*PricingContext` is stored in `Meta["pricing"]` rather than adding a typed field to `checkout.Context`. This keeps the Context struct stable while later steps (e.g. create_order) can retrieve pricing via `GetMeta("pricing")`.
- **Pipeline by value**: `RecalculatePricingStep` takes `pricing.Pipeline` (value type, not pointer) matching how `Pipeline` is used throughout the codebase.

## Test Results

All 29 packages pass. New tests added in `steps_test.go`:

- **ValidateCartStep** (7 tests): Name, success (all variants exist + meta set), missing variant, repo error, nil cart, idempotent (skip on second call), empty cart
- **RecalculatePricingStep** (5 tests): Name, success (finalize computes subtotal/grand total + meta set), nil cart, idempotent (same result on second call), pipeline error
