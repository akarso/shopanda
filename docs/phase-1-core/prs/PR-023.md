# PR-023 — Inventory Domain + Storage

## Summary

Introduces the inventory domain with a `StockEntry` entity, `StockRepository` interface, PostgreSQL migration for the `stock` table, and a Postgres repository implementation with get/set stock operations.

## Scope

Phase 5 — Inventory (first PR).

## Changes

### Domain: `internal/domain/inventory/`

| File | Description |
|------|-------------|
| `doc.go` | Package documentation |
| `stock.go` | `StockEntry` struct (`VariantID`, `Quantity`, `UpdatedAt`), `NewStockEntry` constructor with validation (non-empty variant ID, non-negative quantity), `IsAvailable()` and `HasStock(needed)` query methods |
| `stock_test.go` | 6 tests: valid entry, zero quantity, empty variant ID, negative quantity, IsAvailable, HasStock |
| `repository.go` | `StockRepository` interface — `GetStock(ctx, variantID)` returns zero-quantity entry when missing, `SetStock(ctx, entry)` upserts |

### Migration: `migrations/004_create_stock.sql`

```sql
CREATE TABLE stock (
    variant_id  UUID PRIMARY KEY REFERENCES variants(id) ON DELETE CASCADE,
    quantity    INT NOT NULL DEFAULT 0 CHECK (quantity >= 0),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

- One row per variant (PK = variant_id)
- `CHECK (quantity >= 0)` prevents negative stock at DB level
- Cascades on variant deletion

### Infrastructure: `internal/infrastructure/postgres/stock_repo.go`

| Operation | SQL |
|-----------|-----|
| `GetStock` | `SELECT ... WHERE variant_id = $1` — returns zero-quantity `StockEntry` on `sql.ErrNoRows` |
| `SetStock` | `INSERT ... ON CONFLICT (variant_id) DO UPDATE SET quantity, updated_at` |

### Tests: `internal/infrastructure/postgres/stock_repo_test.go`

4 integration tests (require `SHOPANDA_TEST_DSN`): get with no record, set/create, set/update, nil entry rejection. Uses `seedVariant` helper to create product + variant fixtures.
