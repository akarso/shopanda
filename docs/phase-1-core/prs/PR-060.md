# PR-060 — Wire Email to Order Events

## Summary

Wires `order.paid` events to email confirmation delivery via the job queue. Renders a template, enqueues an `email.send` job, and processes it through the worker with the SMTP mailer.

## Changes

### Application (`internal/application/notification/`)

- **`service.go`** — `NotificationService`: listens to `order.paid`, looks up order + customer, renders `order.confirmed` template, enqueues `email.send` job. `RegisterTemplates()` registers the confirmation email template.
- **`handler.go`** — `EmailSendHandler`: implements `jobs.Handler` for `email.send` job type. Extracts `to`/`subject`/`body` from payload, calls `mailer.Send(ctx, msg)`. Validates `to` is present.
- **`notification_test.go`** — 5 tests: happy path (order.paid → enqueued job with correct payload), order not found, bad event data, handler send, handler missing-to validation. Compile-time interface checks.

### Wiring (`cmd/api/main.go`)

- Created `mail.Templates` + registered notification templates at startup.
- Created `smtp.Mailer` from `cfg.Mail.SMTP.*` config.
- Created `notification.Service` with customerRepo, orderRepo, jobQueue.
- Registered `EmailSendHandler` with the job worker.
- Registered `bus.OnAsync(order.EventOrderPaid, notifSvc.HandleOrderPaid)`.

### Diagrams

- **`c4-code.md`** — Added `NotificationService` and `EmailSendHandler` classes with relationships to Templates, Queue, Mailer, Handler.
- **`c4-component.md`** — Added `NotificationService` component in Application boundary with event bus, repository, and job queue relationships.

## Design

```go
// Flow: order.paid event → NotificationService → email.send job → Worker → EmailSendHandler → SMTP
//
// 1. Checkout publishes order.paid via EventBus
// 2. bus.OnAsync dispatches to NotificationService.HandleOrderPaid
// 3. Service looks up Order (for CustomerID) → Customer (for email)
// 4. Renders "order.confirmed" template with order/customer data
// 5. Enqueues email.send job with {to, subject, body} payload
// 6. Worker dequeues → EmailSendHandler.Handle → mailer.Send
```

Templates are rendered before enqueue (per MAILER.md spec). The Mailer never touches domain logic — it only delivers pre-built messages. Retry is handled by the job queue's exponential backoff (PR-057).
