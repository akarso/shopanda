# PR-025 — Cart Domain

## Summary

Introduces the cart domain: `Cart` aggregate with embedded `Item` collection,
`CartRepository` interface, status lifecycle, and domain events.

## Changes

### Domain (`internal/domain/cart/`)

| File | Description |
|------|-------------|
| `cart.go` | `Cart` entity with `ID`, `CustomerID` (optional), `Status`, `Currency`, `Items`. Constructors, `AddItem` (merge-on-duplicate), `UpdateItemQuantity`, `RemoveItem`. All mutations reject non-active carts. |
| `item.go` | `Item` value object with `VariantID`, `Quantity`, `UnitPrice` (`shared.Money`). `LineTotal()` via `MulChecked`. |
| `repository.go` | `CartRepository` interface — `FindByID`, `FindActiveByCustomerID`, `Save`, `Delete`. |
| `events.go` | Event constants: `cart.item.added`, `cart.item.updated`, `cart.item.removed` + payload structs. |
| `cart_test.go` | 19 unit tests covering construction, add/update/remove/merge, currency mismatch, non-active guards, status validation. |
| `item_test.go` | 5 unit tests covering construction, validation, and `LineTotal`. |

### Shared (`internal/domain/shared/`)

| File | Description |
|------|-------------|
| `money.go` | Exported `IsValidCurrency()` (was `isValidCurrency`) so the cart domain can validate currency codes at construction time. |

### Cart Status Lifecycle

```text
active → checked_out   (checkout begins)
active → abandoned     (timeout / explicit abandon)
```

### Design Decisions

- **Guest carts**: `CustomerID` is optional (empty string). `SetCustomerID` assigns after login.
- **Merge-on-add**: Adding the same variant ID increments quantity rather than creating a duplicate line.
- **Currency validation**: Uses `shared.IsValidCurrency()` at cart creation to enforce ISO 4217.
- **UnitPrice on Item**: Cached from pricing pipeline; not a source of truth (recalculated on checkout).
- **No DB layer**: PR-026 will add Postgres storage + migration.
