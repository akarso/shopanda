# PR-081 ‚Äî Tax Domain + Rate Storage

## Summary

Introduces the tax domain for Phase 20 (Taxes). Adds `TaxClass`, `TaxRate`,
`TaxMode`, and a pure `Calculate` function for deterministic tax computation
using integer arithmetic (basis-point rates). Includes the Postgres repository
implementation and migration.

## Domain Model

### TaxClass

Simple value object identifying a tax classification (e.g. `"standard"`,
`"reduced"`, `"zero"`).

| Field | Type | Description |
|-------|------|-------------|
| Code | string | Unique class identifier |

### TaxRate

| Field | Type | Description |
|-------|------|-------------|
| ID | string | UUID |
| Country | string | ISO 3166-1 alpha-2 (e.g. `"DE"`) |
| Class | string | References a TaxClass code |
| Rate | int | Basis points (2100 = 21.00%) |

### TaxMode

| Value | Meaning |
|-------|---------|
| `exclusive` | Tax added on top of price |
| `inclusive` | Tax already included in price |

### Calculate

Integer arithmetic ‚Äî no floating point.

- **Exclusive:** `tax = amount √ó rate / 10000`
- **Inclusive:** `net = amount √ó 10000 / (10000 + rate)`, `tax = amount ‚àí net`

## Repository Interface

```go
type RateRepository interface {
    FindByCountryAndClass(ctx, country, class) (*TaxRate, error)
    ListByCountry(ctx, country) ([]TaxRate, error)
    Upsert(ctx, *TaxRate) error
    Delete(ctx, id) error
    WithTx(tx *sql.Tx) RateRepository
}
```

## Migration

`021_create_tax_rates.sql` ‚Äî `tax_rates` table:

- `id` UUID primary key
- `country` TEXT (CHECK: 2 uppercase letters)
- `class` TEXT (CHECK: non-empty)
- `rate` INTEGER (CHECK: ‚â• 0)
- UNIQUE(country, class)
- Index on `country`
