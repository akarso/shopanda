# PR-095 — Scoped Pricing + Tax

## Summary

Prices and tax rates are now scoped per store. Each price is keyed by
`(variant_id, currency, store_id)` and each tax rate by
`(country, class, store_id)`. An empty `store_id` represents the
global/default entry, preserving full backward compatibility.

When a store-scoped price or rate is not found, the system falls back
to the global entry automatically.

## Changes

### Migrations

- `migrations/029_add_store_id_to_prices.sql` — adds `store_id TEXT NOT NULL DEFAULT ''` to `prices`, replaces `UNIQUE(variant_id, currency)` with `UNIQUE(variant_id, currency, store_id)`.
- `migrations/030_add_store_id_to_tax_rates.sql` — adds `store_id TEXT NOT NULL DEFAULT ''` to `tax_rates`, replaces `UNIQUE(country, class)` with `UNIQUE(country, class, store_id)`.

### Domain

- `Price` entity: added `StoreID string` field; `NewPrice` now accepts `storeID` parameter.
- `PriceRepository.FindByVariantAndCurrency` → `FindByVariantCurrencyAndStore(ctx, variantID, currency, storeID)`.
- `TaxRate` entity: added `StoreID string` field; `NewTaxRate` now accepts `storeID` parameter.
- `RateRepository.FindByCountryAndClass` → `FindByCountryClassAndStore(ctx, country, class, storeID)`.

### Infrastructure (Postgres)

- `PriceRepo`: all SQL queries include `store_id` column; `ON CONFLICT` uses the new three-column unique constraint; ordering includes `store_id`.
- `TaxRateRepo`: all SQL queries include `store_id` column; `ON CONFLICT` uses `(country, class, store_id)`.

### Application

- `BasePriceStep`: reads `store_id` from `PricingContext.Meta`; falls back to global price when store-scoped price not found.
- `TaxStep`: reads `store_id` from `PricingContext.Meta`; falls back to global rate when store-scoped rate not found.
- `Cart service`: sets `pctx.Meta["store_id"]` from `store.FromContext(ctx)` during recalculate; `lookupPrice` uses store-scoped lookup with global fallback.
- `Checkout RecalculatePricingStep`: forwards `store_id` alongside tax keys from checkout to pricing context.
- `PriceImporter`: supports optional `store_id` CSV column (defaults to `""` when absent).
- `PriceExporter`: writes `store_id` column in CSV output.

### Composition

- `ProductContext`: added `StoreID string` field.
- `JSONLDProductStep`: passes `ctx.StoreID` to store-scoped price lookup.
- HTTP `ProductHandler` and `StorefrontHandler`: populate `StoreID`, `Currency`, and `Country` on `ProductContext` from the resolved store.

### Diagrams

- `docs/diagrams/c4-code.md`: added `StoreID` field to `Price` class.

## Design Decisions

| Decision | Rationale |
|---|---|
| Empty `store_id = ""` is global/default | Backward compatible; existing data works without migration of values |
| Automatic fallback to global | Store operators don't need to duplicate every price/rate |
| `store_id` in Meta map | Consistent with existing `tax_country`/`tax_mode` pattern |
| CSV `store_id` column is optional | Importers using old format continue to work |
