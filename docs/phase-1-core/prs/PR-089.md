# PR-089 ‚Äî URL Rewrite System

## Phase
Phase 23 ‚Äî URL Routing & CMS

## Scope
`url_rewrites` table, migration, resolver middleware, catch-all route.

## Files

### Domain
- `internal/domain/routing/rewrite.go` ‚Äî `URLRewrite` entity (path, type, entity_id) with constructor + validation
- `internal/domain/routing/repository.go` ‚Äî `RewriteRepository` interface (FindByPath, Save, Delete)
- `internal/domain/routing/context.go` ‚Äî Context helpers: `WithRewrite`, `RewriteFrom`

### Infrastructure
- `migrations/026_create_url_rewrites.sql` ‚Äî `url_rewrites` table with `path` PK and `entity_id` index
- `internal/infrastructure/postgres/rewrite_repo.go` ‚Äî PostgreSQL implementation with UPSERT support

### Interfaces
- `internal/interfaces/http/rewrite_middleware.go` ‚Äî `ResolverMiddleware`: looks up path in rewrite repo, injects URLRewrite into context
- `internal/interfaces/http/rewrite_handler.go` ‚Äî `RewriteHandler.Resolve()`: catch-all handler that returns resolved entity type + ID

### Tests
- `internal/domain/routing/routing_test.go` ‚Äî 7 tests: constructor validation (4), FromDB reconstruction (1), context round-trip (2)
- `internal/interfaces/http/rewrite_test.go` ‚Äî 5 tests: middleware match/no-match/error (3), handler resolve/missing (2)

## Design Decisions
- Middleware passes through to next handler when no rewrite is found (lets existing routes work normally)
- Catch-all handler returns entity type + ID as JSON; upstream consumers (API composers, frontend) decide how to render
- Context pattern follows existing `auth.WithIdentity`/`auth.IdentityFrom` convention
- UPSERT on Save allows both insert and update of rewrites via single method

## R1 Review

### Findings (all verified and fixed)

1. **`repository.go`: duplicate package declaration** ‚Äî File had `package routing` twice, causing a compile error. Removed the duplicate.

2. **`rewrite_repo.go`: flattened pq.Error handling** ‚Äî `errors.As(err, &pqErr)` branch discarded structured info (Code, Constraint) and only used `pqErr.Message`. Now checks `pqErr.Code == "23505"` and returns `apperror.Conflict` for unique-constraint violations, matching the pattern in `customer_repo`, `variant_repo`, `shipping_repo`, etc. Falls back to generic `%w` wrap otherwise.

3. **`rewrite_middleware.go`: incorrect doc comment** ‚Äî Comment claimed "a 404 is returned" on no match, but code forwards to `next`. Updated to: "the request is forwarded to next to preserve existing routes".

4. **`rewrite_middleware.go`: plain `http.Error` instead of `JSONError`** ‚Äî Replaced `http.Error(w, ...)` with `JSONError(w, apperror.Internal(...))` for consistent API response envelope.

5. **`026_create_url_rewrites.sql`: missing timestamps** ‚Äî Added `created_at TIMESTAMPTZ NOT NULL DEFAULT now()` and `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()` to match project convention. Updated UPSERT query in `rewrite_repo.go` to set `updated_at = now()` on conflict. No down migration added (none exist in the project).
