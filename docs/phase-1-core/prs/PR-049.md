# PR-049 — Pipeline Registration API

## Summary

Adds `RegisterPricingStep()`, `RegisterCheckoutStep()`, and `RegisterCompositionStep()` to the plugin `App` struct, allowing plugins to inject steps into the three pipeline systems during initialization.

## Changes

### Modified files

| File | Change |
|------|--------|
| `internal/platform/plugin/plugin.go` | Registration methods + getters on `App` |
| `cmd/api/main.go` | Restructured wiring: plugin init before pipelines; plugin steps integrated |

### New files

| File | Purpose |
|------|---------|
| `internal/platform/plugin/app_test.go` | 10 unit tests for registration API |

## Design

### Registration API

```text
App.RegisterPricingStep(step any)                    — pricing.PricingStep
App.RegisterCheckoutStep(step any)                   — checkout.Step
App.RegisterCompositionStep(pipeline string, step any) — composition.Step[T]
```

Steps are stored as `any` so the plugin package avoids importing domain/application types. Type assertions happen in main.go when building the typed pipelines.

### Getters

```text
App.PricingSteps() []any
App.CheckoutSteps() []any
App.CompositionSteps(pipeline string) []any
```

### Wiring order (main.go)

```text
repos → providers → event bus → plugin init → composition (core + plugin)
→ pricing (core + plugin + finalize) → services → checkout (core + plugin)
```

Plugin init now runs BEFORE pipeline construction so registered steps are available.

### Pipeline integration

- **Pricing**: plugin steps inserted between `BasePriceStep` and `FinalizeStep`
- **Checkout**: plugin steps appended after core steps
- **Composition**: plugin steps added via `AddStep()` on PDP/PLP pipelines, keyed by pipeline name (`"pdp"`, `"plp"`)
