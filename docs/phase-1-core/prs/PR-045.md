# PR-045 — Shipping Domain + Storage

## Summary

Adds the shipping domain: `Shipment` entity, `ShippingMethod`/`ShippingStatus`
enums, `Provider` interface for rate calculation, `ShipmentRepository` interface,
PostgreSQL implementation, migration, and domain events.

## Design

### Entity: Shipment

```text
Shipment {
  ID, OrderID, Method, status, Cost, TrackingNumber, ProviderRef, CreatedAt, UpdatedAt
}
```

- `Cost` uses `shared.Money` (allows zero for free shipping, rejects negative)
- `status` is unexported with `Status()` getter
- `Currency()` derived from `Cost.Currency()`

### Status Lifecycle

```text
pending → shipped → delivered
pending → cancelled
```

| Transition | Method | Guard |
|------------|--------|-------|
| `Ship(trackingNumber, providerRef)` | pending → shipped | sets tracking + ref |
| `Deliver()` | shipped → delivered | — |
| `Cancel()` | pending → cancelled | — |

### Enums

- **ShippingStatus**: `pending`, `shipped`, `delivered`, `cancelled`
- **ShippingMethod**: `flat_rate` (extensible for future providers)

### Provider Interface

```go
type Provider interface {
    Method() ShippingMethod
    CalculateRate(ctx, orderID, currency, itemCount) (ShippingRate, error)
}
```

`ShippingRate` contains `ProviderRef`, `Cost` (shared.Money), and `Label`.

### Repository

- `FindByID`, `FindByOrderID` — return `(nil, nil)` when not found
- `Create` — detects unique constraint violation (pq 23505) → `apperror.Conflict`
- `UpdateStatus` — optimistic locking via `prevUpdatedAt`

### Events

| Event | Data Struct |
|-------|-------------|
| `shipment.created` | `ShipmentCreatedData` (shipment_id, order_id, method, cost, currency) |
| `shipment.shipped` | `ShipmentStatusChangedData` (shipment_id, order_id, old/new status, tracking, ref) |
| `shipment.delivered` | `ShipmentStatusChangedData` |
| `shipment.cancelled` | `ShipmentStatusChangedData` |

### Migration (013)

- `shipments` table with `order_id UNIQUE`, status CHECK constraint, `cost >= 0` CHECK
- `shipments_status_idx` index
