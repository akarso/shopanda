# PR-305 — First-Boot Store and Tax Defaults

## Summary

Made the default `migrate -> seed -> serve` path work without requiring merchants to manually create a store, configure tax metadata, or provide an admin password before the storefront can function.

This pass adds real first-boot defaults instead of relying on runtime failures and manual follow-up steps. The seeded defaults are intentionally coherent with each other: a default storefront context is created, a baseline standard tax rate is created for that store country, the seeded catalog currency matches the seeded store currency, and cart pricing can still calculate tax in storefront flows before checkout has supplied explicit tax metadata.

## Scope

### Default store and tax seeders

- added `internal/seed/store.go`
- added `internal/seed/tax.go`
- registered both in the default seeder pipeline from `cmd/api/main.go`
- seeded a default store using Polish-oriented defaults:
  - code `default`
  - name `Sklep Domyslny`
  - country `PL`
  - language `pl`
  - currency `PLN`
- seeded a default global standard tax rate for the default store country
- kept both seeders idempotent so existing stores or existing tax rates are not overwritten

### Seeded currency alignment

- updated `internal/seed/config.go`
- updated `internal/seed/catalog.go`
- changed the seeded default currency from `EUR` to `PLN`
- aligned the seeded product prices with the seeded storefront currency so the sample catalog and store configuration are internally consistent from first boot

### Optional admin seeding

- updated `internal/seed/admin.go`
- changed admin creation from a hard failure into an idempotent skip when `SHOPANDA_SEED_ADMIN_PASSWORD` is not set
- preserved seeded admin creation when the env var is present and `admin@example.com` does not yet exist
- updated deployment and merchant documentation to describe the new optional admin-seeding behavior

### Storefront cart tax fallback

- updated `internal/application/cart/service.go`
- when storefront cart pricing runs with a resolved store but without explicit tax metadata, it now:
  - seeds `tax_country` from the current store country
  - defaults `tax_mode` to `exclusive`
- kept explicit pricing metadata untouched so checkout can still override tax behavior later
- this ensures add-to-cart works on a fresh seeded install instead of failing before checkout because tax metadata is missing

### Regression coverage

- updated `internal/seed/seeders_test.go`
- updated `internal/application/cart/service_test.go`
- added coverage for the new store and tax seeders in the registry
- added focused coverage proving cart add works with store-scoped tax defaults when the tax step is active

## Notes

- this PR is about safe defaults and first-boot usability, not merchant-specific production configuration
- missing tax rates still behave as zero-rated once the storefront has a tax country to price against; this PR ensures the default seeded store provides that baseline country context from the start
- the seeded defaults are intentionally not US-centric; the out-of-box path now starts from a Polish store profile instead of an implicit US assumption
