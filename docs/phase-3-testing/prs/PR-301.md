# PR-301 — Database URL Escaping and Runbook Redirect

## Summary

Hardened PostgreSQL DSN handling for passwords with reserved URL characters and simplified the top-level runbook into a redirect to the maintained guides.

This PR fixes a real startup failure where database passwords containing characters like `%` caused `DATABASE_URL` parsing to fail before the driver could connect.

## Scope

### Database configuration hardening

- updated `internal/platform/config/config.go`
- changed `DatabaseConfig.DSN()` to build PostgreSQL URLs via Go's URL types instead of raw string concatenation
- normalized `DATABASE_URL` environment overrides before passing them to the DB layer
- repaired malformed userinfo in `DATABASE_URL` when credentials contain reserved characters that were entered unescaped

### Regression coverage

- updated `internal/platform/config/config_test.go`
- added coverage for structured DB config with reserved characters in username and password
- added coverage for raw `DATABASE_URL` overrides containing invalid userinfo escapes

### Runbook cleanup

- reduced `RUNBOOK.md` to a short pointer page
- redirected operators to the maintained guides in `docs/guides/` instead of duplicating deployment and operations content at the repo root

## Files Changed

- `internal/platform/config/config.go`
  - safe PostgreSQL DSN construction and `DATABASE_URL` normalization
- `internal/platform/config/config_test.go`
  - regression tests for reserved-character database credentials
- `RUNBOOK.md`
  - replaced duplicated operational content with links to the guide set

## Validation

- `go test ./internal/platform/config -count=1`

## Notes

- this change preserves `DATABASE_URL` precedence while making it tolerant of common unescaped password input from local `.env` files and manual setup flows
- no architecture or diagram changes were required because the fix stays within existing configuration behavior
