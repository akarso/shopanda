# PR-232 — Developer Extension Guide

## Summary

Added the developer-facing extension guide for engineers customizing Shopanda.

The new guide documents the real extension surfaces that exist in the current codebase instead of describing hypothetical plugin capabilities:

- plugin registry and plugin app contract
- payment, shipping, storage, and search-facing ports
- pricing, composition, and checkout extension hooks
- sync and async event listeners
- API reference and authentication for extension testing
- the current limitation that CLI commands are still registered explicitly in `cmd/api/main.go`

## Scope

### Developer documentation

- added `docs/guides/DEVELOPER.md`
- documented the actual plugin contract in `internal/platform/plugin`
- documented the current explicit bootstrap flow in `cmd/api/main.go`
- added code examples for:
  - plugin initialization
  - payment providers
  - shipping providers
  - storage backends
  - pipeline steps
  - event listeners

### Accuracy over aspiration

- documented that plugin registration is compile-time and explicit, not dynamic discovery
- documented that invalid plugin step types are logged and skipped during bootstrapping
- documented that CLI command extension is currently a code-level pattern rather than a plugin registry
- documented the real API docs endpoints:
  - `/docs`
  - `/docs/openapi.yaml`

### Discoverability

- added a `Developer Guide` link to the main `README.md`

## Files Changed

- `docs/guides/DEVELOPER.md`
  - new extension guide for developers and plugin authors
- `README.md`
  - added a direct link to the developer guide

## Notes

- the guide deliberately prefers the current codebase contract over the looser wording in the roadmap spec, especially around plugin registration and CLI commands
- this PR stays within the documentation track only and does not change runtime behavior
